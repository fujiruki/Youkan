取扱説明書 (User Manual)

システムを開く

JBWOSの操作方法、概念、そして「思考のフロー」を解説します。

⚡ クイックスタート (最初の5分)

  1. 1

    すべてをInboxへ

    「Stream」画面の入力欄に、今頭にある"やること"、"悩み"、"単なるメモ"を全部書き出します。
    Enterで連続投稿できます。まずは頭を空っぽにしましょう。

  2. 2

    詳細を開く

    リストのアイテムをクリック（またはAlt+D）して詳細画面を開きます。

  3. 3

    Yes / No / Hold

    「今日やる？」と自問自答してください。
    Yesなら「Focus」へ。
    Holdなら「後でやる（Pending）」か「待ち（Waiting）」へ。
    これでInboxが空になれば、今日のセットアップは完了です。

状態 (Status) の定義

JBWOSは、タスクを以下の5つの状態だけで管理します。複雑なタグ管理は不要です。

状態 (ID) 表示名 あなたの認識 システム上の扱い
inbox 受信 思いつき。未判断。 Stream画面の左側に滞留。
focus 今日やる 今日実行する意志がある。 Stream画面の中央（メイン）に表示。完了対象。
waiting 待機 (Waiting) 他者からの返信待ち、または開始日待ち。 自動的に非表示になり、条件を満たすと再浮上。
pending 保留 (Someday) いつかやる。今はやらない。 視界から消してよい。「棚」にしまわれる。
done 完了 終わった事実。 履歴（Log）として保存される。

※ 内部コード上の status="pending" が、ユーザーから見た "Someday (いつかやる)" に相当します。
※ 内部コード上の status="waiting" が、ユーザーから見た "Waiting (待ち)" に相当します。

4つの主要画面 (Views)

Stream (登録と集中)

日々のメイン画面。「今日やること(Focus)」と「Inbox」しかないコックピットです。

Panorama (全体一覧)

プロジェクトの進行状況を時系列（ガントチャート風）で確認する画面。「俯瞰」用です。

Newspaper (全体一覧２)

タスクを「情報の塊」として新聞のように並べる画面。直感で「量」を感じ取るのに適しています。

Calendar (カレンダー)

旅館の予約台帳のようなリソース管理画面。あなたの「空き時間（部屋）」の状況を確認します。

ケース別ガイド (How-to)

Case 1. 思いついたタスクを一瞬で放り込む

状況: 作業中、ふと「あ、銀行行かなきゃ」と思い出した。

操作:

  1. Stream画面を開く（または常に開いておく）。
  2. 「思いついたことを入力...」欄に「銀行へ行く」とタイプして Enter。
  3. 以上。

結果: タスクは Inbox に入ります。今の作業を中断せず、忘却できます。

Case 2. 途中で別タスクを思いついたときの扱い

状況: プロジェクトAの作業中、プロジェクトBのアイデアが降ってきた。

判断: 「今すぐやる？」→ No（集中を切らさないため）

操作:

  1. Stream画面の下部「Quick Input」へ入力。
  2. プロジェクトBに関連するなら、作成後に詳細画面を開き、Project欄を変更してもよいですが、今はただInboxに入れるだけで十分です。
  3. 現在の作業に戻ります。

コツ: 整理整頓は「今」やらないでください。それは未来のあなたの仕事です。

Case 3. 今日やることを決めて実行する

状況: 朝一番、または昼休憩後。

操作:

  1. Stream画面のInboxにあるタスクを上から順に見ます。
  2. 詳細を開き（クリック）、問います。「今日やる？」
  3. Yesなら、モーダルの「今日やる (Today)」ボタンを押す。 → Focusへ移動。
  4. Noなら、保留(Someday)か待機(Waiting)へ。
  5. Focusリストに残ったタスクを上から順に片付けます。

Case 4. プロジェクトで束ねて進める

状況: 「展示会の準備」のように、複数のタスクが含まれる大きな仕事がある。

操作:

  1. 画面右上の「＋プロジェクト」ボタンを押してプロジェクトを作成します。
  2. タスクの詳細画面で、Project プルダウンからそのプロジェクトを選択します。
  3. Panorama画面（全体一覧）を見ると、そのプロジェクト配下にタスクがまとまって表示されます。

思想: プロジェクトの中にプロジェクトを入れる（入れ子）機能もありますが、複雑化は敵なので推奨しません。フラットに保ちましょう。

Case 5. 期限/納期/見積りが絡むときの扱い

状況: クライアントへの提出期限は来週金曜だが、自分としては水曜までに終わらせたい。

操作 (詳細画面にて):

  • 納期 (Due Date): 来週金曜（外部制約）。 ※遅れると信頼に関わる日付。
  • My期限 (Prep Date): 水曜（内部目標）。 ★★ JBWOSではこちらを重視します。
  • 見積 (Estimate): 「2h」「0.5d」などのプリセットを選択。 ※Ryokan Calendar上で、そのタスクがどれくらいの部屋（時間枠）を占有するかが可視化されます。

📚 用語集 (Glossary)

Someday (いつかやる)

システム上のステータスは Pending。重要だが緊急ではないタスクの保管庫。定期的にPanorama画面で見返して、腐っていないか確認します。

Hold (判断保留)

詳細画面におけるアクションの一つ。「今、Yes(今日やる)とは言えない」状態。これを選ぶと、タスクはWaitingかPendingのどちらかに振り分けられます。

Ryokan Calendar

工数管理機能付きカレンダー。タスクが消費する時間（宿泊客）を、あなたの可用時間（部屋）に割り当てるイメージから命名されました。
