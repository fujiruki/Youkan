# 2026-06-02 会議 — Google カレンダー連携

## 参加者

- 晴樹さん（PM）
- 指揮 AI

## 議題

Youkan に Google カレンダーを統合し、量感（キャパシティ）計算と番頭 AI の判断に外部予定を反映する。

## 元会議結論（Phase 構成）

4 Phase 段階リリースで合意。

| Phase | 内容 | 目的 |
|:--|:--|:--|
| Phase 1 | 量感カレンダー各セル下端に進捗棒グラフ追加（R-034 Phase 1, R-035 と一体実装） | 進捗の可視化基盤 |
| Phase 2 | primary Google Calendar の読み取り連携（`calendar.readonly`）。外部予定を量感計算に加算 | 1 ユーザーで外部予定を取り込む最小ループ |
| Phase 3 | 共有カレンダー＋テナント割当（個人 / 会社 / 無視のラジオ振り分け） | 法人タスクへの紐付け |
| Phase 4 | 番頭 AI へ external_events を渡し、判断材料に組み込む | AI 判断の精度向上 |

## 元会議のセキュリティ・プライバシー方針

- **タイトル・参加者・場所は保存しない**（DB に置かない）。家族・他社の機密プライバシーリスクを避ける
- 保存するのは `{start, end, all_day, calendar_id, event_id}` のみ
- リフレッシュトークンは **AES-256-GCM** で暗号化して保管
- スコープは `calendar.readonly`（読み取り専用）
- 終日イベントの量感換算: ユーザー設定 1 値、デフォルト 240 分（半日相当）
- キャッシュ TTL 15 分、手動更新ボタンは 60 秒クールダウン

## 元会議の OAuth 方針

- Google Cloud Console で OAuth Client 作成、unverified app 状態で開始
- 初期は晴樹さん専用、後に verification 申請を検討
- 認証起点は「設定画面」想定（具体的な UI は Phase 2 着手時に詰める）

### 維持される方針（元会議結論のまま）

- リフレッシュトークン暗号化（AES-256-GCM）
- Phase 3 で共有カレンダー＋テナント割当（Phase 2 では UI 枠だけ用意）
- Phase 4 で番頭 AI に external_events 渡し
- 完了アイテム表示統一（R-035）の対象**外**（Google イベントはタスクではないためグレー化しない）

### Phase 2 で確定した詳細

1. **表示形式**:
   - 通常イベント: `📅 HH:MM タイトル`（例: `📅 10:00 会議`）
   - 終日イベント: `📅 タイトル`（時刻なし）
   - 1 日のイベントが多い場合: 最初の N 件（推奨 3 件）＋「他 X 件」リンク
   - タップでイベント詳細モーダル（タイトル / 時間 / 場所 / Google で開くリンク）
2. **量感計算**: 所要時間（`end - start`、終日は 240 分）を分子に加算。CapacityBar に反映
3. **DB 保存**:
   - `user_google_oauth`: 暗号化リフレッシュトークン、primary カレンダー情報、最終同期時刻
   - `external_events_cache`: `title`, `location` 含むキャッシュ（プライバシー方針変更後）
4. **API**:
   - `POST /api/google/oauth/start`
   - `GET /api/google/oauth/callback`
   - `POST /api/google/calendar/refresh`（60 秒クールダウン）
   - `DELETE /api/google/oauth`（連携解除）
   - `GET /api/calendar/grid?from&to` に `external_events` フィールド追加
5. **設定画面**: 未連携時は「Google でログイン」ボタン、連携済みはメール・最終同期・更新・解除を表示。共有カレンダー一覧の振り分け UI（個人 / 会社 / 無視）は Phase 3 用に枠だけ用意。
