# 専門家会議議事録: JBWOS Global Board UX Redesign

**日時**: 2026/01/19  
**場所**: Virtual Meeting Room  
**テーマ**: 「見えない不安」を解消する、Global Boardの表示モード再設計  
**参加者**:
- **Antigravity (PM)**: 司会・進行。ユーザーの要望（一覧性、切替機能、余白排除）を代弁。
- **Leo (The Architect)**: UI/UXデザイナー（構造・論理担当）。グリッドシステムと情報設計の専門家。
- **Sarah (The Therapist)**: UI/UXデザイナー（感情・心理担当）。ユーザーの心理的安全性と「心地よさ」の専門家。

---

## 1. 議題提起

**Antigravity (PM)**:
お二方、集まっていただきありがとうございます。今回のクライアントからの要望は非常に本質的です。
「JBWOSのトップ画面（Global Board）で、今抱えているタスクの全体像が見えないことに不安を感じる」
「左右のカラム分けもいいが、もっと根源的に『画面全体に全部を表示』して、頭の中を整理したいのかもしれない」
「今の表示（縦積み）も嫌いではないので、切り替えられるようにしたい」

これらを踏まえ、**「一覧性」と「安心感」を両立するUI**を定義してください。

## 2. ディスカッション：不安の正体と解決策

**Sarah (Emotion)**:
ユーザーの言葉にある「本当に全部このシステムに入力できてるんだろうか？」という不安、これは**「隠れている情報（Off-screen information）」への恐怖**ですね。
スクロールしないと見えない場所にある情報は、脳のワーキングメモリに負荷をかけます。「見えていない場所に重要な何かが潜んでいるかもしれない」という疑念が、集中力を削ぐんです。

**Leo (Logic)**:
その通りだ。現在の「Standard View（縦積み）」は、ストーリーテリングや各アイテムの熟考には向いているが、ダッシュボードとしての「計器性」が低い。
解決策は明確だ。**スクロールを排除し、全てのバケット（Active, Preparation, Intent, Log）を一画面に収める「Panorama Mode」**が必要だ。

**Antigravity (PM)**:
その「Panorama Mode」ですが、具体的にどう実現しますか？ 情報量は多いですよ。

**Leo (Logic)**:
**「高密度グリッド（Super Compact Grid）」**を採用する。
余白（Padding/Margin）は「美しさ」のために使われるが、今回の目的は「把握」だ。美しさの種類を変える必要がある。
航空機のコックピットや、トレーダーの画面を見てほしい。余白は最小限だが、情報は整然と並んでいる。あの**「機能美」**を目指す。
具体的には：
1.  **Grid Layout**: 画面を均等、あるいは黄金比で分割し、全バケットを配置する。
2.  **Typography**: フォントサイズを `text-sm` (14px) から `text-xs` (12px) へ落とし、行間も詰める。
3.  **Visual Noise Reduction**: カードの影や角丸を最小限にし、境界線（Border）で見せる。

**Sarah (Emotion)**:
待ってLeo、ただ詰め込むだけだと「圧迫感」が出て、逆にストレスになりませんか？
「高密度」でも「安心」できるための工夫が必要です。
1.  **Muted Colors**: Stock領域（Preparation以下）は、彩度を落とし、背景に馴染ませることで「そこにあるけど、主張しない」状態を作る。
2.  **Clear Boundaries**: 各エリアの境界線ははっきりと引き、「ここからここまでがActive」というテリトリー感を明確にする。これで脳は情報をチャンク（塊）として処理できるわ。

## 3. 具体的なUI仕様策定

**Antigravity (PM)**:
素晴らしい。では「切替機能」について整理しましょう。

### Mode A: Standard View (Current)
- **レイアウト**: 縦積み (Single Column)
- **特徴**: ゆったりとした余白。一つ一つのタスクと向き合う。
- **用途**: 朝の計画時、タスク実行時、スマホ利用時。

### Mode B: Panorama View (New)
- **レイアウト**: 4カラム・フルワイド・グリッド
    - Col 1: **Active** (Inbox)
    - Col 2: **Preparation** (Standby)
    - Col 3: **Intent** (Someday)
    - Col 4: **Log** (History)
- **デザイン**: **Super Compact**
    - `p-1` or `p-2` (極小パディング)
    - カードの背景色は薄く、ボーダーで区切る。
    - アイテム内の日付やタグはアイコン化してスペース節約。
- **用途**: 全体俯瞰、週次レビュー、頭の整理、「抜け漏れがないか」の確認時。

**Leo (Logic)**:
技術的には、親コンテナのクラスを切り替えるだけで実装可能だ。
`viewMode === 'panorama'` の時は `grid grid-cols-4 gap-2 h-full overflow-hidden` にし、各カラムは `h-full overflow-y-auto` にすれば、全体スクロールではなく「カラム内スクロール」になる。これでヘッダーやフッターが隠れることもない。

**Sarah (Emotion)**:
その切り替えボタンですが、「田（グリッド）」と「≡（リスト）」のようなありきたりなアイコンでいいのかしら？
ユーザーのメンタルモデルに合わせて、**「Focus（集中）」**と**「Bird's-eye（鳥の目）」**のようなメタファーがいいかもしれないわね。まあ、アイコンはシンプルでいいけど、ツールチップでその意図を伝えましょう。

## 4. 結論

**Antigravity (PM)**:
議論がまとまりました。方針は以下です。

1.  **View Switcher**: ヘッダーに「Standard (縦)」と「Panorama (横一覧)」の切替ボタンを設置。
2.  **Panorama Mode**:
    -   4カラムグリッドで全バケットを一挙表示。
    -   **Super Compact UI**: 余白と文字サイズを削ぎ落とし、一覧性を極限まで高める。
    -   **Visual Hierarchy**: Activeを強調しつつ、他は「保管庫」として静かに表示。

この方針で実装に入ります。ありがとうございました。
