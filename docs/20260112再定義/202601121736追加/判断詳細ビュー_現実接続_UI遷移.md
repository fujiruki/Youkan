# 判断詳細ビュー_現実接続_UI遷移.md
最終更新: 2026-01-12 (JST)

本ドキュメントは、**判断詳細ビュー**を中心に、
納期・相手・現実条件を確認しつつも、圧を生まないための
UI構成・画面遷移・操作フローを最終固定する。

---

## 1. 画面構成（全体）

```
Today
 └ 判断カード（折りたたみ）
     └ タップ
        ↓
   判断詳細ビュー
```

---

## 2. Today画面（折りたたみ状態）

### 表示
```
◯◯建具｜製作開始を判断する
```

### 非表示（重要）
- 納期日
- 残日数
- 締切表現
- 警告色

---

## 3. 判断詳細ビュー（展開後）

### レイアウト（上→下）

```
【判断対象】
◯◯建具｜製作開始を判断する

【判断材料】
・取付日：10/30
・想定製作日数：5日
・材料リードタイム：3日
・バッファ：2日

（参考）
・RDD_start：10/23

【選択】
[ Yes ] [ Hold ] [ No ]
```

---

## 4. 表示ルール

- 色：通常テキストのみ
- 強調：なし
- 「あと◯日」表現：禁止
- 赤・警告アイコン：禁止

---

## 5. 操作フロー（分岐）

### Yes
- 判断完了
- 必要に応じExecutionBlock生成
- Todayから消える

### Hold
- 判断完了
- Todayから消える
- 新RDD再計算

### No
- 判断完了
- Todayから消える
- Historyに事実として残る

---

## 6. 戻り操作

- 何も選ばず戻る → 未判断のまま
- 自動判断は禁止

---

## 7. 現実接続ポイント

- 納期は「見に行ったときだけ」見える
- 判断を急かすUIは存在しない
- 納期を見た上で断ることが自然にできる

---

## 8. 思想テスト（UI）

- 判断詳細を開かずに納期が見えていないか
- 納期が色・警告で強調されていないか
- 選択肢が3つ（Yes/Hold/No）に限定されているか

YESが1つでもあれば思想違反。

---

## 最終宣言
> 納期は判断を助けるためにだけ存在し、
> 人を追い込むために表示してはならない。
