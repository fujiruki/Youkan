## 1. カタログ機能 (My Catalog)

### 1.1 概要
ユーザーが作成した建具データを「テンプレート（ひな形）」として保存し、いつでも呼び出して再利用できるようにする機能。これにより、建具店独自の「標準仕様カタログ」を構築できる。

### 1.2 データ要件
*   **CatalogItem** エンティティ:
    *   `id`: unique ID
    *   `doorData`: 建具の全パラメータ (`Door` オブジェクトのコピー)
    *   `thumbnail`: プレビュー画像 (Base64/Blob)
    *   `category`: 分類 (例: "玄関", "書院", "フラッシュ")
    *   `tags`: 検索用タグ (例: ["杉", "組子", "モダン"])
    *   `createdAt`: 作成日

### 1.3 UI要件
*   **カタログ一覧画面 (Gallery UI)**:
    *   サムネイルをグリッド表示。カード型デザイン。
    *   カテゴリ・タグによるフィルタリング機能。
    *   「このデザインから新規作成」ボタン。
*   **登録フロー**:
    *   エディタ画面の「保存」ボタンから「カタログに登録」を選択。
    *   サムネイル自動生成。タグ・カテゴリ入力モーダル。

## 2. 視覚表現の高度化 (Visual Enhancement)

### 2.1 概要
「木目」「障子」「ガラス」などの質感をCanvas上でリアルに表現し、施主への提案力を高める。また、部位ごとの細かい材質指定を可能にする。

### 2.2 データ構造の拡張 (DoorSpecs)
各部材 (`GeometryPart`) に対して、以下のテクスチャ情報を付与できるようにする。

```typescript
type MaterialType = 'wood' | 'glass' | 'shoji' | 'paper' | 'none';
type GrainDirection = 'vertical' | 'horizontal'; // 縦目 / 横目
type GrainType = 'masame' | 'itame'; // 柾目 / 板目

interface PartTextureSpec {
    material: MaterialType;
    color?: string;       // ベースカラー (HEX)
    grainDir?: GrainDirection; // 木目の向き
    grainType?: GrainType;     // 木目の種類
    opacity?: number;     // 透明度 (0.0 - 1.0)
}
```

### 2.3 描画ロジック (Canvas)
*   単なる塗りつぶし (`fillRect`) から、パターン描画 (`createPattern`) へ変更。
*   **アセット**: 木目（柾目・板目）、障子紙、ガラスのシームレスパターン画像を用意。
*   **動的生成**: 木目画像を指定された角度（縦/横）と色（乗算合成）で描画するユーティリティ関数の実装。

### 2.4 UI要件 (Design Mode)
*   **テクスチャ設定パネル**:
    *   部材をクリックして選択し、プロパティパネルで素材を変更。
    *   「一括変更」ボタン（例: 全ての框を"杉・柾目"にする）。
*   **カラー調整**:
    *   木材の「色味」をスライダーで調整（赤み、明るさ）。

## 3. 実績写真比較 (Photo Comparison)

### 3.1 概要
実際の施工写真や製作中の写真をシステムに取り込み、設計データ（プレビュー）と並べて表示または重ねて表示する機能。

### 3.2 データ要件
*   `Door` エンティティへの追加:
    *   `photoData`: 画像ファイルの実体 (Blob)。容量制限に注意。
    *   `photoMemo`: 写真に対するメモ書き。

### 3.3 UI要件
*   **Compare View**:
    *   **Split Mode**: 画面を左右に分割。左：Canvasプレビュー、右：写真。
    *   **Overlay Mode**: 写真の上に半透明でCanvasを重ねる（寸法の整合性チェック用）。
*   **Before/After**: 提案時のCGと、完成写真を見せるプレゼンモード。

## 4. 拡張機能要件 (User Feedback Refinements)

### 4.1 帳票レイアウトのカスタマイズ
*   **要件**: 「建具表」の1マスサイズや、1ページあたりの表示数をユーザーが調整可能にする。
*   **設定項目**:
    *   グリッド設定: `1x1` (詳細), `2x2`, `3x2` (一覧) などのプリセット選択。
    *   表示項目: 「姿図のみ」「サイズのみ」「仕様込み」の切り替え。

### 4.2 リアルタイム原価計算の精度向上
*   **要件**: 木取りの自動配置までは行わないが、「余裕分（ロス率）」を考慮した原価をリアルタイム表示する。
*   **計算ロジック拡張**:
    *   `Gross Volume` (粗木積) = `Net Volume` (仕上積) × `(1 + Waste Rate)`
    *   **Waste Rate (余裕率)**:
        *   反り・曲がり対策: +X%
        *   鋸（ノコ）厚・削り代: +Y mm (厚み/幅への定数加算)

### 4.3 組子デザインの自由度強化
*   **要件**: 等間隔だけでなく、意匠性の高い配置を可能にする。
*   **機能レベル**:
    *   **Level 1 (Preset)**: 「吹き寄せ（Fukiyose）」ピッチのプリセットを用意。
    *   **Level 2 (Partial)**: 「途中までの組子」や「変則ピッチ」の表現（データ構造の配列化）。
    *   *※まずは Visual 重視で Level 1 から実装し、操作性を確認する。*

## 5. 製作物管理・現場管理 (Production & Field)

### 5.1 製作物リスト (Production List)
*   **概要**: 建具以外の製作物（枠、家具、金物）を一元管理する。
*   **データ構造**: `Door` エンティティの拡張（`category` フィールド追加）。
*   **UI**: リスト表示でのアイコン区別、汎用アイテム用の簡易入力モーダル。

### 5.2 現場管理 (Field Management)
*   **概要**: 現場での採寸記録や写真をプロジェクトに紐付ける。
*   **機能**:
    *   **Field Notes**: 日時付きのテキストメモ。
    *   **Photos**: 現場写真のアップロードと表示。


## 5. 優先順位と実装ステップ

1.  **Visual Enhancement**: 最優先。Canvas描画エンジンの改修。これができないとカタログの見栄えも悪くなる。
2.  **Catalog System**: DB設計と一覧画面の実装。
3.  **Cost & Report**: 積算ロジックへの余裕率導入と、帳票印刷プレビューの実装。
4.  **Photo Comparison**: 画像アップロード機能の実装。
