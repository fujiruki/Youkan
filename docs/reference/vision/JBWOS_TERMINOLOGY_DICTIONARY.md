# JBWOS 用語辞典 (Terminology Dictionary)

JBWOS（Judgment-Based Work Operating System）における主要な概念と用語の定義をまとめます。

## カレンダー・期間に関する用語

| 用語 | 定義 | 備考 |
|---|---|---|
| **納期 (Deadline)** | 外部（顧客や現場）との約束としての絶対的な締切。 | 守るべき最低ライン。 |
| **マイ期限 (My Due Date)** | 納期よりも前に、自分が「ここまでには終わらせたい」と設定する目標日。 | 心理的なゆとりを生むための自己防衛ライン。 |
| **目安日数 (Estimated Days)** | そのタスクを完了させるために必要な正味の作業時間（日数換算）。 | 1日あたりの稼働時間に基づいて計算される。 |
| **目安期間 (Estimate Period)** | **(マイ期限 − 目安日数 ＋ 1日) 〜 マイ期限** までの期間。 | カレンダー上で色付け（量感表示）される期間。「この期間までに着手・判断を終えていないと危険」であることを可視化する。 |

## タスクの状態 (Status) に関する用語

| 用語 | 定義 | 役割 |
|---|---|---|
| **インボックス (Inbox)** | 入ってきたばかりの未判断なタスク。 | 0秒で放り込む場所。判断は後回しでよい。 |
| **フォーカス (Focus)** | ステータスの一種。今日触る可能性がある、あるいは優先的に意識するタスクの集合。 | 「今日必ずやるリスト」ではない。判断を終えた置き場。※「Project Focus（特定プロジェクトの絞り込み表示）」とは異なる概念なので実装時は注意。 |
| **保留 (Pending)** | 今はやらないが、捨てないタスク。 | 罪悪感なく「やらない」を選択するための棚。 |
| **待ち (Waiting)** | 他者からの連絡や資材の到着を待っている状態。 | 自分が今すぐ動けない正当な理由がある状態。 |
| **完了 (Done)** | 作業が終了した状態。 | 達成の記録。 |
| **Someday (いつかやる)** | 「保留」の一種。詳細画面から「No」→「Someday」で遷移する。 | ステータス上は `pending` として扱われる。完了（Done）ではない。 |

## システム・画面に関する用語

| 用語 | 定義 | 備考 |
|---|---|---|
| **詳細画面 (Detail Modal)** | 個々のアイテムの詳細情報を編集・判断するための画面。 | 旧称: コックピット。 |
| **量感カレンダー (Ryokan Calendar)** | タスクの密度や納期、目安期間を可視化した独自のカレンダー。 | 1日あたりの「熱量」が色の濃淡で表現される。 |
| **パノラマビュー (Panorama View)** | 全体の親子関係や進捗を俯瞰するための画面。忘却防止が目的。 | |
| **プロジェクトフォーカス (Project Focus)** | 特定のプロジェクトに関連する全タスクのみを表示する絞り込みモード。 | ステータスの「Focus」とは無関係。ダッシュボードや一覧画面の表示スコープを指す。 |
| **プロジェクトフォーカス状態 (Project Focused)** | 上記モードが有効になっている状態。 | この状態で作成されたアイテムは、自動的にそのプロジェクト（および会社）に紐づく。 |

## アカウント・組織に関する用語

| 用語 | 定義 | 備考 |
|---|---|---|
| **法人アカウント (Company Account)** | 会社・組織全体を表すアカウント。メンバー管理等を行う。 | |
| **ユーザーアカウント (User Account)** | 個人を表すアカウント。複数の会社に所属可能。 | |

## アイテムの種類に関する用語

| 用語 | 定義 | 備考 |
|---|---|---|
| **プロジェクト (Project)** | 子タスクを持つアイテム。複数の工程やタスクを包含する大きな塊。 | |
| **タスク (Task)** | 具体的な作業単位。 | |
| **熱量 (Intensity)** | 特定の日に割り当てられた作業負荷の密度。 | カレンダー上で色の濃さとして表示される。 |

## 機能に関する用語

| 用語 | 定義 |
|---|---|
| **ブースト (Boost)** | 特定のタスクを一時的に（今日だけ）リストの前方に押し出す操作。 |
| **判断疲れ (Decision Fatigue)** | 膨大な選択肢や不確実性によって、次に何をすべきか決めるだけでも脳が疲弊する現象。JBWOSはこの軽減を最優先する。 |

## ダッシュボード・実行フェーズに関する用語

| 用語 | 定義 | 備考 |
|---|---|---|
| **今日やる (Queued for Today)** | 今日取り組む候補として選ばれたタスク群。 | ダッシュボードのメインストリームに表示される。FocusCard と Next Queue の総称。 |
| **現在のタスク (Focus Task)** | 「今日やる」リストの先頭にあり、今まさにユーザーが直面しているタスク。 | 他のタスクは視界から消え、これ一つに集中する（Focus）ためのデザイン。 |
| **Intent (やりたいこと)** | 人生において実現したい、緊急ではないが重要なこと。（第2領域） | 日々のタスク消化に埋没せず、パノラマビュー等で常に意識すべき北極星。 |
| **Engage (実行中)** | 現在のタスクに対して「今まさに手を動かしている」と宣言した状態。 | 旧称: Intent。ただリストにあるだけでなく、ユーザーの意志が注入されている状態。計測の開始点。 |
| **次に控えているタスク (Next Queue)** | 「今日やる」リストのうち、現在のタスク以外の残り。 | 現在のタスクが終わるかスキップされると、ここから次のタスクが繰り上がる。 |
| **スキップ (Skip)** | 現在のタスクを「次に控えているタスク」に戻し、順番を回す操作。 | 「今はできない」「後でやる」という判断を即座に反映し、次のタスクを呼び込む。 |
