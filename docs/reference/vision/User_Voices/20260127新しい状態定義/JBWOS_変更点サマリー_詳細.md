
# JBWOS 変更点サマリー（完全版）

## 1. なぜ変更するのか（背景）
- 従来システムは「プロジェクト単位」「タスク管理単位」で分断され、
  - 今日何をやるかが朝にならないと決まらない
  - 納期は見えているのに、判断の疲労で着手が遅れる
  - 割り込みや未達成が罪悪感として蓄積する
- 藤田晴樹の特性として
  - 判断コストに弱い
  - 登録は早いが整理が後回しになりやすい
  - 全体を一望できると安心する
  - 納期遅れへの心理的ストレスが大きい

## 2. 根本的な思想変更
### Before
- タスク = 作業単位
- 画面 = 管理のための画面
- Today = 実行リスト

### After
- タスク = 判断を要するアイテム
- 画面 = 判断疲労を減らす装置
- Focus = 判断済みゾーン（実行保証ではない）

## 3. 状態定義の変化
| 状態 | 旧 | 新 |
|---|---|---|
| Inbox | 未整理 | 判断不能時の避難所 |
| Ready | 今日やる | Focus（今日向き合う覚悟） |
| Pending | 保留 | 判断再開待ち |
| Waiting | 待ち | 外因停止 |
| Today | あり | 廃止（Focusに統合） |

## 4. 期限の再定義
- 納期 = 現実世界の締切
- マイ期限 = 自分を守るための内部締切
- 期限の本質 = 判断を再開するフック

## 5. 行動上の変化
- 朝にTodayを作らない
- 前夜または余裕時にFocusを選ぶ
- 当日未達成は失敗ではなく「判断更新」

## 6. 開発上の示唆
- Today画面削除
- Focusの内部状態（未着手 / 着手中）をUI表現で吸収
- Stateは増やさず ViewModelで解釈を分ける
