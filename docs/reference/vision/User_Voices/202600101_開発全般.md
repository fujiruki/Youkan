# アイテム・プロジェクトの権限ルール

## 個人アイテム作成時の所有権
- [x] 個人アイテム（tenant_idなし）を作成すると、`created_by` と `assigned_to` の両方に作成者のユーザーIDが自動設定される。

## 編集権限チェック
- [x] 個人アイテム（`project_id` がNULL かつ `tenant_id` なし）は、**作成者**または**担当者**であれば編集可能。
- [x] 会社アイテム（`tenant_id` あり）は、同じテナントに所属していれば編集可能。
- [x] 403エラー発生時は、`created_by`、`assigned_to`、`currentUserId` をエラーメッセージに含めてデバッグしやすくする。
- [x] この権限チェックは `ItemController.php` の `update()` と `ProjectController.php` の `update()` の両方で一貫して適用する。
