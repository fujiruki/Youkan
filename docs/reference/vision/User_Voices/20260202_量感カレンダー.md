# 量感カレンダーについて
- [ ] 量感カレンダーでは、各日付のユーザーがどのくらいの時間を使いたいか、というのをMAX値として、その日のタスクの合計時間がそのMAX値にどれだけ近いか、超えているかを色の濃ゆさで表現する。例えば、MAX値が10時間で、タスクの合計時間が5時間なら薄い色、10時間なら濃い色、15時間ならさらに濃い色、というようにする。
- [ ] そのmaxというのは、プライベート２時間＋仕事（複数の会社を合計して）８時間　などというように別々に設定できる。そして、その合計値がその日のmax値となる。ただし、プライベートの予定を入れる時はプライベートの予定とそのmax時間を元にした量感カレンダーが表示されたら良いなと思う。逆に仕事に絞った量感カレンダーでは、プライベートの時間は考慮しせず、仕事の日毎のmax時間、仕事のタスクの目安期間による。

# 量感カレンダー内のアイテムとセル・それのクリック
- [ ] 量感カレンダーに表示されるアイテムは、アイテム名＋プロジェクト名の先頭４文字。この１行のアイテムカードが量感カレンダーの中、納期の場所に表示されている。左クリックでそのアイテムの詳細画面が開く。
- [ ] 日のマスを左シングルクリックすると、その日の背景色に影響を与えているタスクすべてのカード中心に向けて、その日の中央から曲線がのびて指示する。曲線が伸びるアニメーションは0.5秒のみ。
- [ ] 日のマスを左ダブルクリックすると、その日の背景色に影響を与えているタスクの一覧が表示される。
- [ ] カレンダー上のアイテムカードをクリックすると、そのアイテムが影響与えているマスが光る。カレンダー上を１回クリックするとその表示状態がリセットされる。
- [ ] 量感カレンダーのレイアウトや背景色をそのまま詳細画面の納期選択・マイ期限選択・目安期間表示で使いたい。そのためにも、再利用しやすい構造にしてほしい。
- [ ] カレンダーに表示されるアイテムも表示されるかどうかはダッシュボードと同じような判断基準で行います。一番上のヘッダーのフィルターもう反映されるし担当者となっているアイテム・個人の自分のアイテムなどです。


- [ ] アイテムが作成されるとき、個人のアイテムなら作成者、会社に属するアイテムなら未割り当てで割り当てられる。
- [ ] プロジェクトに所属するアイテムとして作成される場合は、そのプロジェクトの担当者が割り当てられる。
- [ ] アイテムがプロジェクトに移動されたときは、移動先プロジェクトが属する会社に元の担当者のメールアドレスがあれば、担当者はそのまま。移動先プロジェクトが属する会社にもとの担当者のメールアドレスがなければ、その移動先プロジェクトの担当者がその担当者になる。

## 単語
- 業務キャパシティ
    ユーザーが自分の個人的なタスク・または会社に割り当てようと思っている時間のこと。
- 業務キャパシティ合計
    ユーザーが、「この日はタスク消化に使える時間はこれだけ。個人タスクと会社タスクに使える時間は合わせてこれだけ」と考えている時間
- 個人キャパシティ
    ユーザーが、「この日は個人的なタスク消化に使える時間はこれだけ。」と考えている時間
- A社キャパシティ
    ユーザーが「この日はA社のタスク消化に使える時間はこれだけ。」と考えている時間
- 会社キャパシティ
    A社キャパシティやB社キャパシティなどの総称。
- 各社キャパシティ合計
    ユーザーが所属している各社に割り当てているキャパシティの合計
- フリー時間
    ユーザーが会社にも個人にも割り当ててないタスク消化のための余裕の時間。残業などに使える。
    　　フリー時間 ＝ 業務キャパシティ合計 - 個人キャパシティ - (各社キャパシティ合計)


でも今設計を考えていますね。その前の仕様を考えたいんです。ユーザーのために、また、会社アカウントのために、どういうことを表示できるのが良いんだろうか？まずはその企画に注力しましょう。分母が切り替わるのは正しいです。
でも詳細画面でも何かしら切替が欲しいと思ったのです。たぶん、「まずは会社に割り当てている時間だけで考える。それでも空いてない。でもやりたいと思ったときに、他に空いている業務キャパシティ合計時間の残りを使ってやろうかな」と思ったときに、表示を切り替えれると良いなと思ったんです。これは個人アカウントに向けた機能ですね。

## 場面ごとに「誰の、何のために見ているか」
### ① 個人アカウント × プライベートモード
立場：「自分の人生を管理する人」

ここが一番複雑で、フィルタの出番があります。

全て：「私、全体的にどのくらい忙しい？」
分子＝自分の全アイテム、分母＝業務キャパシティ合計
個人：「プライベートの用事、いつ入れられる？」
分子＝個人タスクだけ、分母＝個人キャパシティ
会社：「会社の仕事、どこに入れられる？」
分子＝会社タスクだけ、分母＝会社キャパシティ（各社合計 or 特定の1社？）
ここでの質問：「会社」フィルタは全社合計ですか？ それとも会社ごとに選べるべきですか？（A社だけ見たい、B社だけ見たい、等）

### ② 個人アカウント × 会社モード
立場：「A社のマネージャーとして見ている人」

あなたが書いた通り、ここでは個人の時間は関係なく、会社の主力社員たちが「この会社に割り当ててくれている時間」の総和が分母になるべき、という考えは自然だと思います。

つまり：

分子＝A社のタスク全件
分母＝A社の主力社員たちのA社キャパシティの総和
この場面では「個人」「会社」フィルタは意味がないので、フィルタボタン自体が消えるか、自動的に「A社」固定になるのが自然だと思います。

質問：このモードは「会社の健康状態」を見る場面ですよね。であれば、ここで見たいのは「チーム全体の余裕度」だけですか？ それとも「社員Aは忙しいけど社員Bは空いてる」のような社員別の内訳も見たいですか？（今やるかは別として、将来の方向性として）

### ③ 会社アカウント × 社内モード
立場：「会社として、社内の業務を見ている」

ここが少し掴みにくいです。先ほどの忘年会の例のように「受注ではない、社内の用事」を管理する場面ですよね。

この場面の「分母」は何だと思いますか？ ②と同じく「主力社員のキャパ総和」でしょうか？

### ④ 会社アカウント × 会社モード
立場：「会社として、事業（受注・顧客対応等）を見ている」

③との違いは社内業務 vs 事業活動、ということでしょうか。分母は同じ「主力社員のキャパ総和」になりそうですが、分子（表示アイテム）の範囲が違う — 社内タスクは含まず、事業関連のタスクだけ？



会社アカウントにとっての業務キャパシティ設定があれば、、会社キャパシティは自分の受注の仕事、個人キャパシティは「社内キャパシティ」と言い換えて、社内の業務（改善業務や忘年会）に当てることもできますね。会社アカウントの業務キャパシティのフリー時間は、社員からもらった時間の余力ということか。　

しかし上記は現実的ではないように感じます。なぜなら会社にはユーザーが提供できる時間を自由に設定するので、会社側で固定の何時間、という指定ができませんよね。どう思いますか？

### 社内業務と事業は同じキャパシティプールを奪い合うのか？
そうですね、同じキャパシティプールを奪い合うのでいいかな。なぜなら、重要な業務なら会社業務が入る前に社内業務を入れておけばいい話なので。

この理解で合っています。
