# このシステム全体の紹介
このシステムの名前はYoukan（羊羹）です。
開発者である晴樹が苦手なスケジュール管理？をしやすくするために開発中です。
旧名称はJBWOSです。この単語がシステム中に使われていますが、減らしたいと思っている。現在のシステム名はYoukanです。
Youkanは、目玉機能の１つである「量感カレンダー」から来ています。この量感カレンダーは、（Ryoukan Calendar）は、グリッド形式のカレンダー上に忙しさが背景色の濃さとして表示される。これによって、数日間かかる製造などによってこのあたりは忙しいとかがわかる。このあたりは予定がないというのも見えるので、仕事を受ける際に「いつこの仕事を実行できるのか」がわかりやすい。


## ターゲット１：晴樹
私のことです。
３２歳　男　結婚しています。子供も生まれて、「夜も仕事する」というのがとっても難しくなりました。なので、日中の仕事の時間を有効活用したい。でも、、、
### 晴樹の特徴
私は整理することが苦手です。タスクが多くなると、なにから手を付けたら良いのかわからない。思い出したタスクなども1分後にはそれを忘れてしまうこともある。だけどこのシステムは描いたタスクの名前を一覧表示する機能もあって、忘れてないことを見れるという安心感がある。
また、晴樹は製造業である建具店をしている。業務内容としては、
1. 打合せ
2. 現場調査
3. 見積作成
4. 受注
5. 設計
6. 材料発注
7. 製造
8. 現場搬入
9. 施工
10. 請求
を行う必要がある仕事だ。1件の小さな仕事のためにもこれらがあるのだ。特に、製造が5日かかるものなどもあるが、この5日間をスケジュールで開けておく事ができずに、期限が近づいてきて納期遅れをすることがある。なぜなら、5日間のスケジュールを組んでいても、その間で他の仕事の打合せや見積をする必要が出るからだ。

ほかにも言語化できていないこともあるだろうが、晴樹が苦手なことはこのタスク管理、スケジュール管理、進捗管理だと思う。
あと、仕事を依頼されて断ることが苦手？というか、まあできるだろう。と思って、結局できないことが頻発している。小さい口約束も大きい仕事でもそれが起こる。これをなんとかするためのシステムになると良いなと思っている。

### 晴樹の得意なこと
    ・人当たりがよくて期待値を高くすること
    ・きれいなものを作る（建具・家具・簡易的なDTP）
    ・相手が聞いてほしいことを聞くこと
    ・プログラミング（ハードウェアからソフトウェア・クラウドまでできる）
    ・厳しい指摘をしてくれる人にこちらから近づくことができる
    ・人に教えることが好き
というところかな

    Youkanシステムの旧称。

    このシステムの名前。量感（Ryoukan）の一部から命名された。
- アイテム
    タスクとプロジェクトの総称。itemsテーブルに保存されている。
- プロジェクト
    サブタスクを持つことができるアイテム。サブタスクはアイテムのサブアイテムとして保存される。
- タスク
    プロジェクト化されていないアイテム。サブタスクを持たない。
    持っているフィールド：納期・マイ期限・目安時間・メモ・所属会社・担当者・所属プロジェクト
    また、プラグインの作用で、このフィールドを増やす仕組みもある。
- 納期
	そのタスクを必ず終えなくてはいけない期日。
- マイ期限
	納期よりも手前で完了しておきたいときに設定する自分なりの期限。
- マイ期日
	マイ期限と同じ。マイ期限で統一すべき。
- 目安時間
	そのタスク達成にかかると見込んでいる時間。単位：min
- 目安期間
	次のテキストに詳細を記載する。
	[目安期間の説明](20260211_目安期間の説明.md)
- 上部ヘッダーメニュー
    画面上部に表示されるメニュー。
    1段目と2段目と複数の段がある。
- コンテンツエリア
	コンテンツを表示するエリア。
	上部ヘッダーなどによって表示内容が切り替わる。

- 会社アカウント
	会社側が登録するアカウントのこと。
	会社が「このユーザーは弊社のメンバーです」ということを記録することで所属を示す。
- ユーザーアカウント
	個人が登録するアカウントのこと。
	メールアドレスと割り当てられるUUIDで識別する。
	会社に所属している場合は、会社側が「このユーザーが弊社メンバーです」と登録することで、その会社のプロジェクトなどにアクセスできるようになる。
- 個人アカウント
	ユーザーアカウントのこと。

# 全体像の理解
## アカウント種別・モード・フィルタ・プロジェクトフォーカス・立場ラベル
１．ログインしているのがユーザーアカウントか、会社アカウントか？
２．モード切替は何になっているか？
３．フィルタ状態はどうなっているか？
４．プロジェクトフォーカスモードになっているか？
５．それぞれの状態のときどんなものを表示するのかという立場ラベルの内容
これらによって表示できるものなどが変化しますよね？これらを表にしてどのようなパターンがあるか書くことはできますか？
パターン	アカウント種別	モード切替	フィルタ状態	プロジェクトフォーカス	立場ラベル (Perspective Label)	表示内容・役割
① 個人・統合	個人	プライベート	全て (all)	なし	「自分の時間管理」	個人＋所属全テナントのタスクを統合表示
② 個人・専念	個人	プライベート	個人 (personal)	なし	「自分の時間管理」	個人タスクのみ（他社のノイズを除去）
③ 会社横断	個人	プライベート	会社 (company)	なし	「会社業務の俯瞰」	所属する全テナントの業務のみ（個人のノイズを除去）
④ 特定社管理	個人	特定テナント	(テナントID)	なし	「○○社マネージャーとして」	特定の会社の業務に没入して管理
⑤ 事業全体	会社	会社固定	事業 (all)	なし	「事業の管理」	会社アカウントとして案件・収益事業を俯瞰
⑥ 社内事務	会社	会社固定	社内 (personal)	なし	「社内業務の管理」	事務・管理等の非収益/社内タスクに集中
⑦ 案件集中	(任意)	(任意)	(任意)	あり	「(案件名)」※	特定案件に全集中。Reality Loadも案件単位。
NOTE

※ 案件集中モード（プロジェクトフォーカス）時は、ヘッダーに「Project Focused」バッジが表示され、立場ラベルが案件名に切り替わります。



「会社アカウントにもプライベート（社内）モードは存在する」が正解です。

しかし、用語としての「モード」が画面上のどこを指しているかでズレが生じていました。

左側のドロップダウン（コンテキスト切替）: マトリックスの通り、会社アカウントは「会社固定」であり、他のテナントや個人ユーザーになることはできません。そのため、ここを無効化した前回の対応自体は（セキュリティ的にも）正解でした。
中央のボタン群（フィルタ切替）: 会社アカウントの場合、ここで「全て（事業全体）」と「社内（個人ルール）」を切り替えるのが正しい仕様です。
