# 労働する時間
個人アカウントは、「どういう時間配分で人生を過ごしたいと思っているか」を設定しておきます。
それと同時に、量感カレンダー上に「この日は完全に休む日にしたいんだよね」とか、「この日は午前中だけA社の作業をしたいんだよね」というイレギュラーも入力できます。


個人アカウントは労働する時間をそれぞれの日に、所属している会社ごとに設定できる。
例）　・1/3　には、
            A社に３時間、B社に４時間、C社に１時間、プライベートで２時間　時間を使おう。（制限ではなく、あくまで希望）
        ・1/4　には、
            A社に３時間、B社に2時間、C社に0時間、プライベートの用事に2時間　時間を使おう
        ・1/5　は休みにしよう。なにもしない日にするぞ。
これのデフォルト設定をユーザー設定にて設定できる
    例）とある個人アカウントの設定
        月曜　A社8時間
        火曜　A社4時間　B社4時間　C社0時間　プライベート2時間
        水曜　A社4時間　B社4時間　C社0時間　プライベート2時間
        木曜　A社4時間　B社4時間　C社0時間　プライベート2時間
        金曜　A社8時間　0社4時間　C社0時間　プライベート2時間
        土曜　A社0時間　B社0時間　C社4時間　プライベート2時間
        日曜　A社0時間　B社0時間　C社0時間　プライベート2時間


個人アカウントでログインした状態と会社アカウントでログインした状態があるじゃないですか、その中で個人アカウントが自分自身のユーザー設定画面を開いてその中で私が所属するA社には月曜日は何時間働きますとかB社には火曜日と水曜日に4時間働きますとかそういう設定ができるようにしたいんですよこれは個人アカウント自身が設定するものです

しかしそれとは別で確かに会社として何曜日は休みとかっていうのもありますよねまでもその設定ができることも今後必要になるかもしれないですがそもそも会社は主力メンバーが自社に対してこの日に何時間働くよと伝えてくれている内容をもとにじゃあこの仕事はこのあたりでできるとかそういうことを考えるようにしていますなのでユーザーが自分が働く時間を設定してくれていれば別に会社側がいつか仕事いつが休みってあえて言う必要をもないかなとも思います。

# 目安期間のロジック
- 所属組織の尊重: 「会社A」のタスクであれば、優先的に「会社Aの稼働枠（Budget）」を使って目安期間を計算します。
- フィルタとの連動:
    - カレンダーで「会社B」に絞り込んでいる時は、他社のタスクの目安はあえて出さない（またはフィルタ組織の枠で考える）。
    - カレンダーが「All（全体表示）」の時は、特定の組織枠を優先しつつ、設定がない場合に限り自分の総稼働時間にフォールバックして「一日の中のどこを占有するか」を表示します。
