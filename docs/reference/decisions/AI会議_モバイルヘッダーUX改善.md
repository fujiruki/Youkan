# AI会議：モバイルヘッダーUXのエコシステム化

## 議題
スマホ画面（特にiPhone SE等の375px幅）において、ダッシュボードヘッダー（Today's Decisionと表示される列）の要素が横幅に収まらず、右端（CLOSEボタン等）が見切れてしまっている問題の解決。

## 参加者（シミュレーション）
- **PM（JBWOS思想）**: 「思考の速度を落とさない」ことが最優先。情報は減らさず、ノイズは減らす。
- **Designer (Mobile)**: 375pxは絶対的な物理制約。全てを1行に収めるなら、テキストはアイコン化するしかない。
- **Developer**: 実装コストは抑えたいが、使い勝手が悪ければ意味がない。

## 現状分析
現在のヘッダー構成要素（左から）:
1.  **Title**: `⚡ Today's Decision` (長い)
2.  **Toggle**: `[BOARD] [CALENDAR]` (幅を取る)
3.  **Project**: `[+ Project]` (スマホで `[+]` に短縮済)
4.  **Close**: `[CLOSE]` (テキスト)

これらが375pxに収まっていない。

## 議論プロセス

**Designer**: 「Today's Decision」というテキスト、モバイルで毎回読む必要ありますか？
**PM**: 確かに。ユーザーは自分が「今日の判断」画面にいることは文脈で理解している。このテキストはPCなどの広い画面での「見栄え」や「世界観」のための装飾的意味合いが強い。
**Developer**: ではモバイルではアイコン `⚡` だけにする？
**Designer**: それで大幅にスペースが空きます。あと「CLOSE」ボタンも「×」アイコンにすべきです。モバイルの文法として「閉じる＝×」は共通言語です。

**PM**: `[BOARD] / [CALENDAR]` の切り替えは重要だ。これはタップしやすいサイズを維持したい。
**Designer**: 同意。ここは指で触れるメイン操作なので小さくしすぎてはいけない。

**Developer**: もう一つ提案。「+ Project」ボタンはここにあるべき？
**PM**: プロジェクト作成は「頻繁に行う判断業務」ではない。日々のタスク処理（Decision）とは少しレイヤーが違う。
**Designer**: ならば、ハンバーガーメニューの中に移動しても良いのでは？
**PM**: いや、アクセス性は残したい。ただ、ヘッダーの一等地に鎮座する必要はないかも。
**Design**: 現状の `[+]` ボタンで十分小さいので、タイトルさえ削れば収まるはずです。

## 解決策の提案

モバイル（幅768px未満）におけるUI変更案：

### 1. タイトルのアイコン化
- **Before**: `⚡ Today's Decision`
- **After**: `⚡` (アイコンのみ、または非表示)

### 2. ボタンのアイコン化（極小化）
- **Before**: `CLOSE`
- **After**: `✕` (アイコン)

### 3. 要素の優先順位と配置
`[⚡]` ... `[BOARD][CALENDAR]` ... `[+]` `[✕]`
この順で、中央の切り替えスイッチに最もスペースを割く。

## AI推奨案 (Implementation Specs)

**コンポーネント: `GlobalBoard.tsx`**

1.  **タイトルエリア**:
    - `md:block` でテキスト "Today's Decision" を表示。
    - モバイルでは `hidden` または `⚡` アイコンのみ表示。
2.  **CLOSEボタン**:
    - モバイルでは `<X />` アイコンを表示。
    - PCでは `CLOSE` テキストを表示。
3.  **レイアウト調整**:
    - Gap（余白）を `gap-2` から `gap-1` に詰める。

これにより、375px幅でも全ての操作要素が無理なく収まり、かつ誤タップ（Fat Finger）も防げるUIになります。
