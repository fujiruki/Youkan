# AI会議：マイ期限（My Date）の直感的操作UI

**日時**: 2026-01-21
**テーマ**: Side Calendarを利用した「マイ期限」と「公式納期」の使い分けUI
**入力ソース**: ユーザー要望「できればこのカレンダー表示を利用してマイ期限も設定変更できるようにしたい」

**参加者**:
- **Facilitator**: UX設計責任者。
- **Designer**: 直感的操作の専門家。
- **Engineer**: 実装可能性の検証担当。

---

## 1. 課題設定
現在、`DecisionDetailModal`（ワイドモード）には右側にカレンダーが表示されており、クリックすると「公式納期（Due Date）」が設定されます。
ユーザーは、この便利なカレンダーを使って「マイ期限（My Date / 自分の中での締め切り）」も設定したいと考えています。
**「どうすれば直感的に、納期とマイ期限をカレンダーから設定し分けられるか？」**が問いです。

## 2. アイデア出し

### 案A: 「フォーカス連動」 (Focus-Driven)
**Designer**: 
最も自然なのは、**「今どちらの日付を入力しようとしているか」**に入力欄とカレンダーが連動することです。

1.  左側のフォームで「公式納期」の入力欄をクリック（フォーカス）する。
    → カレンダーの枠が「赤色（納期のテーマカラー）」になり、カレンダーをクリックすると納期が入る。
2.  左側のフォームで「マイ期限」の入力欄をクリック（フォーカス）する。
    → カレンダーの枠が「緑色（マイ期限のテーマカラー）」になり、カレンダーをクリックするとマイ期限が入る。

**メリット**: モード切替ボタンを押す必要がない。入力フォームという既存のメンタルモデルと一致する。
**デメリット**: スマホなどタッチデバイスで「フォーカス」が少しわかりにくい場合があるが、PC操作メインなら問題ない。

### 案B: 「ドラッグ＆ドロップ」 (Drag & Drop)
**Engineer**: 
カレンダー上の日付をドラッグして、フォームに落とす…逆ですね。
フォームの横にある「納期アイコン」「マイ期限アイコン」を、カレンダーの日付にドラッグ＆ドロップする。
**評価**: 直感的だが、マウス移動距離が長く、素早い入力には向かない。

### 案C: 「カレンダー上の修飾キー/右クリック」 (Modifier Key)
**Engineer**: 
クリックで納期、右クリック（またはShift+クリック）でマイ期限。
**評価**: 「隠し機能」的で、初心者には発見できない（Discoverabilityが低い）。

### 案D: 「カレンダー上部でのモード切替」 (Toggle Switch)
**Designer**: 
カレンダーの上に `[ 納期モード | マイ期限モード ]` というタブを置く。
**評価**: 確実だが、ワンクリック余計にかかる。操作が冗長。

## 3. 推奨案：案A「フォーカス連動 ＋ 視覚的フィードバック」

**Facilitator**: 
案Aが最もスマートです。「JBWOS」はプロ向けツールとして効率を重視しています。
これに**「視覚的フィードバック」**を強化することで、現在のモードを明確にします。

### 具体的なUI挙動
1.  **初期状態**: 「マイ期限」の入力欄にフォーカスが当たっている状態（マイ期限設定を推奨するため）。
    - カレンダーヘッダー: 「マイ期限を選択中」と緑色で表示。
    - カレンダー上の既存マーカー: 納期は「赤」、マイ期限は「緑」の丸で表示済み。
2.  **カレンダー操作**: 日付をクリックすると、フォーカス中の入力欄に日付がセットされる。
3.  **入力欄切替**: ユーザーが「納期」入力欄をクリックすると、カレンダーヘッダーが「納期を選択中（赤）」に変化。

**Engineer**: 
実装としては、`SmartDateInput` に `onFocus` イベントハンドラを仕込み、親の `DecisionDetailModal` で `activeDateType: 'due' | 'my'` を管理すれば可能です。
カレンダー側は `activeDateType` を受け取り、クリック時の挙動とハイライト色を変えます。

## 4. 結論
**「フォーカス連動型カレンダー操作」**を採用します。
ユーザーは特別な切替ボタンを押すことなく、「入れたい欄を選んで、カレンダーを押す」だけで直感的に設定できます。
