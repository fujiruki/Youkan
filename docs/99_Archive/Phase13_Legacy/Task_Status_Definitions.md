# JBWOS タスクステータス定義表

ユーザー様からのフィードバックに基づき、用語の定義を明確化しました。現状の実装（`types.ts`）に基づいています。

## 1. Inbox & 判断プロセス（入口）
| ステータスID | 日本語表記（目安） | 定義・使用意図 | 表示場所 |
| :--- | :--- | :--- | :--- |
| `inbox` | **Inbox (未定)** | 新規作成されたタスク。まだ「いつやるか」「そもそもやるか」が決まっていない状態。 | **FutureBoard 左側 (Inbox)**<br>GDB Inbox |
| `decision_required` | 判断待ち | GDBでの整理が必要なもの（古いInboxなど）。`inbox`とほぼ同義だが、明示的にシステムが判断を求めている状態。 | GDB Decision |
| `decision_hold` | **保留 (Hold)** | 「今は判断しない」「塩漬け」。<br>**やるともやらないとも決めていない**状態。<br>※「いつかやる」に近いが、意思決定を先送りにした状態。 | GDB Shelf (Hold欄)<br>**FutureBoardには出ない** |
| `decision_rejected` | 却下 / ゴミ箱 | 「やらない」と決めたもの。削除の一歩手前。 | GDB Trash |

## 2. 計画・準備フェーズ（明日以降）
| ステータスID | 日本語表記（目安） | 定義・使用意図 | 表示場所 |
| :--- | :--- | :--- | :--- |
| (`prep_date`あり) | **予定 (Scheduled)** | 「いつやるか（日付）」が決まっている状態。Inboxからドラッグして日付に置くとこれになる。 | **FutureBoard 右側 (カレンダー)** |
| `ready` | **候補 (Today Candidate)** | 「今日やる候補」として選ばれたもの。朝の時点で候補リストに挙がっている。 | Today画面 下部 (Candidates) |
| `waiting` | **待ち (Waiting)** | 他者からの連絡待ちや、物理的な到着待ち。<br>**自分では進められない**状態。<br>※「後でやる」とは明確に区別される。 | GDB Shelf (Waiting欄)<br>**FutureBoardには出ない** |

## 3. 実行フェーズ（今日）
| ステータスID | 日本語表記（目安） | 定義・使用意図 | 表示場所 |
| :--- | :--- | :--- | :--- |
| `today_commit` | **コミット (Commit)** | 「今日絶対にやる」と誓ったもの（最大2-3個）。<br>これを完了することに集中する。 | Today画面 中央 (Commit) |
| `execution` | **実行中 (Doing)** | 今、現在進行形で手を動かしている**たった1つの**タスク。 | Today画面 中央 (Active) |
| `done` | 完了 (Done) | 終わったもの。 | Today画面 完了ログ |

## 4. その他（特殊）
| ステータスID | 日本語表記（目安） | 定義・使用意図 | 表示場所 |
| :--- | :--- | :--- | :--- |
| `intent` | 意思 (Intent) | タスクというより「やりたいこと」「願い」。より抽象度が高いもの。 | GDB Intent |
| `life` | 生活 (Life) | 家事、ルーチン、遊びなど、仕事（Work）以外の活動。 | (現状未実装/Intentに統合) |

## 今回の誤解ポイントの整理

ユーザー様が「保留」として使っていたものが、これまでどの挙動を期待されていたかによりますが、システム上の定義は以下のようになっています。

- **`status: inbox` (Inbox)**:
    - **意味**: 「未定」。まだ計画していない。
    - **FutureBoard**: 出る（計画対象だから）。

- **`status: decision_hold` (Hold)**:
    - **意味**: 「判断保留」。しばらく考えたくない、今は見たくない。
    - **FutureBoard**: **出ない**（ノイズになるため）。

## 5. ユーザーケース・マッピング (ユースケース対照表)

ユーザー様の分類(A-F)が、システムでどう扱われるかは以下の通りです。

| ユーザー分類 | 概要 | 対応ステータス / 場所 | 解説 |
| :--- | :--- | :--- | :--- |
| **A** | **とりあえず置いたもの**<br>(入力優先・詳細なし) | `inbox`<br>**FutureBoard Stock** | まさに「Inbox (未定)」の定義そのものです。 |
| **B** | **詳細あり・判断先送り**<br>(納期・目安ありだが日程未定) | `inbox` (基本)<br>**FutureBoard Stock** | まさにその通りです。「誰かを待っている（Waiting）」わけではないので、**Inbox**です。<br>「情報は揃っているが、いつやるかは未定」という状態なので、Stockに置いておき、空いている日にドラッグするのを待ちます。 |
| **C** | **今日やる！**<br>(できなくてもOK) | `today_commit` / `ready`<br>**Today画面** | 「今日」の画面で管理します。今日終わらなくても、自動で履歴に残るか、明日へ持ち越す(Inboxに戻す)ことができます。 |
| **D** | **明日以降の予定**<br>(連係タスクなど) | `scheduled` (`prep_date`あり)<br>**FutureBoard Calendar** | 日付が決まっているので、Stockではなく右側のカレンダー(週ビュー)に表示されます。 |
| **E** | **いつでもいい仕事**<br>(Someday / ゆるい納期) | `decision_hold` (または `inbox`)<br>**GDB Shelf** | これが一番迷うところですが、「毎日目に入れたくない」なら**Hold (Shelf)**です。「空きがあったら入れたい」なら**Inbox (Stock)**です。<br>JBWOSでは**期限(Due)**と**実行日(Prep)**を分けられるので、「期限は5月」だけど「実行日は未定(Inbox)」という運用が可能です。 |
| **F** | **やりたい！ (Intent)**<br>(仕事ではないかも) | `intent`<br>**GDB Intent** | 「タスク(義務)」と混ぜるとノイズになるため、「Intent」として別枠管理します。 |

### 結論
- **Stock (Inbox)** に表示されるのは **A** と **B (の未スケジュール版)**、そして **E (の空きに入れたい版)** です。
- **C** は Today画面。
- **D** は カレンダー画面。
- **F** と **E (の塩漬け版)** は判定・保存棚 (Shelf/Intent) です。


### 6. Inboxの2段構造とアクションの思想 (Inbox Structure)

「未整理の不安」と「準備完了の安心」を分けるため、Inboxを2つのセクションに分けます。

| セクション | 状態 (メンタル) | システム定義 (条件) |
| :--- | :--- | :--- |
| **未整理 (Inbox)** | **不安・焦燥**<br>「早く詳細を入れなきゃ」<br>「忘れないうちに！」 | `inbox` かつ **詳細不足**<br>(`!due_date` && `!estimatedMinutes`) |
| **スタンバイ (Stock)** | **安心・待機**<br>「あとはやるだけ」<br>「空きができたら入れる」 | `inbox` かつ **詳細あり**<br>(`due_date` or `estimatedMinutes`) |

#### UIアクションとボタン配置の思想
詳細モーダル（Grooming画面）におけるボタンは、ユーザーの心理状態に合わせて配置されます。

| 位置 | 意味合い | ラベル例 | 機能 |
| :--- | :--- | :--- | :--- |
| **左下 (Negative)** | **疲弊・拒絶**<br>「今は見たくない」 | **[今は隠す]** (Sleep)<br>※旧: 他のアクション | `decision_hold` (Shelf) へ移動。<br>視界から消して、週次レビュー等で再浮上させる。 |
| **右側 (Positive)** | **前進・完了**<br>「よし、片付いた」 | **[スタンバイに置く]** (Inbox)<br>※旧: 保存して閉じる | 詳細を保存して閉じる。<br>未整理→スタンバイへ昇格させるアクション。 |
| **右端 (Strong)** | **決意・実行**<br>「今日やるぞ！」 | **[今日やる]** (Today) | `today_commit` へ移動。<br>今日やるリストに入れる。 |

### 7. タスク分類・表現マッピング表 (保存版)

| ユーザー分類 | 状態 (メンタル) | システム Status | UI表現 (ボタン・場所) | アクションのゴール |
| :--- | :--- | :--- | :--- | :--- |
| **A. 放り込み** | **不安・未処理** | `inbox` (Draft) | **[未整理]** (New Section)<br>※Inbox上段 | 詳細を入れる<br>→ **Bへ** |
| **B. 詳細あり** | **安心・待機** | `inbox` (Ready) | **[スタンバイ]** (New Section)<br>※Inbox下段 | カレンダーへ配置<br>→ **Dへ** |
| **C. 今日やる** | **集中・義務** | `today_commit` | **[今日やる]** (Today) | 実行・完了する<br>→ **Done** |
| **D. 予定** | **拘束・約束** | `scheduled` | **[カレンダー]** (Week)<br>※日付指定 | その日が来るのを待つ<br>→ **Cへ** |
| **E. 塩漬け** | **無視・隠蔽** | `decision_hold` | **[今は隠す]** (Sleep)<br>※Shelfへ移動 | 視界から消す<br>→ (見直し) |
| **F. Intent** | **願望・方向性**| `intent` | **[Intentへ]** (Menu)<br>※別枠 | タスク化するまで温める<br>→ **Bへ** |
