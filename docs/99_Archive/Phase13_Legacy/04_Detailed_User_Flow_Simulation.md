# 04. 具体的ユーザーフローと体験シミュレーション (User Flow Simulation)

## 1. タイムライン・シミュレーション

### 朝：起動とコミット (07:30)
1.  **起動**: アプリを開く。画面には「プロジェクト一覧」や「カレンダー」ではなく、**Internal View (GDB)** が広がる。
2.  **Inbox確認**: 昨夜Inboxに放り込んだ「○○邸の金物確認」がある。「今は考えたくない」のでそのまま放置。
3.  **Ready確認**:
    - Inboxから重要な1件「△△邸 製作開始」をドラッグして **Ready** に入れる。
    - **トースト表示**: **「今日は、これと向き合います。」**
    - もう1件入れようか迷うが、あえて入れない。
4.  **終了**: Close。所要時間30秒。

### 昼：割り込みと「放り込み」 (13:00)
1.  **電話**: 現場から電話。「□□邸の寸法、枠が変わったから測り直して」
2.  **Inbox投入**: スマホでアプリを開き、**「□□邸 寸法再計測」とだけ入力してInboxに放り込む**。
3.  **思考停止**: 期限も優先度も設定しない。放り込んだことで「忘れていい」状態になり、目の前の作業に戻る。
4.  **安心感**: 「後でGDBが適切なタイミングで見せてくれる」と信じている。

### 夕方：量感の確認と終了 (17:30)
1.  **量感カレンダー**: ふとカレンダーを見る。来週のエリアが「濃いオレンジ色（量感）」になっている。
    - 「ああ、来週はちょっと重そうだな」と**直感する**。
    - 具体的に何があるかは見ない。ただ「重い」という気配だけを感じて、来週の新規案件（電話）を断る心づもりをする。
2.  **Ready完了**: 朝決めた「△△邸 製作開始」が終わった。チェックを入れて **Done** にする。
3.  **停止の肯定**: Readyが空になった。画面中央に大きなメッセージ。
    - **「今日はもう、やるものはありません。」**
    - これを見て、まだ17:30だが仕事を切り上げる。罪悪感はない。システムが「十分だ」と言ってくれたから。

## 2. 量感カレンダー体験 (Quantity Calendar Experience)

### 「気配」を見る
ユーザーは、カレンダーの日付セルに文字（タスク名）を見に行かない。
画面全体をぼんやりと見て、**「色の濃淡（ヒートマップ）」**を感じる。

- **薄い色**: 「まだ余裕がある。新しい仕事を入れてもいいかも。」
- **濃い色**: 「ここは修羅場だ。近づかないようにしよう。」
- **圧力ライン**: 特定の日付をクリックすると、そこに向かって伸びる光の線（Pressure Line）だけが見える。「ああ、この納期のせいで今週が赤いのか」と納得する。

### 守らなくていい「備え」
「備え完了目安」を設定した日を過ぎてしまった。
- 既存ツールなら：赤字で「遅延！-1日」と表示される。→ ストレス。
- **JBWOS**: 何も起きない。ただ、カレンダー上の「ぼやけた色の塊」が今日以降にズレ込むだけ。
    - ユーザーは「ああ、ズレたな」と思い、ドラッグして未来に置き直す。
    - 誰にも責められない。

## 3. やさしい救済フロー (Gentle Relief Flow)

### 「今日はできなかった」時
Readyに入れたタスクが終わらなかった。翌朝、アプリを開く。

1.  **ダイアログ**: 「昨日の『○○』、どうなりましたか？」
2.  **選択肢**: 
    - [A] 完了した（実は終わってた） → Historyへ
    - [B] 今日はやらなかった → **Inboxに戻る**
3.  **結果**: [B]を選んでも、ペナルティなし。バツ印もつかない。ただ「未判断」の状態に戻るだけ。

### 「断る」時
Inboxにある案件。どうしても気が乗らない、あるいはキャパオーバー。
1.  **詳細ビュー**: 「断る / 今は無理」ボタンを押す。
2.  **Historyへ**: 「今回は見送った」という事実だけがログに残る。
3.  **心理**: 削除（無かったこと）にするのではなく、「断るという判断をした」という**完了形**にすることで、脳のメモリを開放する。
