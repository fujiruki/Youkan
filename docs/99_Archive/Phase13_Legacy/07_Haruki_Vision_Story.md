# 07. 晴樹の物語：夢と現実を統合するOS (Haruki's Vision Story)

## 1. システム構成の進化：OS、モジュール、プラグイン

JBWOSは単なる一つのアプリではなく、明確な階層構造を持つプラットフォームとして構想される。

### Layer 1: JBWOS Core (Universal Judgment Engine)
- **対象**: 全ての人（特に既存の管理ツールに挫折した人）。
- **機能**:
    - Inboxへの「放り込み」と一時停止の肯定。
    - 「量感（Volume）」による負担の可視化。
    - 判断を終わらせるためのGDB（Ready 2件制限）。
- **価値**: 生活と仕事のすべてにおける「判断疲れ」からの解放。

### Layer 2: Project Module (Life & Work Integration)
- **対象**: 複数の文脈を持つ個人事業主、マルチポテンシャライト。
- **機能**:
    - 「建具工事」も「旅行」も「お葬式」も並列なプロジェクトとして扱う。
    - 全プロジェクトを横断して、自分のリソース（時間・精神力）が枯渇しないかを調整する。

### Layer 3: Tategu Plugin (Domain Specific)
- **対象**: 建具屋（および類似の受注生産型・現場対応型の製造業）。
- **機能**:
    - **ドメイン特化**: 見積もり、原価計算、労務費算出、請求書発行。
    - **超高速提案**: 作図（姿図）と見積もりの連動。

## 2. 物語：夢を食って生きていく

### 稼ぎと夢のトレードオフではない、「統合」
今まで、夢（自分の作品作り）は「仕事の空き時間にやるもの」だった。だから後回しになり、結局できなかった。
JBWOSでは、夢も一つのプロジェクトとして、仕事と同列にカレンダーに置かれる。

### シミュレーションの魔法
1.  **量感の確保**:
    「来月のこの週は、仕事（受注案件）が少ないな（色が薄い）。」
    → ここに夢のプロジェクト「新作キャビネット作成」を**ぼんやりと置く**。
    → するとカレンダーが埋まり（色が濃くなり）、他の仕事を受注できなくなる（＝断る理由ができる）。

2.  **粗利の合算（覚悟の数値化）**:
    - 目標月間粗利: **40万円**
    - 受注案件 A+B: 粗利 20万円（時間が埋まる）
    - 夢プロジェクト（新作販売）: 予想粗利 30万円（売れたら）
    
    **判定**:
    「仕事だけで40万稼ごうとすると時間が足りない。でも、夢プロジェクトに賭ければ、合計50万円になる可能性がある。よし、貯金を崩すリスクを背負ってでも、今月は夢をやろう。」

この**「経営的な覚悟」**を、ドンブリ勘定ではなく、データ（見積もり・原価・カレンダー）に基づいて下すことができる。これがJBWOSの真価である。

## 3. 建具屋のスーパーフロー

このプラグインを入れた晴樹氏は、無敵である。

1.  **超高速提案**:
    お客さんの前で、スマホ/タブレットで姿図を見せる。「こんな感じ？」→「いいね！」
    その場でサイズを入れるだけで、材料費・労務費・粗利が計算され、見積もりが出る。
2.  **受注とスケジューリング**:
    「やります」→ その場で受注ボタン。
    量感カレンダーを見て、「3月の納期だけど、2月中旬は空いてるからここで作ります」と**自分の都合**で予定を押さえる。
3.  **現場での積み上げ**:
    現場で追加工事発生。「これ直して」→ その場でInboxへ「追加修理」と放り込む。
    これが後で**そのまま請求明細行になる**。
    「請求し忘れ」という現場あるある（＝タダ働き）がゼロになる。
4.  **完了＝請求書完成**:
    工事が終わった瞬間、日々の「やったこと」が集計され、請求書が出来上がっている。
    あとは送るだけ。事務作業のために夜中まで起きている必要はない。

## 4. ターゲットの広がり (Potential Users)

この「JBWOS Core + Project Module」の構成は、建具屋以外にも刺さる。

- **ハンドメイド作家**: 制作、販売、発送、そして新作開発（夢）。
- **フリーランス・クリエイター**: 受託案件と、自分の表現活動。
- **兼業農家**: 農業と、別の仕事と、家庭。

既存の「効率化ツール」が切り捨ててきた、**「公私混同で泥臭く生きる個人」**にとっての、唯一の救いとなるシステム。
