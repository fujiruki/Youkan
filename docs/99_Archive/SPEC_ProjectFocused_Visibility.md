# 仕様書：ProjectFocused モードにおけるアイテム表示ロジックの改善

## 1. 背景と課題
現在、アイテムに「納期」または「マイ期限」を設定すると、そのアイテムは「棚（Inbox）」ステータスから「格上げ（Active）」ステータスへと内部的に移行します。
この際、バックエンドの「今日やる / 候補リスト」取得APIが、**「自分が作成した」または「自分に割り当てられた」アイテムのみ**を抽出するように厳格に制限されているため、自分以外の担当アイテムが納期設定された瞬間に画面から消えるという問題が発生していました。

## 2. 解決後の動作（理想の状態）

### A. 全体ダッシュボード（非集中モード）
- **動作**: 従来通り、自分に関係のない他人のタスクは非表示（ノイズの除去）。
- **理由**: 全体の景色では、自分のこなすべき仕事の総量（量感）を把握することが優先されるため。

### B. プロジェクト手中モード（集中モード）
- **動作**: 自分が担当かどうかにかかわらず、**そのプロジェクトに属するすべてのアイテム**が表示されます。
- **理由**: 特定のプロジェクトに没入している際は、「プロジェクト全体の地形（納期や段取りの山）」を把握することが、最適な個人の判断を助けるため。

## 3. 論理的な変更点

### 抽出フィルタ（バックエンド）の拡張
データの抽出条件（WHERE句）を以下のように論理拡張します。

| 項目 | 修正前の条件 | 修正後の条件 |
| :--- | :--- | :--- |
| **所有権フィルタ** | 自分が作成 OR 自分が担当 | 自分が作成 OR 自分が担当 |
| **プロジェクトフィルタ** | (上の条件を満たす) AND プロジェクトID一致 | **(上の条件を満たす) OR (このプロジェクトIDに所属)** |

### 副次的な効果
- **「マイ期限」の共有**: 他のメンバーが「いつ着手するつもりか（prep_date）」がプロジェクトビューで見えるようになり、チーム内での段取りの「空気感」が共有されます。
- **データ不整合の解消**: 編集保存時にプロジェクト所属が消えてプロジェクト迷子になるリスク（ID形式の差による不備）があっても、このロジックにより「所属さえしていれば見える」ため、ユーザーが問題を検知しやすくなります。

## 4. 影響範囲
- `backend/TodayController.php` のデータ取得ロジック。
- フロントエンドへの破壊的変更はありません（既存のAPIレスポンス形式を維持）。
