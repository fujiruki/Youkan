# 専門家会議：ユーザーと会社の関係・表示モデルの定義

**日時**: 2026-01-24
**テーマ**: 個人(User)と会社(Company)の表示統合とプライバシー分離について

## 参加者
*   **PM (進行)**: ユーザー体験の統合性を重視。
*   **Security (セキュリティ担当)**: 情報漏洩とプライバシー保護を重視。
*   **UI/UX (デザイナー)**: 直感的な操作と視認性を重視。

---

## 議題1: 個人の「Today」画面の統合

**PM**:
ユーザー様からの要望は、「会社タスクも個人タスクも、個人のToday画面では同列に扱いたい」です。これについては異論ありませんね？

**UX**:
賛成です。人間のリソースは一つなので、画面を切り替えて確認するのは認知負荷が高すぎます。
ただし、**「混ざっても区別できる」**デザインが必須です。
例えば、会社のタスクには「🏢 (会社名)」のバッジを付け、個人タスクは「🏠」や「無印」にするなどが考えられます。

**Security**:
個人の画面（Internal View）で本人が見る分には、セキュリティ上の問題はありません。
重要なのは、**「その画面をスクリーンショットして他人に送った場合」**や、**「背後から覗かれた場合」**のリスクです。
仕事中に個人の「転職活動」のタスクが見えてしまうとマズイですよね。

**PM**:
なるほど。では、**「プライベートモード（目のマーク）」**のようなトグルを用意し、ワンクリックで個人タスクを隠せる機能があると安心かもしれません。

## 議題2: 会社モード（管理者視点）からの見え方

**PM**:
次に、会社としてログイン（管理者モード）した際の、メンバーの見え方です。
「他の社員のタスク詳細は見えないが、忙しさは把握したい」という要件があります。

**Security**:
ここが最重要ポイントです。
会社モードの画面では、以下のルールを厳格に適用すべきです。

1.  **会社タスク**: 管理者は「内容」「進捗」を全て閲覧・編集可能。
2.  **個人タスク**: 管理者からは**「完全に不可視」**または**「単なるブロック」**として扱う。

**UX**:
「単なるブロック」として見せるのが良さそうです。
ガントチャートやタイムラインにおいて、個人タスクが入っている時間は**「灰色（Private / Busy）」**の帯として表示し、クリックしても「予定あり」としか出ないようにする。
これなら「あ、この時間は何かあるんだな（＝仕事振れないな）」とわかります。

**PM**:
「量感カレンダー」という言葉がありましたが、これはどう実現しましょう？

**UX**:
1日ごとの「負荷（%）」をヒートマップで出すのが直感的です。
*   会社タスク: 4時間
*   個人タスク: 2時間（灰色扱い）
*   合計: 6時間稼働予定 → **「やや忙しい（薄い赤）」**

このように、**「中身は見せないが、リソースは食っている」**という計算結果だけを管理者に伝えるのがベストプラクティスです。

## 議題3: プラグインとプロジェクト構造

**PM**:
「一般プロジェクト」「仕事プロジェクト」「製造業プロジェクト」の構造統一については？

**UX**:
基本構造は同じ（`Projects` > `Items`）にしつつ、適用される**「View（見た目）」**と**「Action（操作）」**を変えるのが良いでしょう。
*   **一般**: シンプルなリスト表示。
*   **仕事**: 期限・担当者が強調される。
*   **製造**: 寸法入力欄や「木取り」ボタンが出現する。

**Security**:
データ構造としても `project_type` カラムを持ち、それによってAPIが返すフィールドを制限するのが安全です。

## 結論（アクションアイテム）

### 1. ユーザーモデルの定義
*   **Unified Dashboard**: 個人・会社タスクを混合表示（アイコン区別）。
*   **Privacy Toggle**: 個人タスクの一時非表示ボタンを実装。

### 2. 会社（管理者）ビューの仕様
*   **Member Timeline**: メンバーの予定をガントチャート風に表示。
*   **Masking**: 個人タスク部分は「Private (Busy)」としてマスク表示。時間は占有する。
*   **Resource Heatmap**: 「その人が今日どれくらい埋まっているか」を色で可視化。

### 3. 設定画面
*   **Current Context**: 今「個人」として見ているのか「会社」の設定をしているのかを明確にヘッダーなどで区別する。
