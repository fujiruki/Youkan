# 量感カレンダー仕様策定 (2026/02/15)

## 方針転換
**抽象的なメタファー（コップ、水など）は廃止し、機能的かつ直感的な「色の濃淡」と「数値」による表現を採用する。**

## 1. ビジュアル表現 (Visual)

**基本方針**: 
`docs/晴樹の口頭入力/20260202_量感カレンダー.md` の記述に準拠します。

> その日のタスクの合計時間がそのMAX値にどれだけ近いか、超えているかを色の濃ゆさで表現する。

- **背景色**: 日付セルの背景色の「不透明度（Opacity）」または「明度」で稼働率を表現します。
- **ベースカラー**: ユーザーが設定したテーマカラー（デフォルトは青系）。

### 稼働率による変化
- **0%**: 背景色なし（または限りなく薄い灰色）
- **1〜99%**: 稼働率に応じて色が濃くなる。
    - 例: 50%なら不透明度0.5、80%なら不透明度0.8。
- **100%**: 定められた最も濃い色（例: 濃い青）。

## 2. オーバーフロー（100%超え）の表現

**方針**: 「物理的な事実」を淡々と表示する。

- **背景色**: 100%の時点での「最も濃い色」を維持、または識別可能な「超過色（例: さらに暗い青、または彩度が高い青）」とする。※赤色は使用しない（Judgment-Free）。
- **数値表示**: 
    - セル内に「予定合計時間」を明示する（例: `8.5h / 8h`）。
    - または、超過分を `+0.5h` のように表示する。

## 3. 計算ロジック (Logic)

```
ユーザー設定キャパシティ (DailyCapacity) = 
    A社設定時間 + B社設定時間 + プライベート設定時間
    (曜日ごとのデフォルト設定 + 日次オーバーライド)
```

```
ユーザー稼働予定時間 (TotalTaskTime) = 
    (A社タスクの見積合計) + (B社タスクの見積合計) + (プライベートタスクの見積合計)
```

- **判定式**: `TotalTaskTime > DailyCapacity` の場合、オーバーフローとして扱う。
- **対象タスク**: 本システムに登録されているタスクのみ計算対象とする（Googleカレンダー連携は後日）。

## 4. ユーザーアクション (UI/UX)

- **確認**:
    - ユーザーは「色の濃さ」で直感的に「忙しさ」を把握する。
    - 具体的な時間は、セルを見るか、カーソルを合わせることで確認できる。

- **調整 (オーバーフロー時)**:
    - 溢れている日のセルをクリックすると、その日のタスク一覧が表示される。
    - ユーザーはタスクの日付を変更（ドラッグ＆ドロップ）するか、その日のキャパシティ設定（日次オーバーライド）を変更して枠を広げる。
