# JBWOS フェーズ 1：基盤修理と超高密度化 行動計画

## 1. 目的
「理想の全体構造」を実現するための第1段階として、本番環境でのバグを一掃し、視覚的・認知的ノイズを削ぎ落とした「超高密度リスト」を構築する。

## 2. 実装ステップ（詳細順序）

### Step 1: ルーティングと環境基盤の修理
- [ ] **1.1 相対パス対応**: 本番URL（`/contents/TateguDesignStudio/`）配下で正しく動作するように `vite.config.ts` と `App.tsx` のルーティングパスを修正。
- [ ] **1.2 404対策**: ページリロード時に `/Focus` 等がサーバーで404にならないよう、`.htaccess` または SPAリダイレクトを配置。
- [ ] **1.3 API接続エラー修正**: 個人設定（PersonalSettings）でのAPIエラー、およびプロジェクト一覧での通信エラーを論理的に特定し解消。

### Step 2: 統合データ取得のバックエンド強化
- [ ] **2.1 `scope=dashboard` の再設計**: ユーザーの全所属テナントからタスクをJOINして取得するSQLの最適化（`ItemController.php`）。
- [ ] **2.2 合算キャパシティの算出**: クエリ結果に、その日の全テナント合計見積時間を含める実装。

### Step 3: フロントエンド・超高密度化の実装（MVVM/TDD）
- [ ] **3.1 `HeaderProgressBar` の作成**: 24時間プログレスバーをヘッダーの最上部に実装。
- [ ] **3.2 `useDashboardViewModel` の刷新**: 統合データを扱い、Undo/Redoを管理するViewModelの実装。
- [ ] **3.3 `SmartItemRow` の作成**: 
    - 1行表示、属性（会社・プロジェクト）の薄色表示。
    - マージン2px、パディング2pxの極限まで削ったCSS。
- [ ] **3.4 「ふんわり推し」ロジックの組み込み**: 推奨タスクに⚡アイコンを表示するロジックをViewModelに統合。

### Step 4: 標準コックピット（詳細画面）のバグ修正
- [ ] **4.1 タイトルEnter更新**: `DecisionDetailModal` でのタイトル編集→Enterで即時保存。
- [ ] **4.2 サブタスク追加バグ**: サブタスク名入力→Enter/＋ボタンで追加・表示が正常に行われるよう修正。
- [ ] **4.3 プロジェクト化ボタン**: 詳細画面からのプロジェクト化がDBに正しく反映されるよう修正。

## 3. 検証項目 (TDD/SVP)
- [ ] **Unit Test**: キャパシティ計算ロジック、ふんわり推しスコアリング。
- [ ] **Manual Check**: スマホでの左右スクロールが発生しないこと。
- [ ] **Integration Check**: 本番環境にデプロイし、ページ更新で404にならないこと。

## 4. 完了定義
- ダッシュボードが「属性付き1行リスト」として表示され、以前の重厚なUIが消えている。
- 公私のタスクが混在し、合計負荷がヘッダーで確認できる。
- 画面上の余白が大幅に削られ、一覧性が向上している。
- 基本的なバグ（更新・保存・APIエラー）がすべて解消されている。
