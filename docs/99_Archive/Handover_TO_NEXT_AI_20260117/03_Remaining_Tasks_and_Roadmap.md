# 03_Remaining_Tasks_and_Roadmap: 残タスクとロードマップ

## 直近のタスク (Priority High)

### 1. Future Board (Tomorrow Planning) の機能強化
- **休業日・祝日の表現**:
    - 「毎週火・水は定休日」「祝日」などをカレンダー上に表現し、制作目安日数（リードタイム）の計算から除外するロジックの実装。
    - 現状は実装されていないか、不完全な可能性があります。
- **タスクの分割・またがり**:
    - 「火曜の午後に着手し、水・木曜にかけて完了する」といった、複数日にまたがるタスクの表現。
    - 現在のDnDは単一の日付スロットへの割り当てのみです。

### 2. UI/UX の改善
- **Today画面**:
    - 「完了」ボタンを押した後の挙動（GDBへ移動後の戻りなど）の検証と修正。
    - 「今日やることを確定」ボタンの挙動安定化。
- **カレンダー表示**:
    - 目安期間（背景色）が正しく表示されないバグの調査と修正。
- **入力UI**:
    - 制作目安日数の入力UI改善（スクロール選択、タップ入力などスマホ最適化）。
    - サイドメモ（スマホ時）の格納・展開UI。

## 中長期ロードマップ (Priority Medium/Low)

### 1. アーキテクチャ整備
- **フォルダ名変更**:
    - `JWCADTategu.Web` というディレクトリ名を、プロジェクトの実態（JBWOSプラットフォーム + 建具プラグイン）に合わせて適切な名前に変更する検討。
    - PHP設定やVite設定への影響調査が必要。

### 2. 新機能・プラグイン
- **「な～んにもしない日」プラグイン**:
    - 休息日を積極的に計画するためのプラグイン開発。
- **建具管理の分離**:
    - 現在の「建具」機能をきれいなプラグインとして分離・確立する。
- **見積もり・顧客管理**:
    - これらを建具プラグインの一部とするか、独立したプラグインとするかの設計検討。

### 3. Server-Side Migration
- 現状の IndexedDB/Memory 保存から、PHP + SQLite への本格移行（議論中）。
