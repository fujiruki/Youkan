# JBWOS View Definitions (画面・表示仕様)
作成日: 2026-02-02
バージョン: 1.0

本ドキュメントでは、Dashboard内で提供される各ViewModeの役割、実装要件、および固有の挙動について定義する。

## 1. View Mode 一覧

`DashboardScreen` は `viewMode` stateにより以下の4つのモードを切り替える。

| モード名 | URL | 役割 | 英語名 |
| :--- | :--- | :--- | :--- |
| **登録と集中** | `/stream` | **コックピット**。実行 (Today) と判断 (Inbox) に特化。 | Stream |
| **全体一覧** | `/panorama` | **俯瞰**。プロジェクト構造と時系列 (Gantt) を確認。 | Panorama |
| **全体一覧２** | `/newspaper` | **カタログ**。サムネイル的な塊としてプロジェクトを閲覧。 | Newspaper |
| **カレンダー** | `/calendar` | **資源管理**。旅館スタイルでのキャパシティ確認。 | Calendar |

---

## 2. Stream View (登録と集中)

システムのデフォルトビューであり、日々の活動の拠点。

### 2.1 フィルタリング (Strict Filtering)
「今日やること」と「未判断」を完全に分離するため、以下のフィルタリングを厳格に適用する。

*   **Inbox List (Left)**: `status: inbox` のみ。`status: focus` は**絶対に除外**する。
*   **Focus Area (Center)**: `status: focus` のみ。

### 2.2 実装クラス
*   `SmartItemRow`: リストアイテム表示コンポーネント。
*   `FocusCard`: 中央の大きなカード。DnD対応。
*   `QuickInputWidget`: 高速入力欄。

---

## 3. Panorama View (全体一覧)

### 3.1 役割
「いつ、どのプロジェクトが動いているか」を可視化する。

### 3.2 実装仕様
*   **コンポーネント**: `JbwosBoard` (GlobalBoard)
*   **レイアウト**: 2ペイン構成
    *   左: プロジェクト・タスクツリー
    *   右: タイムライン (Gantt)
*   **仮想スクロール**: 大量のタスクを描画するため、Virtual Scrollingを使用。

---

## 4. Newspaper View (全体一覧２)

### 4.1 役割
新聞記事のように、プロジェクトごとのタスクの「塊」を並べて、全体像をざっと把握する。
Ganttチャートほど厳密ではなく、Streamほど近視眼的ではない中間ビュー。

### 4.2 レイアウト仕様 (Masonry)
*   CSS `column-count` を使用したマルチカラムレイアウト。
*   アイテムは `break-inside: avoid` により、カラムまたぎで分割されるのを防ぐ。

### 4.3 実装上の注意 (Click Handling)
Column LayoutやMasonry配置においては、ブラウザの描画レイヤーが特殊になり、通常の `onClick` イベントが不安定になる（ドラッグ操作と誤判定されやすい）問題がある。

*   **解決策**: アイテム (`NewspaperItem`) へのクリックイベントは **`onMouseUp`** でハンドリングすること。
*   **安全性**: `user-select: none` (`select-none`) を併用し、テキスト選択操作の終了時に誤爆しないように対策すること。

---

## 5. Calendar View (カレンダー)

### 5.1 役割
「いつが忙しいか」「どこに空きがあるか」を直感的に把握する。

### 5.2 旅館カレンダー (Ryokan Style)
通常のGoogleカレンダーのような「時間割」ではなく、旅館の予約表のように「部屋（リソース）」と「日付」のマトリクスで表現する。

---

## 6. 共通モーダル (Global Modal)

アイテムの詳細（Decision Detail）を表示・編集するモーダルは、**すべてのビューで共通して利用可能**でなければならない。

*   **実装配置**: `DashboardScreen.tsx` のルートレベル（条件分岐の外）に配置する。
*   **理由**: 条件分岐（`if (viewMode === ...)`）の中に配置すると、モード切り替え時にアンマウントされたり、特定のモードでレンダリングされずクリックしても開かないバグの原因となる（2026-02-02修正済み）。
