# JBWOS Item State Matrix (アイテム状態定義表)
作成日: 2026-02-02
バージョン: 1.0 (Official)

JBWOSにおいて、アイテム（Task/Project）が取りうる状態と、主要フィールドの組み合わせによる意味の違いを定義します。

## 1. 基本ステータス (JudgmentStatus)

| ステータス | 日本語 | 意味 | 典型的な居場所 (View) |
| :--- | :--- | :--- | :--- |
| **`inbox`** | 受信箱 | まだ判断されていない、未処理の状態。 | **Inbox** (左カラム) |
| **`focus`** | 今日やる | 「今日実行する」と意思決定された状態。 | **Stream** (中央カラム) |
| **`waiting`** | 待機中 | 他者待ち、または開始日待ちの状態。 | **Waiting** (左カラム) |
| **`pending`** | 保留 | 「いつかやる」または「条件待ち」の状態。 | **Pending** (左カラム) |
| **`done`** | 完了 | 作業が終了した状態。 | **Log** / **Archive** |

---

## 2. フィールドの組み合わせによる状態バリエーション

基本ステータスに加え、他のフィールド（`isEngaged`, `prep_date`, `isProject` など）との組み合わせで意味が変化します。

### A. Inbox (未判断) フェーズ

| Status | 組み合わせ | 意味・振る舞い |
| :--- | :--- | :--- |
| `inbox` | (標準) | **純粋な未処理アイテム**。判断待ち。 |
| `inbox` | `interrupt: true` | **割り込み/緊急**。Inboxの最上部に強調表示されるべき。 |

### B. Focus (実行) フェーズ

| Status | 組み合わせ | 意味・振る舞い |
| :--- | :--- | :--- |
| `focus` | (標準) | **今日のタスク (Queue)**。順番待ちリストに入る。 |
| `focus` | `isEngaged: true` | **現在実行中 (Active Focus)**。Focus Cardとして画面中央で最大化される。 |
| `focus` | `is_executing: true` | **実行中フラグ** (Backend)。`isEngaged` と同義だが、永続化された状態。 |
| `focus` | `isProject: true` | **プロジェクト全体のFocus**。そのプロジェクト自体を進める意思表示。 |

### C. Waiting (待機) フェーズ

| Status | 組み合わせ | 意味・振る舞い |
| :--- | :--- | :--- |
| `waiting` | (標準) | **他者待ち (Delegated)**。誰かの返答や作業完了を待っている。 |
| `waiting` | `prep_date` (未来) | **開始日待ち (Scheduled)**。指定の日付が来るまで着手できない。カレンダーに表示。 |
| `waiting` | `prep_date` (過去/今日) | **開始可能 (Ready)**。待機理由が解消されたため、InboxまたはFocusへ移動すべき候補。 |

### D. Pending (保留) フェーズ

| Status | 組み合わせ | 意味・振る舞い |
| :--- | :--- | :--- |
| `pending` | (標準) | **いつかやる (Someday)**。今はやらないが、将来やるかもしれない。 |
| `pending` | `due_date` (未来) | **期限付き保留**。期限は決まっているが、まだ着手しない（開始日未到来など）。 |

---

## 3. 特殊なフラグと状態

| フィールド | 値 | 意味 |
| :--- | :--- | :--- |
| **`isProject`** | `true` | **プロジェクト化**。アイテム自体が親コンテナとして振る舞う。子タスクを持つことができる。 |
| **`projectId`** | (ID) | **プロジェクト所属**。特定のプロジェクトの子タスクであること。 |
| **`tenantId`** | (ID) | **会社/組織所属**。特定の会社に関連するタスクであること。 |
| **`domain`** | `private` / `business` | **公私区分**。明示的に指定された場合、フィルタリングの対象となる。 |

---

## 4. 遷移の例 (ライフサイクル)

1.  **発生**: `inbox` (思いつき)
2.  **判断**:
    *   今日やる -> `focus`
    *   人に頼む -> `waiting`
    *   来週やる -> `waiting` + `prep_date`
    *   今はやらない -> `pending`
    *   もう終わった -> `done`
3.  **実行**: `focus` -> `isEngaged: true` (実行中) -> `done` (完了)
