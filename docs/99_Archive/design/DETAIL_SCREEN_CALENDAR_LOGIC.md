# 詳細画面（Cockpit）における量感カレンダーの挙動定義

ユーザー様の問い「それぞれの画面・フィルタ状態のときに開く詳細画面のカレンダーにはどういうタスクを表示し、それをもとに計算すべきなのかな？」に対する提案です。

## 基本方針: 「今、見ている世界（フィルタ）」を引き継ぐ

詳細画面のカレンダーは、メイン画面でユーザーが選んでいる「視点（フィルタ）」をそのまま継承します。
これにより、**「メイン画面で空き状況を見て、詳細画面でそこに入れる」** という直感的な操作が可能になります。

| メイン画面フィルタ | 詳細画面カレンダーの表示 (分母/分子) | ユーザーの思考・ユースケース |
| :--- | :--- | :--- |
| **All (全体)** | **全体容量 / 全タスク** | 「人生全体の空き時間を見て、無理のない日に設定しよう」<br>※最も安全かつ推奨されるモード |
| **Personal** | **個人容量 / 個人タスク** | 「プライベートの用事（買い物や趣味）を、個人の空き時間に入れたい」<br>※仕事の忙しさは考慮外 |
| **Company** | **仕事容量 / 仕事タスク** | 「仕事のデッドラインを決めたい。他の仕事の詰まり具合を見て判断しよう」<br>※プライベートの用事は考慮外 |

## コンテキストの優先順位（Smart Context Logic）

ユーザー様のご指摘通り、**「そのアイテムがどこに所属しているか（Item Context）」** は非常に重要です。
特に、**アイテムの所属を変更（例: 個人 → 会社）した際、カレンダーの「空き状況」もその移動先に合わせて変化するべき**です。

以下のロジックを提案します。

1.  **基本ルール**: 
    *   メイン画面が **「All (全体)」** の場合
        *   → 詳細画面も常に **「All (全体)」** で表示します。（最も安全な「人生全体」の視点）
  
2.  **アイテム主導ルール (Context Override)**:
    *   メイン画面が「Company」や「Personal」に絞り込まれている場合でも、**アイテムの実態（所属）を優先**してカレンダーを切り替える場合があります。

    | アイテムの所属 | メイン画面フィルタ | 詳細画面カレンダー (採用される量感) | 理由 |
    | :--- | :--- | :--- | :--- |
    | **会社 (A社)** | Company | **Company** (一致) | 仕事モードで仕事を計画する。正常。 |
    | **会社 (A社)** | Personal | **Company** (自動切替) | 「個人の用事」モードで「仕事」を開いた場合、その仕事を入れるべき「仕事の枠」を見る必要があるため。 |
    | **個人 (Private)** | Company | **Personal** (自動切替) | 「仕事」モードで「私用」を開いた場合、私用の枠（家の用事）を確認する必要があるため。 |

3.  **動的変化 (Live Change)**:
    *   詳細画面左上の「プロジェクト/会社選択」を変更した場合（例: 仕事 → 個人へ変更）、**即座にカレンダーの背景（量感）も追従して切り替わります**。
    *   これにより、「あ、個人の枠はいっぱいだから、仕事の枠に戻そう（またはその逆）」という判断がその場で可能になります。

## 4. ローカルフィルタ（カレンダー内切り替えスイッチ）

ユーザー様のご提案通り、**「詳細画面内でもフィルタを切り替えられるポーズ」** は非常に有用です。

**理由**:
「仕事のタスクを入れたいが、その日の個人の予定（歯医者など）が空いているか確認したい」というケースで、いちいち画面を閉じるのは手間だからです。

*   **実装イメージ**: カレンダーの右上（または左上）に小さなセグメントボタン `[ All | Company | Personal ]` を配置します。
*   **初期値**: 前述の「Smart Context Logic」で決定されたモードが選択されています。
*   **挙動**: ボタンを押すと、**「タスクの所属」は変えず、「背景の量感」だけ**を切り替えて確認できます。

## まとめ
詳細画面のカレンダーも、**「All / Personal / Company」の3つの世界観を忠実に守る** ことで、一貫性のある操作体験を提供します。
