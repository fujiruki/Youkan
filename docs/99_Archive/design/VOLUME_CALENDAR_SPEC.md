# 個人ユーザー向け「量感カレンダー (Volume Calendar)」仕様書

**作成日:** 2026年1月25日
**Version:** 1.0

## 1. 概要
本ドキュメントは、個人ユーザーが日々のタスク負荷（量感）を直感的に把握するための「量感カレンダー」機能の仕様を定義する。
企業における「リソース管理（Harukiモデル）」の考え方を、個人のライフスタイルに合わせて適応させる。

## 2. 決定されたモデル: Flexible Capacity Model (選択型キャパシティモデル)
AI専門家会議（PM, UX, System Architect）での議論とユーザーフィードバックに基づき、**「個人の働き方や契約形態に合わせて、仕事と生活の管理方法を選択できる」** モデルを採用する。

### 採用理由
ユーザーによって「仕事のみ管理したい（従業員）」「生活も含めて管理したい（個人事業主・ライフハッカー）」というニーズが明確に異なるため、単一モデルの押し付けは不適切と判断した。

### モード定義

#### Mode A: Separation (分離モード) - 従業員・典型的企業ユース
仕事時間と生活時間を明確に分けるモード。
*   **設定:** 労働キャパシティ（例: 8h）のみを設定。
*   **挙動:** 会社の「量感カレンダー」には労働タスクのみが反映される。生活タスクを入れても、労働キャパシティのリソースは消費しない（別枠管理、または無視）。
*   **ユースケース:** 定時で帰る会社員、業務委託契約など。

#### Mode B: Integration (統合モード) - 個人事業主・フリーランス
仕事と生活を区別せず、1日の総リソースで管理するモード。
*   **設定:** 1日の総キャパシティ（例: 24h中、可処分時間12h）を設定。
*   **挙動:** 仕事タスクも生活タスクも同じリソースを食い合う。
*   **ユースケース:** ワークライフインテグレーションを実践する人、副業を持つ人。

#### Mode C: Dual Stack (積み上げモード) - ハイブリッド
労働時間と生活時間をそれぞれ別のバケツとして管理する。
*   **設定:** 労働枠（8h） + 生活枠（4h）。
*   **挙動:** タスク属性により、どちらのバケツを消費するか決定。労働時間内に生活タスクを入れると「労働枠」を圧迫する（または警告が出る）。

## 3. 計算ロジック
個人の1日あたりの負荷率（Volume）は、選択モードにより異なる。

### Mode A (Separation)
```math
Work Load Ratio (%) = \frac{\sum (\text{Work Tasks})}{\text{Work Capacity}} \times 100
```

### Mode B (Integration)
```math
Daily Load Ratio (%) = \frac{\sum (\text{All Tasks})}{\text{Total Capacity}} \times 100
```

### 判定基準（UI表現）
モードに関わらず、設定されたキャパシティに対する比率で色分けを行う。
| 負荷率 | 判定 | UIカラーイメージ |
| :--- | :--- | :--- |
| **0 - 60%** | 余裕あり | 緑 / 青 (Chill) |
| **60 - 90%** | 適正〜忙しい | 黄 / 橙 (Moderate) |
| **90 - 110%** | 満杯 | 赤 (Busy) |
| **110% -** | **キャパオーバー** | 紫 / 濃赤 (Overload!) |

## 4. 設定項目 (Personal Settings / Volume Settings)
個人設定画面に以下の項目を追加する。

| 項目名 | 型 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- |
| **管理モード** | `enum` | `Separation` | `Separation` (仕事のみ) / `Integration` (統合) / `Dual` (別枠管理) |
| **Work Capacity** | `number` | `8` | 仕事に使える時間 (h) |
| **Life Capacity** | `number` | `4` | 生活に使える時間 (h) ※Mode B/Cのみ有効 |
| **休日設定** | `string[]` | `['Sun', 'Sat']` | キャパシティを0（または減少）とする曜日。 |
| **Work in Life?** | `boolean` | `false` | 休日（Life日）に仕事タスクが入った場合の扱い（警告/許可）。 |

## 5. 実装への影響
*   **Database:** `UserConfig` または `users` テーブルに `dailyCapacityHours` カラムが必要（現状はハードコードまたは未定）。MVPでは `localStorage` でも可。
*   **Logic:** `VolumeCalendarScreen` において、上記設定値を参照して負荷率計算を行うロジックを実装する。
*   **Sync:** 会社プロジェクトのタスク（作業時間）と、個人の「やること（Inbox/Today）」の時間見積もりを統合して計算する必要がある。

## 6. 将来の拡張性 (Out of Scope for MVP)
*   **Weight Factor:** 「頭を使うタスク(x1.5)」と「単純作業(x0.8)」のような係数設定。
*   **Time Block Sync:** Google Calendar等の外部カレンダー連携による「既に埋まっている時間」の除外。
