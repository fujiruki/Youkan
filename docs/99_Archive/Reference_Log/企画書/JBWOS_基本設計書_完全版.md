# JBWOS 基本設計書（完全版）

## 0. 位置づけ
本書は「JBWOS_企画仕様_統合_完全版.md」を受け、
**実装者が迷わず作れるレベル**まで落とし込んだ基本設計書である。
思想を壊さず、UI/UX・データ・振る舞いを一貫させることを最優先とする。

---

## 1. システム全体構成

### 1.1 レイヤー構造
- 判断レイヤー（Internal View / JBWOS）
- 説明レイヤー（External View / 対外説明）
- 記憶補助レイヤー（横メモ）
- 時間フックレイヤー（Googleカレンダー）

### 1.2 画面構成
- Home : Global Decision Board
- Project View : プロジェクト詳細（External）
- Calendar Sync View : 連携状況確認
- Concept / Guide View : 思想・使い方LP

---

## 2. Global Decision Board 基本設計

### 2.1 カラム構成
- Inbox（未判断）
- Waiting（外部待ち）
- Ready（今日向き合う）
- Pending（意図的保留）
- Done（出口・履歴なし）

### 2.2 Ready制御
- 最大2件まで
- 3件目投入時は必ず1件戻すUI
- Readyが空になったら「今日はもう十分」表示

### 2.3 並び順ルール
優先度順：
1. 期限フック（今日→明日→今週→来週）
2. 割り込み由来カード
3. 通常カード
※重さ（weight）は並び替えに使わない

---

## 3. Inbox耐久設計

### 3.1 表示制御
- 初期表示は最大5件
- 残りは「まだ考えなくていい」に折りたたむ

### 3.2 危険検知
- 明日期限カードは自動で最上部に浮上
- 「考える必要が出たものだけ見える」設計

---

## 4. 期限フック設計

### 4.1 粒度
- 今日
- 明日
- 今週
- 来週
- そのうち（非通知）

### 4.2 意味定義
期限 = 実行期限ではない  
**判断を再開するタイミング**

### 4.3 無視された場合
- スヌーズ扱い
- 責めるUI・赤表示は禁止
- 次の粒度へ自動繰越

---

## 5. 割り込み対応設計

### 5.1 割り込み発生時
- ワンタップで「割り込みとしてInboxへ戻す」
- 自動タグ：interrupt = true

### 5.2 並び順
割り込みカードは通常Inboxより上位に配置

---

## 6. 横メモ設計

### 6.1 UI
- 常時表示（TODO横）
- フリーテキスト
- 軽量入力最優先

### 6.2 振る舞い
- Inbox登録時に横メモ内容を自動添付
- 検索可能
- プロジェクト単位でも参照可能

---

## 7. Googleカレンダー連携

### 7.1 登録トリガー
- 判断が終わった瞬間のみ
- Ready投入時 or Waiting理由設定時

### 7.2 内容
- イベント名：判断結果サマリ
- 時間：仮（30分など）
- リマインド：必須

---

## 8. 初回体験（Onboarding）

### 8.1 初回起動フロー
1. 思想1枚LP表示
2. Inboxの意味説明
3. 「考えられない瞬間＝放り込む」強調
4. デモカード操作

### 8.2 学び直し導線
- 常設「このツールの考え方」ボタン
- 判断OSマップ表示

---

## 9. 「今日は何もしない日」UI

### 9.1 明示的操作
- 「今日は何もしない」ボタン
- Readyを強制的に空にする

### 9.2 表示
- 「今日は判断も作業も終わりです」
- Inbox・Pendingは開かない

---

## 10. データ設計（要点）

```ts
type JudgmentStatus = 'inbox' | 'waiting' | 'ready' | 'pending' | 'done';

interface Item {
  id: string;
  title: string;
  projectId?: string;
  judgmentStatus: JudgmentStatus;
  interrupt: boolean;
  weight: 1 | 2 | 3;
  dueHook?: 'today' | 'tomorrow' | 'this_week' | 'next_week';
  memo?: string;
}
```

---

## 11. 設計上の禁止事項（再掲）
- 赤色・遅延警告
- 完了率・進捗バー
- 正確な日付強制入力
- 無限Todo表示

**判断を終わらせ、人を前に進ませるためのOS**である。
