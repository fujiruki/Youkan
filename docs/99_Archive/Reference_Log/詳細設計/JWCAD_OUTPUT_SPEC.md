# JWCAD出力 機能仕様書 (JWCAD Output Specification)

## 1. 概要
本機能は、Tategu Design Studioで作成された建具データを、建築業界標準の **JWCAD形式 (.jww)** の図面ファイルとして出力するものです。
現場施工、製作発注、および設計図書への統合を目的とします。

## 2. 出力仕様 (Output Specs)

### 2.1 用紙・レイアウト
*   **用紙サイズ**: A3横 (420mm x 297mm)
*   **縮尺**: 1/50 (S=1/50)
*   **配置**: 2行 × 3列 のグリッドレイアウト（1ページあたり最大6建具）
    *   7建具以上の場合は複数ページ（グループ）に分割、または別ファイルとして出力（要検討）。
*   **グリッドサイズ**: 約 130mm x 120mm (マージン除く)

### 2.2 グリッド内構成
各建具枠（グリッド）は以下の3エリアにより構成されます。

#### A. ヘッダーエリア (上部 H:15mm)
1.  **建具記号 (Tag)**:
    *   左上に配置。
    *   文字サイズ: 6.0mm (強調)
    *   四角または円で囲む。
2.  **部屋名・名称 (Name)**:
    *   記号の右側に配置。
    *   文字サイズ: 4.0mm
3.  **数量 (Count)**:
    *   右上に配置。「× 3」形式。
    *   文字サイズ: 3.0mm

#### B. 描画エリア (中部 H:80mm)
1.  **基準線**: FL (Floor Line) を実線で描画。
2.  **建具姿図**:
    *   FL上に配置。
    *   縮尺 1/50。
    *   勝手記号（三角形）を表示。
3.  **人型 (Scale Figure)**:
    *   建具の右側に配置。
    *   **添景レイヤ** に配置し、表示/非表示を切り替え可能にする。
4.  **寸法線**:
    *   **左側**: H寸法（枠外・内法）
    *   **下側**: W寸法（枠外・内法）
    *   文字サイズ: 3.0mm

#### C. スペックエリア (下部 H:25mm)
テキスト形式で仕様を箇条書き。
文字サイズ: 2.5mm / 行間: 4.0mm

*   **仕様**: 樹種、ガラス種類、仕上げ
*   **金物**: 引手、蝶番、鍵など
*   **枠**: 見込み寸法、形状
*   **備考**: その他の特記事項

## 3. JWWデータ属性 (Attributes)

### 3.1 レイヤ構成 (Layers)
JWWのレイヤグループ、またはレイヤを活用して情報を分離します。

| レイヤ | 内容 | 線色 | 線種 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| **0** | 枠・建具外形 | 色2 (白/黒) | 実線 | |
| **1** | 見え掛かり線 | 色2 (白/黒) | 実線 | 框、組子など |
| **2** | 中心線 | 色1 (水色) | 一点鎖線 | |
| **3** | 寸法線・文字 | 色1 (水色) | 実線 | |
| **4** | 文字 (詳細) | 色1 (水色) | 実線 | スペックなど |
| **5** | 添景 (人型) | 色3 (緑) | 実線 | |
| **6** | 金額 (原価) | 色5 (紫) | 実線 | **デフォルト非表示** |

### 3.2 文字設定 (Fonts)
*   フォント: MSゴシック (等幅)
*   文字種設定 (推奨):
    *   文字種3: W2.5 / H2.5 (詳細)
    *   文字種4: W3.0 / H3.0 (標準・寸法)
    *   文字種5: W4.0 / H4.0 (見出し)
    *   文字種6: W6.0 / H6.0 (強調・記号)

## 4. UI連携
*   **出力ボタン**: 「建具一覧ページ」に配置。
*   **オプション**:
    *   「原価を含める」（レイヤ6に出力するかどうかの制御）
    *   「人型を含める」（レイヤ5に出力するかどうかの制御）

## 5. 技術的アプローチ
*   C# Backend (`JWCADTategu.Core`) にて JWW 生成ロジックを実装済み（既存資産活用）。
*   Webフロントエンドから JSON 形式で建具リストを送信し、Backend が JWW バイナリを生成して返却する API を利用（または新規実装）。
