# 憲法に基づく設計監査 (Design Audit by Constitution)

**目的**: 「建具業界ソフト設計思想（憲法）」に基づき、既存機能を監査し、今後の扱い（残す・分離・廃止）を判定する。

---

## 1. 思想（憲法）の再確認
> **「正確に計画させない」**
> **「時間ではなく存在を管理する」**
> **「外向き説明と内向き進行を分離する」**

---

## 2. 構成要素の仕分け (Component Classification)

憲法に基づき、現状の機能を以下に分類する。

### A. 思想違反 (VIOLATION)
**「正確な計画」を強制し、ユーザーを疲れさせる機能。内部管理からは排除する。**

*   **[機能] 納期（日付指定）入力**:
    *   *判定*: 違反 (Violation)
    *   *理由*: 原則⑥「納期は予測ではない」。日付を入れた瞬間、それは「遅れ」の計測器になる。
    *   *処置*: **廃止**。内部的には「1月案件」「急ぎ」「未定」という**ラベル（タグ）**のみで管理する。
*   **[機能] 遅延アラート (赤色表示)**:
    *   *判定*: 違反 (Violation)
    *   *理由*: 原則③「仕事は進まない前提」。遅れていることは本人も分かっており、追い打ちは不要。
    *   *処置*: **廃止**。
*   **[機能] 工数積算 (Man-Hour Calculation)**:
    *   *判定*: 違反 (Violation)
    *   *理由*: 原則②「時間ではなく存在」。不確実な現場に、分単位の積み上げは無意味。
    *   *処置*: **隠蔽**。見積もり根拠としてのみ裏で保持する。

---

### B. 分離すべき部分 (SEPARATE)
**「対外的な説明」には必要だが、「自分の精神衛生」には有害な機能。**

*   **[機能] ガントチャート (Gantt Chart)**:
    *   *判定*: 分離 (Separate)
    *   *理由*: 「いつ終わるのか」を他人に説明するためのツール。自分が見るとプレッシャーになる。
    *   *処置*: **「対外説明モード (External View)」へ完全隔離**。
*   **[機能] 既存カンバンボード (Detailed Kanban)**:
    *   *判定*: 分離 (Separate)
    *   *理由*: Todo/Doing/Done は「流れ」の管理。「Doing」が滞留すると罪悪感の塊になる。
    *   *処置*: **「対外説明モード」へ隔離**。
*   **[機能] カレンダー表示**:
    *   *判定*: 分離 (Separate)
    *   *理由*: 時間軸管理そのもの。
    *   *処置*: **「対外説明モード」へ隔離**。

---

### C. 再利用できる部分 (REUSE)
**「存在の管理」「判断の補助」として有用で、新UIの核となるベース。**

*   **[機能] 製作物リスト (Production List)**:
    *   *判定*: 再利用 (Reuse)
    *   *理由*: 「何を作るべきものが存在しているか」の証明。これは憲法に合致する。
    *   *活用*: **「判断管理ボード」のプール（未着手・未判断の山）として利用**。
*   **[機能] 汎用アイテム (Generic Items)**:
    *   *判定*: 再利用 (Reuse)
    *   *理由*: 「枠」「金物」などをざっくり登録できる点は、「判断を軽くする」思想に近い。
    *   *活用*: **そのまま利用**。
*   **[機能] 現場野帳 (Field Notes)**:
    *   *判定*: 再利用 (Reuse)
    *   *理由*: 事実の記録であり、計画ではないため無害。
    *   *活用*: **「判断の材料」として新UIからアクセス可能にする**。
*   **[データ構造] Project / Door / DB Schema**:
    *   *判定*: 再利用 (Reuse)
    *   *理由*: データそのものに罪はない。見せ方（View）を変えるだけで対応可能。
    *   *活用*: **共通バックエンドとして維持**。

---

## 3. 分離後の世界観 (Architecture Shift)

この監査により、システムは明確に2つのモードに分かれる。

### A. Internal View (自分のための道具)
*   **目的**: 精神的負荷の低減、今日やることの迷い除去。
*   **UI**: **Decision Board (判断管理ボード)**。
*   **時間軸**: **「今」「それ以外」** のみ。
*   **アクション**: 「仮決めする」「塩漬けにする」「誰かにボールを投げる」。

### B. External View (説明のための道具)
*   **目的**: 顧客・元請けへの誠意ある説明、予算・スケジュールの根拠提示。
*   **UI**: **Classical Scheduler (既存ガント・カンバン)**。
*   **時間軸**: カレンダー日付 (2026/01/XX)。
*   **アクション**: 「遅延の言い訳を考える」「リスケジュール線を引く」。

---

## 4. 結論 (Conclusion)
ユーザーの指摘通り、この順序で監査することで、
**「既存スケジューラの完全な隔離」** と **「全く異なるUXを持つ内部管理ツール」** の必要性が論理的に導き出された。

よって、**Feature: Constitution Scheduler** は、既存画面の改修ではなく、**「裏モード（真のメインモード）」の新規開発**として定義する。
