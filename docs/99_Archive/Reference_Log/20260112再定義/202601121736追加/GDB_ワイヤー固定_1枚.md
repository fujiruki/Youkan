# GDB_ワイヤー固定_1枚.md
最終更新: 2026-01-12 (JST)

本ドキュメントは、**Global Decision Board（GDB）のUIを1枚で固定**するための
最終ワイヤー定義である。

GDBはタスク管理画面ではない。
**判断対象だけが並ぶ場所**であり、この思想から逸脱してはならない。

---

## GDBの役割（再確認）

- 判断対象を一元的に集める
- 判断を急かさない
- 作業・実行・TODOを一切持ち込まない

---

## ワイヤー（1枚固定）

```
┌────────────────────────────────────────┐
│ Global Decision Board                   │
├────────────────────────────────────────┤
│                                        │
│ 【今ここにある判断】                   │
│                                        │
│  ・◯◯建具｜製作開始を判断する          │
│  ・△△案件｜見積提出可否を判断する    │
│  ・□□案件｜取付日変更への対応を判断  │
│                                        │
│────────────────────────────────────────│
│                                        │
│ 【保留中の判断（Hold）】               │
│                                        │
│  ・◯◯建具｜製作開始（保留）            │
│  ・△△案件｜受注可否（条件待ち）      │
│                                        │
│────────────────────────────────────────│
│                                        │
│ 【過去に判断したもの（参照）】         │
│                                        │
│  ・××案件｜今回は見送る（No）          │
│  ・■■案件｜製作開始（Yes）            │
│                                        │
└────────────────────────────────────────┘
```

---

## 表示ルール（厳守）

### 各リストの性質

- 今ここにある判断  
  - 未判断のみ  
  - タップで判断詳細ビューへ遷移

- 保留中の判断  
  - Hold状態のみ  
  - Todayには出ない  
  - 参照・再判断のみ

- 過去に判断したもの  
  - Yes / No 完了判断  
  - 操作不可（参照のみ）

---

## 表示してはいけないもの

- 納期
- 残日数
- 優先度
- スコア・点数
- 並び替えUI
- フィルターUI

※ 並び順はシステムが決める。人は触らない。

---

## 見た目・トーン

- 色：通常テキストのみ
- 強調：なし
- アニメーション：なし
- アイコン：最小限

---

## 操作フロー

- 判断したい項目をタップ
- → 判断詳細ビューへ遷移
- 眺めるだけでも問題なし

---

## 禁止事項（思想テスト）

- GDB上でYes/Noを決められるUI
- スワイプ判断
- 優先度入力
- おすすめ判断の提示

1つでもあれば思想違反。

---

## 実装AIへの最終指示

> GDBは「考える場所」ではない。  
> **考える前の机**である。  
>  
> 判断は必ず判断詳細ビューで行うこと。
