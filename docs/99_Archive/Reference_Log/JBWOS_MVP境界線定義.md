# JBWOS_MVP境界線定義.md

## 目的
本ドキュメントは、JBWOS（Judgment-Based Work Operating System）の思想を守るため、
**MVP（最小実用製品）に含めるもの／含めないものの境界線**を明確に定義することを目的とする。

本ツールの価値は「多機能」ではなく、
**ユーザーの判断回数と精神的負荷を減らすこと**にある。

---

## MVPの基本原則（絶対条件）

### 原則1：判断を増やす機能は入れない
- ユーザーに「どうする？」と問い続けるUIは禁止
- 選択肢が増えるほど、判断疲労が増える

### 原則2：罪悪感を生むフィードバックは禁止
- 遅延・未完了・未達成を強調しない
- 数値評価・達成率・連続記録を持ち込まない

### 原則3：「今日は何もしない」を肯定できること
- Readyが空でも問題ない
- 行動しない日を公式に許可する

---

## MVPに【含める】機能（必須）

### 1. Global Decision Board
- 全プロジェクト横断ビュー
- Inbox / Waiting / Ready / Pending の4バケツ
- Readyは**全体で最大2件まで**

### 2. Inboxの思想
- Inbox = 「今は考えられないものを放り込む場所」
- 登録時に分類・期限・優先度は不要

### 3. Readyの制約
- 2件制限は絶対ルール
- 1件入った時点で「今日は十分」と肯定表示

### 4. 横メモ
- 構造化しない
- 検索しない
- 思考を一時的に逃がすためのスペース

### 5. Googleカレンダー連携（片方向）
- Judgmentの結果を予定として登録
- カレンダーからJBWOSへは戻さない

### 6. Nothing Day UI
- 何もしない日を明示的に肯定
- 行動しないことを「正解」として扱う

---

## MVPから【除外する】機能（後回し）

### 通知・リマインド関連
- 無視した通知の履歴
- 再通知ロジック
- 通知の最適化・分析

### Inbox制御の高度化
- 自動浮上の複雑条件
- 期限未設定カードの高度ソート
- 割り込み専用レーン

### 管理・分析機能
- 振り返り画面
- 達成率・スコア
- ゲーミフィケーション要素

### 期限設定の詳細化
- 今日 / 明日 / 今週 / 来週の手動選択
- カスタム期限粒度

---

## 判断基準チェック（実装前テスト）

以下の質問に **YES** が含まれる機能は、MVPから除外する。

- その機能は新しい「判断」をユーザーに要求していないか？
- 使わなかった時に罪悪感を生まないか？
- なくても「判断を終わらせる」体験は成立しないか？

YES → **削る**
NO → **残す**
