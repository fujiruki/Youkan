# 検証手順ガイド: 計算ロジックと保存の堅牢化 (Phase 19)

このドキュメントは、Phase 19 で実施した検証内容を再現するための手順書です。

## 1. 工数・日数の自動同期と二重保存の検証

### 手順
1. **ダッシュボード**から任意のタスクをクリックし、**詳細モーダル**を開く。
2. 「見積工数」セクションのプリセットボタン（例: **4h**）をクリックする。
3. **確認項目**:
    - [ ] UI上の「稼働日数」が即座に「0.5日」に更新されること。
    - [ ] ブラウザの「検証(F12)」→「Console」にエラーが出ていないこと。
    - [ ] IndexedDB (`jbwos-db` > `items`) 内の当該アイテムの `estimatedMinutes` が `240`、`work_days` が `0.5` に更新されていること。
    - [ ] (可能であれば) ネットワークタブで `PUT /items/{id}` が飛び、サーバー側も同様に更新されていること。

## 2. 休日稼働枠の優先表示と分配検証

### 手順
1. **個人設定**（または会社設定）にて、特定の日（例: 土曜日）を「休日」に設定する。
2. 同じ設定画面の「稼働枠設定」にて、その土曜日に **480分** (8h) の枠を入力して保存する。
3. **カレンダー画面**（グリッド表示）に移動する。
4. **確認項目**:
    - [ ] 土曜日のセルが（休日色ではなく）**白色**（稼働日）として表示されていること。
    - [ ] 納期をその土曜日の後の月曜日に設定し、工数を16h（2日分）に設定した際、その土曜日にも工数が分配されている（青い線や数字が出る）こと。

## 3. 画面別フィルタの整合性検証

### 対象画面
- **全体一覧２**
- **カレンダー グリッド**
- **カレンダー ガント**

### 手順
1. 各画面において、上部ヘッダーのフィルタ（**全て・個人・会社**）を順に切り替える。
2. **確認項目**:
    - [ ] **個人**: `tenantId` が空、かつ `domain` が `business` 以外のアイテムのみが表示される。
    - [ ] **会社**: `tenantId` が設定されている、または `domain` が `business` のアイテムが表示される。
    - [ ] 表示内容の切り替わりに不自然なチラつきや、計算結果の矛盾がないこと。
