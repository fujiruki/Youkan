# GDB UI・ワイヤ（確定版 v2）

## 画面の位置づけ
本画面は内部構造としての GDB に対応する。
UI表示名は **「今日の約束にするか」** とする。

この画面は「並べ替え」や「計画」をする場所ではない。
**今日の約束にするかどうかを一つずつ判断するための画面**である。

## 画面構成（1画面固定）

```
┌──────────────────────────────┐
│ 今日の約束にするか            │
├──────────────────────────────┤
│                              │
│ ・◯◯建具 製作                │
│ ・銀行に行く                  │
│ ・書類をまとめる              │
│                              │
│──────────────────────────────│
│ [今日の約束にする] [今回はしない] │
└──────────────────────────────┘
```

## 表示ルール
- 一覧には「判断対象」のみを表示する
- 納期・時刻・優先度は表示しない
- 量が多い場合は縦スクロールのみ許可

## 操作
- **今日の約束にする**
  - 遷移先：Today
  - 意味：今日の唯一の約束として確定

- **今回はしない**
  - 遷移先：Intent / Life / History（選択）
  - 意味：今日は約束にしない

## 禁止事項
- 複数選択
- 並び替え
- ドラッグによる順序変更
- 未来日入力

## 未確定事項
- 判断対象が大量にある場合の分割表示
- 音声入力対応
