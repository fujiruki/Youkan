# GDB 全体レーン構成・最終図（判断／備え／履歴）（確定版 v2）

## 目的
本ドキュメントは、GDB 周辺のすべての要素を
**1つの最終構成図として固定**することを目的とする。
ここに記載された構成を最終形とし、再解釈は禁止とする。

## レーンの考え方（再掲）
- レーンは「カテゴリ」ではない
- レーンは「意味の状態」を表す
- 1アイテムは常に1レーンにのみ存在する

## 全体構成（上から下へ）

```
┌────────────────────────────────────────────┐
│ 今日の約束にするか（GDB）                  │
│ ─ 判断対象                                 │
│   ・今日の約束にする／しない／今は無理     │
│                                            │
│────────────────────────────────────────────│
│ 備え（ぼやけ）                             │
│ ─ 備え完了目安                             │
│   ・淡色・背景量感                         │
│   ・約束ではない                           │
│                                            │
│────────────────────────────────────────────│
│ 履歴（History）                            │
│ ─ 事実ログ                                 │
│   ・完了／断った                           │
│   ・評価なし                               │
└────────────────────────────────────────────┘
```

## 各レーンの役割（要点）

### 今日の約束にするか
- 判断を行う唯一の場所
- 問いは1つだけ
  - 「今日の約束にするか？」

### 備え（ぼやけ）
- 将来への安心を置く場所
- 判断も約束も発生しない

### 履歴
- 実際に起きたことだけを残す
- 参照専用

## 他レーンとの関係
- Inbox：GDBに入る前の入口
- Today：判断の結果としてのみ到達
- Intent / Life：判断を使わない別系統

## 禁止事項
- レーンの追加
- レーンの統合
- レーン間の自動移動

## 未確定事項
- レーンの折りたたみUI
