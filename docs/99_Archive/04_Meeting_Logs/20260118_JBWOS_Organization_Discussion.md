# AI会議ログ: JBWOSの組織運用論 (3人以上の壁)

**日時**: 2026-01-18
**テーマ**: 個人のためのOSであるJBWOSを、複数人の組織でどう運用するか？
**参加者**:
*   **PM (Project Guardian)**: JBWOSの思想「あくまで個人の判断を助ける」を固守する。
*   **Haruki (User Persona)**: 社長兼職人。自分もプレイヤーだが、人にも任せたい。
*   **Consultant (Org Expert)**: 小規模組織のマネジメント専門家。

---

## 1. 問いの確認

**Haruki**: JBWOSを使って「見積もりしながら脳内整理して、タスクに分解して、カレンダーでぼんやり計画する」っていうフロー、すごくいいと思う。自分が一人親方ならね。でも、もし社員が3人に増えたらどうなる？ 全員で一つのJBWOSを見るの？ それとも社長の俺だけが使うの？

**PM**: まず大原則として、**「JBWOSは個人の脳内OS」**です。これを「社員を監視・管理するツール」として使おうとした瞬間、JBWOSの最大の価値である「精神的平穏」は死にます。社員にとっては「仕事を押し付けられる恐怖のInbox」になってしまうからです。

**Consultant**: その通りですね。しかし、3人以上の組織には「共有された事実（案件）」と「個人の責任（タスク）」の両方が存在します。これをどう接続するかがカギです。

## 2. 3つの運用モデル

**Consultant**: 以下の3つのモデルが考えられます。

### Model A: Central Commander (社長だけが使う)
*   社長がJBWOSですべて計画・判断する。
*   社員には「今日これやって」と口頭や別ツールで指示する。
*   **判定**: 3人までならギリギリ機能するが、社長の判断負荷が限界に達する。社員は「指示待ち」になる。

### Model B: Shared Giant Brain (全員で1つのJBWOS)
*   巨大なInboxを全員で共有。「あいてる人がやる」方式。
*   **判定**: **JBWOS的には最悪**です。「判断（誰がいつやるか）」が完了していない状態で放置されるため、誰も責任を持たず、Inboxがゴミ捨て場になります。

### Model C: Distributed Intelligence (分散型OS) ★推奨
*   **建具データ（共有の事実）**: クラウド上の共有DBにある。
*   **JBWOS（個人の脳）**: **全員が、自分専用のJBWOSを持つ。**
*   **連携**: 案件が発生した時（または朝礼時）、担当者を決める。決まった瞬間に、その担当者のJBWOS Inboxにタスクが飛ぶ。

## 3. 「脳内整理」は誰の仕事か？

**Haruki**: 俺がやりたいのは「見積もりしながら脳内整理」なんだけど、それを社員にもやってほしいのかな？

**PM**: それは「設計・段取り」という高度な判断業務です。
*   **社長の仕事**: 見積もり～全体工程の設計（Future Boardでのぼんやり計画）。
*   **社員の仕事**: 割り振られた担当分の製作（Today画面での確実な実行）。

最初は社長がJBWOSで全体を整理し、タスク化したものを社員のJBWOSに「渡す」のが自然でしょう。社員が成長すれば、社員自身が自分のJBWOSで段取りを組むようになります。

## 4. 推奨フロー：共有と分散のハイブリッド

**Consultant**: 具体的には、このようなシステム構成が理想です。

1.  **Project Pool (共有の場)**
    *   ここには「建具データ」「顧客データ」がある。全員が見れる。
    *   まだ「誰がやるか」は決まっていない。

2.  **Dispatch (分配)**
    *   Harukiさんが見積もり時に「これはA君担当」と入力する。
    *   または、朝のミーティングで「これ誰やる？」「はい、やります」と決める。

3.  **Individual JBWOS (個人の場)**
    *   担当が決まった瞬間、その人のJBWOS Inboxにタスクが入る。
    *   **ここからは不可侵領域**。
    *   A君は自分のJBWOSで「今日はこれをやる」「明日は休む」を**自分で判断**する。
    *   社長はA君のToday画面を覗き込んだり、勝手に並び替えたりしてはいけない。

## 5. 結論

**Haruki**: なるほど。「共有の建具データ」と「個人のJBWOS」を分けるのか。

**PM**: そうです。
*   **会社として管理すべきは「納期と品質（建具データ）」**。
*   **個人に任すべきは「日々の進め方（JBWOS）」**。

**Haruki**: それなら3人でも10人でもいける気がする。俺は自分のJBWOSで「経営判断」や「設計」をタスクとして回し、社員はそれぞれのJBWOSで「製作」を回す。で、**建具プラグイン**がその間をつなぐパイプ役になるわけだね。

**Consultant**: その通りです。そのための機能として、建具プラグイン側には**「担当者割り当て(Assignee)」**機能が必要になるでしょう。これが将来的な機能拡張の方向性です。

---

### まとめ
*   JBWOSはあくまで「個人の持ち物」として運用すべき。
*   組織対応のためには、「一つの巨大なタスクリスト」を作るのではなく、「建具データをハブにして、各個人のInboxにタスクを配る」仕組みを目指すべき。
