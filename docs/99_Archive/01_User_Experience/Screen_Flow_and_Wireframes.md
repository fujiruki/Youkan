# JBWOS Core - Screen Flow & Wireframes (2026 Edition)

## 0. 基本原則 (UI Philosophy)
*   **Judgment Free**: ユーザーを責めない。「未完了」や「遅延」を赤字で強調しない。
*   **Reality Visualization**: 「キャパオーバー」を警告ではなく、単なる「ライン（境界線）」として淡々と表示する。
*   **Context Persistence**: どこにいても「今やっていること (Active Task)」が最優先で表示される。

    end

## 2. 画面レイアウト定義 (Wireframes)

### 2-1. アプリシェル (共通レイアウト)
*   **ヘッダー (PC/Mobile共通)**:
    *   **左**: ハンバーガーメニュー (個人/会社 モード切替)
    *   **中央**: `実行中タスクバナー` - 「実行中: [タスク名]」
    *   **右**: 同期ステータス
*   **サイドバー (PC) / ドロワー (Mobile)**:
    *   モード表示: "モード: 個人" / "モード: 会社"
    *   メニュー: ダッシュボード、パノラマ、プロジェクト、履歴
    *   設定

### 2-2. ダッシュボード (ホーム画面) - "コックピット"
この画面だけで「今日」が完結する設計。

#### A. 現在のタスク & 時間認識 (Top, Sticky)
*   **Active Task Banner**: `active_task_id` が存在する場合、最上部に大きく「実行中: [タスク名]」と表示。
    *   **Focus Mode**: 実行中は、下のリストを自動で折りたたむ。「今」以外を視界から消す。
*   **1日の流れバー (Sky Gradient)**:
    *   **見た目**: 画面上部の細いプログレスバー。
    *   **ロジック**: 現在時刻 / 24時間（または起床～就寝時間）。
    *   **色**: 朝の青 → 昼の空色 → 夕暮れのオレンジ → 夜の紫。
    *   **意味**: 「残り時間ゼロ」の恐怖ではなく、「夕暮れだからそろそろ店じまい」という感覚を直感させる。
*   **今日のインスピレーション (Random Intent)**:
    *   Intent（人生の指針）の中から、ランダムで1つだけをヘッダー付近に静かに表示。
    *   例：「💡 家族との時間を大切にする」

#### B. 今日やるリスト (Focus Queue) - 高密度表示
*   **詳細設計**: **一覧性最優先**。スクロールせずに全体が見える高密度レイアウト。
*   **UIアクション**:
    *   **朝のシャッフル (Morning Shuffle)**: ワンクリックで「いい感じ」に並べ直すボタン。
*   **カードスタイル**:
    *   **高さ**: 1行（Single Line）。
    *   **余白**: 上下 3px程度。
    *   **アクション**:
        *   **完了**: チェックボックス。
        *   **先送り (Not Today)**: ワンクリックでリストから消すボタン。「受信箱 (Inbox)」へ戻す。
            *   **理由**: 「保留」だと塩漬けになり、「明日」だと強制感が出るため、**「もう一度判断し直す」**という意味でInboxが最適。
    *   **表示項目**: タイトル、目安時間、Intentアイコンのみ。
*   **キャパライン (夕暮れの境界線)**:
    *   上から順に目安時間を積み上げ、自分の稼働限界（例: 8時間）を超えた位置に赤線を引く。
    *   **フェードアウト**: ラインより下のタスクは、不透明度を下げて（Opacity 0.6程度）表示する。「今日のフォーカス外」であることを表現。

#### C. 受信箱クイック入力 (Bottom or FAB)
*   常時表示される入力欄。「思いついたら即Enter」。
*   **モードフリー**: 個人モードでも会社モードでも、ここからの入力は常に「受信箱」へ。

### 2-3. パノラマビュー (全体俯瞰)
「全体を忘れていない」ことを確認し、安心するためのビュー。

#### A. Intent (人生のコンパス)
*   最上部に配置。「緊急ではないが重要なこと」を常に掲示。

#### B. 分類マトリクス
*   **受信箱 (Inbox)**: まだ判断していないもの。
*   **保留 (Pending)**: 今はやらないと決めたもの。
*   **待ち (Waiting)**: 他人からの返事待ち。
*   **完了 (Done)**: 終わったこと。

### 2-4. プロジェクト一覧 & 製造業連携
#### A. 個人/会社 モード切替
*   サイドメニューのスイッチで完全に切り替える。
    *   **個人モード**: 家事、趣味、旅行など。
    *   **会社モード**: 工事案件、見積、事務など。

#### B. 製造業連携 (Deep Link)
*   **親タスク自動生成**: 案件を作ると、JBWOSに「[案件名]」という親タスクが生まれる。
*   **連携ボタン**: 詳細画面に「案件詳細を開く」ボタンを配置。図面や見積もり画面へ一発ジャンプ。

## 3. インタラクション定義 (Interaction Model)

### 3-1. カード操作
*   **クリック**: **詳細パネルを開く**。PCは右側パネル、スマホはモーダル。
*   **ドラッグ＆ドロップ**: 並び替え（Focus内）。即時保存。
*   **右クリック**: コンテキストメニュー（実行中にする、保留へ送る、削除）。

## 4. UIコンポーネント詳細 (Specifics)

### 4-1. 容量ライン (Capacity Line)
*   **見た目**: 1pxの破線（赤）。
*   **挙動**: ドラッグ＆ドロップで順番を変えると、リアルタイムにラインの位置が上下する。これが「物理的な量感」となる。

### 4-2. 横メモ (Side Memo)
*   **PC**: 通常はアイコンのみ。クリックまたはタスク詳細を開くとスライドイン。
*   **Mobile**: アイコンタップでオーバーレイ表示。
*   **役割**: 「ちょっとメモしたい」「計算用に数字を置きたい」といった一時作業領域。

### 4-3. 統合フォーカスロジック (One Life)
*   個人タスクも会社タスクも、設定により「今日やるリスト」に**混在**させる。
*   キャパラインの計算は、すべてのタスクの合計時間で行う。「仕事も家事も、私の時間を使う」という現実を反映する。
