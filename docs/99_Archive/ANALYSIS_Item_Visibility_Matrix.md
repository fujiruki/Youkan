# アイテム状態パターンと表示条件マトリクス

この表は、アイテムの各属性（フィールド）の組み合わせが、システム上でどのような「景観」として扱われ、どこに表示されるかを整理したものです。

## 1. 状態マトリクス

| 属性の組み合わせ (Pattern) | 代表的な意味・状態 | 全体一覧2 / ダッシュボード | プロジェクト集中モード | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| `isProject: true`<br>`projectId: NULL` | **ルートプロジェクト**<br>プロジェクトそのもの | 見出しとして表示 | 非表示または自身の詳細 | 全ての親 |
| `isProject: true`<br>`projectId: SET` | **サブプロジェクト**<br>大きな塊の分解 | 親の下にインデント表示 | 所属アイテムと共に表示 | 入れ子の管理 |
| `isProject: false`<br>`projectId: NULL` | **迷子のタスク**<br>Inbox（受信箱）の住人 | 「未分類」に表示 | 非表示 | いずれかの棚へ |
| `status: 'focus'`<br>`isEngaged: false` | **今日やるタスク**<br>（青い●） | 青い●付きで表示 | 表示 | 決意の地形 |
| `status: 'focus'`<br>`isEngaged: true` | **着手中のタスク**<br>（緑の点滅●） | 緑の点滅●で表示 | 表示 | 現在の活動 |
| `status: 'pending'` | **保留アイテム** | 「保留」バッジ付き | 表示 | 判断を先送りにしたもの |
| `status: 'done'` | **完了アイテム** | 取り消し線付き | 表示 | 過去の記録（景観の一部） |
| `due_date: SET`<br>`status: 'inbox'` | **期限付きの未着手** | 納期順に並ぶ | 表示 | 地形としての「山」 |

## 2. フィールドの組み合わせの意味

### A. 地形 (Landscapes)
- **`due_date` / `prep_date`**: 
    - これらがセットされると、アイテムは単なる「文字」から、カレンダーやダッシュボード上の「重みを持った地形」へと変化します。
    - ユーザーはこれを見て「いつまでに山を越えればいいか」を判断します。

### B. 決意 (Decisions)
- **`status: 'focus'` (今日やる)**: 
    - 溢れるタスクの中から、今日だけはこれに向き合うという「決意」の印です。
    - 画面上では青い●として、常に意識の端に留まるようになります。

### C. 活動 (Engagement)
- **`isEngaged: true` (とりかかる)**: 
    - 決意（青い●）を行動（緑の点滅●）に移した状態です。
    - 現在このタスクに「入り込んでいる」ことを示し、システムはそのコンテキストを維持します。

## 3. 「表示されない」時の不整合チェック

もしアイテムがProjectFocusedで表示されない場合、以下の論理矛盾が発生している可能性があります：

1. **ID形式の不一致**: プロジェクトのIDが `item_123` なのに、アイテム側の `projectId` が `prj-123` になっている（またはその逆）。
2. **所有権の壁**: （実装上の不備）自分が作成・担当していないアイテムが、プロジェクト所属よりも所有権フィルタで弾かれている。
3. **ステータス制限**: 保存時に意図せず `status` が `archive` 等に書き換わっている。
