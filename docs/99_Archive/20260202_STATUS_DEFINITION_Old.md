# アイテムステータス定義とダッシュボード表示仕様

## 1. ステータス (JudgmentStatus) 定義

システム内でアイテムが持ちうる5つの基本ステータスです。

| ステータス | 日本語 | 意味・役割 | ダッシュボード「登録と集中」での表示 |
| :--- | :--- | :--- | :--- |
| **`inbox`** | 受信箱 | まだ判断されていない、未処理のアイテム。 | 左カラム「受信箱 (Inbox)」リスト |
| **`waiting`** | 待機中 | 他者への依頼中、または開始日待ちの状態。 | 左カラム「待機中 (Waiting)」リスト |
| **`focus`** | 今日やる | 今日実行する意図があるアイテム。 | **中央メインエリア** (Focus Card または Next Tasks) |
| **`pending`** | 保留 | 今はやらない、いつかやる、または条件待ち。 | 左カラム「保留 (Pending)」リスト |
| **`done`** | 完了 | 作業が終了したアイテム。 | 表示されません (ログへ移動) |

---

## 2. ダッシュボード「登録と集中 (Stream View)」の表示ロジック

ダッシュボードは、ステータスと「実行意図 (Engaged)」に基づいてアイテムを配置します。

### A. 中央メインエリア (Focus Zone)

`status: focus` のアイテム、または `isEngaged: true` のアイテムが集まります。

1.  **Focus Card (現在実行中/最優先)**
    *   このエリアの **先頭の1件** が、画面中央に大きくカード形式で表示されます。
    *   **「今日やる」ボタンを押したアイテムは、基本的にここ（またはその直下のキュー）に移動します。**
    *   Inboxから移動して「消えた」ように見える場合、まずはこの中央の大きなカードに表示されていないか確認してください。

2.  **次に控えているタスク (Queue)**
    *   Focus Card の下にリスト形式で表示される、2番目以降のタスク群です。
    *   ここには `todayCommits` (確定済み) と `todayCandidates` (候補) が表示されます。

### B. 左サイドバー (Judgment Zone)

*   **受信箱 (Inbox)**: `status: inbox` のアイテムが表示されます。
*   **待機 (Waiting)**: `status: waiting` のアイテム。折りたたみ可能です。
*   **保留 (Pending)**: `status: pending` のアイテム。折りたたみ可能です。

### C. フィルタリング仕様

上部ヘッダーのフィルタ設定（Capacity Bar）により、表示されるアイテムが制限されます。

*   **ALL (全表示)**: すべてのアイテムを表示。
*   **Business (Company)**: 会社に関連するアイテム (`tenantId` または `projectId` あり、または `domain='business'`) のみ表示。
*   **Personal**: 個人アイテム (`domain='private'` または所属なし) のみ表示。

> **注意**: フィルタにより非表示になっているアイテムがある場合、リストの下部に「+ X hidden items」のような注釈が表示されます。
