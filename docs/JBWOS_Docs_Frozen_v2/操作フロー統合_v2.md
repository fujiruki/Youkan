# 操作フロー統合（Inbox → 今日の約束にするか → Today → History）（確定版 v2）

## 目的
本ドキュメントは、日常の操作を**最短・最少思考**で完了させるための
全体操作フローを定義する。意味の補完や自動化は行わない。

## フロー概要
1. 放り込む（Inbox）
2. 今日の約束にするか（判断）
3. 実行（Today）
4. 事実として残る（History）

## 1. Inbox（放り込む）
- 操作：テキスト入力 / 音声入力
- 意味：未整理の思いつきを即座に保存
- 次の操作：整理する → 今日の約束にするか

### 可能操作
- 整理する
- アーカイブ（捨てる）

### 禁止
- 期限入力
- 優先度設定

## 2. 今日の約束にするか（判断）
- 操作：
  - [今日の約束にする]
  - [今回はしない]
- 意味：今日の約束を作るか否かの**唯一の判断**

### 遷移
- Yes → Today
- No → Intent / Life / History

## 3. Today（実行）
- 操作：
  - 完了
  - 保留
  - 取り消す
- 意味：今日の約束を実行・解除

### 割り込み対応
- 新規発生は Inbox に放り込む
- Today の約束は変更しない

## 4. History（事実ログ）
- 操作：閲覧のみ
- 意味：実際に起きた事実を残す

## 例：割り込みが起きた日
- 朝：Today を1件設定
- 昼：割り込み発生 → Inbox に放り込む
- 夜：Today を完了 → History

## 禁止事項
- フローのショートカット
- Today での新規追加
- 自動判断

## 未確定事項
- 音声入力の精度
- アーカイブ復元可否
