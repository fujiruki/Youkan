# 例外・やさしい救済（完了し忘れ／翌日起動）（確定版 v2）

## 目的
本ドキュメントは、操作忘れ・中断・翌日起動といった
**現実的な例外**を責めずに回収するためのUIと操作を定義する。
意味の再解釈や自動評価は行わない。

## 基本原則
- 約束は Today に入った瞬間だけ成立する
- 失敗という概念を作らない
- ユーザーに選択権を残す

## ケース1：Todayを完了せずに日付を跨いだ

### 翌日起動時の表示
```
┌──────────────────────────────┐
│ 昨日の約束について            │
├──────────────────────────────┤
│ ・◯◯建具 製作                │
│                              │
│ どうなりましたか？            │
│ [完了した] [今日はやらなかった] │
└──────────────────────────────┘
```

### 選択肢の意味
- **完了した**
  - 遷移先：History
  - 意味：事実として記録
- **今日はやらなかった**
  - 遷移先：今日の約束にするか
  - 意味：約束を解除し、再判断対象に戻す

※ どちらを選んでもペナルティはない

## ケース2：Todayを開かずに2日以上経過

### 振る舞い
- 直近の Today のみを確認対象にする
- それ以前は自動的に「今日はやらなかった」と扱う
- 履歴に失敗記録は残さない

## ケース3：途中で取りかかれなくなった

### 操作
- [保留] を選択

### 結果
- 翌日、今日の約束にするか に戻る
- 自動で Today に再投入しない

## 文言設計（やさしさ）
- ×「未達成です」
- ○「どうなりましたか？」
- ×「期限切れ」
- ○「今日はやらなかった」

## 禁止事項
- 未完了数の表示
- 連続未達成の警告
- 強制的な再Today投入

## 未確定事項

- 複数Todayが存在した場合の扱い
