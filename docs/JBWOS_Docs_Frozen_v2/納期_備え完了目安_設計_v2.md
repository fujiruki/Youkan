# 納期／備え完了目安（ぼやけた状態）設計（確定版 v2）

## 位置づけ
本設計は「納期」と「備え完了目安」を区別して扱う。
どちらも **判断（今日の約束にするか）を発生させない情報**である。

- 納期：相手との約束（確定）
- 備え完了目安：自分の安心のための目安（非確定）

## 納期（確定）
- 取付日・提出日など、相手が関与する日付
- 文字情報として明示表示
- カレンダー上では明確な文字と線で表示

## 備え完了目安（ぼやけた状態）
- 「いつごろ終わらせておきたいか」の目安
- 約束ではない
- 守れなくても失敗にならない

### ぼやけた状態の意味（再掲）
- 日付は持つが、確定ではない
- 背景色・量感・ぼかしで表現
- Today に入るまで約束にならない

## 量感カレンダー（ワイヤ）

```
月  火  水  木  金  土  日
──────────────────────
 29  30  31   1   2   3   4
 ███ ███ ░░░ ░░░ ░░░
        ▲備え目安（ぼやけ）

           ■納期（確定）
```

- ■：納期（確定・文字あり）
- ░░░：備え完了目安（ぼやけ）
- ███：負荷量の背景表現

## 操作
- 備え完了目安はドラッグで移動可能
- 納期は編集画面からのみ変更

## 判断との関係
- 納期・備え完了目安が近づいても自動判断はしない
- 判断は必ず GDB（今日の約束にするか）で明示的に行う

## 禁止事項
- 備え完了目安を約束扱いする
- 備え完了目安から自動で Today に入れる
- 納期から自動で判断を生成する
