# Today 画面 UI・操作フロー（確定版 v2）

## 画面の位置づけ
Today は **今日の唯一の約束** を実行するための画面である。
ここに入った瞬間だけ約束が成立する。

## 画面構成

```
┌──────────────────────────────┐
│ 今日やること（Today）          │
├──────────────────────────────┤
│                              │
│ 【集中】                      │
│ ・◯◯建具 製作                │
│                              │
│ 【軽】                        │
│ ・工場を片付ける              │
│                              │
│ 【ついで】                    │
│ ・銀行／買い物／手紙          │
│                              │
│──────────────────────────────│
│ [完了] [保留] [取り消す]       │
└──────────────────────────────┘
```

## 表示ルール
- 約束は最小限に表示する
- 「集中」「軽」「ついで」は視覚的強弱のみ
- 色は目立たせすぎない

## 操作

### 完了
- 遷移先：History
- 意味：事実として記録

### 保留
- 意味：今日は実行しなかった
- 翌日に自動で判断対象に戻る

### 取り消す
- 遷移先：今日の約束にするか
- 意味：約束を解除

## 完了し忘れた場合
- 翌日起動時に「昨日の約束」確認UIを表示
- 完了／未完了を選択できる

## 割り込みが起きた場合
- 新しいものは Inbox に放り込む
- Today の約束は変更しなくてよい

## 禁止事項
- Today での新規追加
- 優先度変更
- 未来日設定

## 未確定事項
- 集中枠の最大数
- ついでの上限数
