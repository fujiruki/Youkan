# README_for_Humans_and_AI（確定版 v2）

## 本システムの目的
このシステムは、**約束を減らし、守れる約束だけを作る**ための判断・実行支援システムである。
約束は未来に拡散させない。**約束は「今日」だけに限定する。**

## 最重要原則
- 約束が発生するのは **Today に入った瞬間のみ**
- それ以外は約束ではない（失敗にならない）
- 自動判断・自動優先はしない
- 未来の計画は「ぼやけた状態」で扱う

## 判断の定義（UI表示名）
- 内部概念：Judgment
- **UI表示名：今日の約束にするか**

> 判断とは、それを「今日の約束にするかどうか」を決める行為のみを指す。
> いつやるか／何時間かかるか／順番は判断しない。

## ぼやけた状態の意味
- 日付・時間・確定順序を持たない
- 視覚的には「背景量感」「淡色」「ぼかし」で表現される
- ぼやけた状態は約束ではないため、守れなくても失敗にならない

## レーン（状態）
- Inbox：放り込む場所（未整理）
- 今日の約束にするか：判断対象が置かれる場所
- Today：今日の唯一の約束
- Intent：やれたらいいこと（約束ではない）
- Life：習慣・生活維持
- History：事実ログ（完了・判断結果）

## Today の性質
- 量は少ない（集中できる範囲）
- 取り消し可能
- 今日を過ぎると約束ではなくなる

## 禁止事項
- GDB（今日の約束にするか）でスケジュールを決めさせない
- 未来日を確定させない
- 優先度ランキングを作らない
