# 実装後テスト観点 v2

本ドキュメントは、JBWOS 実装後に必ず確認すべきテスト観点をまとめたもの。
思想・UI・状態遷移が破壊されていないかを検証する。

---

## 1. 思想テスト（最重要）

### 1-1. 約束は Today のみか
- Today に入っていないアイテムが「約束」扱いになっていない
- 備え・Intent・Life が暗黙の期限や義務を持たない

### 1-2. ぼやけた状態は約束にならないか
- 備え完了目安は曖昧表示（背景量感・ぼかし）のみ
- テキスト強調・期限警告が出ていない

### 1-3. 判断は「今日の約束にするか」だけか
- 判断UIで聞かれるのは Yes / Hold / No のみ
- 実行計画や細分化判断を求めていない

---

## 2. UIテスト

### 2-1. 一覧性
- 一画面で大量のアイテムが視認できる
- スクロールしてもレーン構造が壊れない

### 2-2. 色と強度
- 確定（Today・納期）ははっきり
- 未確定（備え・Intent）は弱く・ぼんやり

### 2-3. 操作の軽さ
- 1操作で Inbox → 備え / 捨てる が可能
- Today で編集不能でも「怖さ」が出ない

---

## 3. 状態遷移テスト

### 3-1. 正常遷移
- Inbox → GDB → Today → History が成立する
- Today 完了忘れ時に救済UIが出る

### 3-2. 例外遷移
- キャパ不足時に「断る判断」が自然に出る
- waiting_external 長期化で再判断が発火する

---

## 4. ユーザー心理テスト

### 4-1. プレッシャー
- 開くだけで責められない
- 未消化が罪悪感にならない

### 4-2. 安心感
- 「表示されてないだけ」と理解できる
- History を見て自信が積み上がる

---

## 5. NG判定

以下が一つでも起きたら失敗とする。

- 備えが締切に見える
- Today が編集恐怖になる
- 判断回数が増えて疲れる
- 約束を破った感覚が強く出る
