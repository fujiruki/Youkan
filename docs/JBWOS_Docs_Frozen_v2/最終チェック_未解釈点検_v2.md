# 最終チェック（未解釈・矛盾点検）v2

本ドキュメントは、これまで出力された設計ドキュメント群を**再解釈せず**に点検し、
矛盾・未確定・注意点を明示するためのチェックリストである。

---

## 1. 用語・概念の一貫性

### 1-1. 判断（今日の約束にするか）
- 全ドキュメントで「判断＝今日の約束にするか」に統一されている
- 判断がスケジューリングや計画立案に拡張されていない

### 1-2. Today
- Today は唯一の約束領域として扱われている
- 編集不能である思想が UI / 操作フローで保持されている

### 1-3. ぼやけた状態
- 備え完了目安・量感カレンダーにおいて
  - 約束・期限・義務に見えない
  - 強調表示されていない

---

## 2. レーン構造の整合性

- Inbox / GDB / Today / History の役割が混線していない
- GDB は「判断を行う場」であり、作業管理ボードになっていない
- レーン移動は最小操作で完結している

---

## 3. UI・心理負荷

- 未完了・放置が責めに見えない
- 「表示されていないだけ」という安心導線がある
- Intent・ついで・備えが Today を侵食しない

---

## 4. 例外・救済

- 完了し忘れた場合の救済が存在する
- キャパ不足・納期逼迫時の「断る判断」が自然に出る
- waiting_external の再判断が強制になっていない

---

## 5. 未確定事項（明示）

以下は意図的に未確定として残されている。

- AIによる将来的な助言・注意喚起の頻度と表現
- 他者（父・外注）との共有UIの具体実装
- デバイス間（PC/スマホ）の完全な操作差分

※ 実装時に勝手に補完・推測してはならない。

---

## 6. 最終判定

以下が満たされていれば「完成形」とみなす。

- 約束の数が増えない
- 判断回数が増えない
- 失敗しても自信が減らない

一つでも崩れた場合、本設計は破壊されている。
