# アンチケース（失敗パターン）（確定版 v2）

## 目的
本ドキュメントは、本システムの思想を破壊する
**典型的な失敗パターン（アンチケース）**を明示し、
実装・運用時に踏み越えてはいけない境界を固定する。

## アンチケース一覧

### 1. 未来の約束を増やしてしまう
- 判断時に「明日やる」「来週やる」を選ばせる
- カレンダーで日付確定を促す

**なぜダメか**
- 約束が増え、守れない前提になる

### 2. 備え完了目安を約束扱いする
- 備えを Today に自動投入する
- 備え達成率を表示する

**なぜダメか**
- ぼやけた安心を約束に変えてしまう

### 3. 判断を自動化する
- 納期が近づいたら自動で判断対象にする
- 背景量感が高いと警告を出す

**なぜダメか**
- 人の判断を奪う

### 4. Today を増やしすぎる
- Today に上限を設けない
- 追加を容易にしすぎる

**なぜダメか**
- 約束の精度が下がる

### 5. 未達成を可視化する
- 未完了数を表示する
- 連続未達成を示す

**なぜダメか**
- 自己否定を生む

### 6. Intent をタスク化する
- Intent に期限を持たせる
- Intent の優先度を付ける

**なぜダメか**
- 気持ちの置き場が消える

## 原則の再確認
- 約束は Today のみ
- 判断は「今日の約束にするか」だけ
- ぼやけは守れなくてよい
