# Frozen v2 追補：納期・備え完了日・量感カレンダー思想固定

本ドキュメントは JBWOS Frozen v2 に対する**追補**であり、
既存思想を変更・拡張するものではない。
議論の結果、**暗黙的だった前提を明文化**する目的のみを持つ。

---

## 1. 日付の役割定義（再確認）

### 1-1. 納期（Deadline）
- 外部との約束
- 確定日
- 守る必要がある
- カレンダー上では「くっきり」表示
- テキスト（アイテム名）を表示してよい
- タップで詳細を開く

### 1-2. 備え完了日（Preparation Completion Target）
- 自分の安心のための目安
- 外部への約束ではない
- 変更可能
- 守れなくても失敗にならない
- カレンダー上では「ぼやけ」表示
- テキスト表示しない（原則）

### 1-3. 目安期間（Preparation Span）
- 備え完了日に向けて「忙しくなりそうな期間」
- 開始日は確定しない
- 判断・予定・約束ではない
- 背景量感（色の加点）としてのみ表現

---

## 2. 量感カレンダーでの表現固定

### 2-1. 表示ルール
- 納期：
  - はっきりした色
  - 輪郭あり
  - テキストあり
- 備え完了日：
  - 輪郭なし
  - ぼやけた色
- 目安期間：
  - 日付セル背景に色の加点
  - テキストなし

### 2-2. 操作ルール
- 色が濃くなっても判断は発生しない
- Today への自動遷移は禁止
- 判断は必ず GDB でのみ行われる

---

## 3. 空白タップ時の一覧表示

### 3-1. 目的
- 「このあたり何があるのか」を**確認するだけ**
- 判断や操作を誘発しない

### 3-2. 表示内容
- この期間に気配を持つアイテム一覧
- 文言例：
  - 「この期間に“気配”があるもの」

### 3-3. 禁止事項
- Today への直接操作
- 判断UIの表示
- 期限・予定を想起させる表現

---

## 4. 思想固定文

納期は約束であり、  
備え完了日は安心のための目安であり、  
その間の期間は「忙しくなりそうな気配」であって、  
いかなる判断・約束・予定でもない。
