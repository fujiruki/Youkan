# 実装AI_チェックリスト_量感カレンダー連動_Frozen_v2

本ファイルは JBWOS Frozen v2 における  
**量感カレンダー（Volume Calendar）と納期／備え完了目安の関係**について、  
実装時に思想逸脱が起きていないかを検証するためのチェックリストである。

## 前提（再解釈禁止）
- 納期：確定した約束。文字表示＋くっきり色
- 備え完了目安：安心したい目安。ぼやけ表示
- 開始目安期間：色の「加点」でのみ表現される
- 約束になるのは Today に入った瞬間のみ

---

## チェック項目

### A. 納期と備えの分離
- [ ] 納期は 1日単位で明確に表示されている
- [ ] 備え完了目安はタスク化されていない
- [ ] 備え完了目安は Judgment / Today に流入しない

### B. 開始目安期間（色加点）
- [ ] 開始日〜備え完了目安日の期間は「色の濃淡」でのみ表現されている
- [ ] 期間内に文字タスクが生成されていない
- [ ] 色は「ぼやけて」おり確定感がない

### C. カレンダー操作
- [ ] 日付セル（文字なし）タップで、色加点の由来アイテム一覧が表示される
- [ ] 一覧表示は「予定」ではなく「気配」として扱われている
- [ ] ドラッグ操作は備え完了目安のみ対象

### D. NG挙動（1つでもあれば失敗）
- [ ] 備え期間が自動でスケジュール化される
- [ ] 備え完了目安が期限扱いされる
- [ ] 納期と同一UIで表示される
- [ ] カレンダーが空白＝暇と誤解される

---

## 建具案件シミュレーション検証

### ケース1：自分完結
- 納期：3/1
- 備え完了目安：2/20
- 制作日数目安：8日

確認：
- [ ] 2/12〜2/20 に色加点が出る
- [ ] 3/1 のみ文字＋確定色
- [ ] Today に入るまで約束にならない

### ケース2：父・外注あり
- [ ] 外注工程は色加点のみ
- [ ] 他者都合は判断に昇格しない
- [ ] History に失敗ログが残らない

---

## 最終判定
- [ ] 本チェックを全通過している
- [ ] 「予定管理ツール化」していない
- [ ] Today 以外に約束が存在しない
