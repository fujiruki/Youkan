# 量感カレンダー（QuantityCalendar） インタラクション仕様案

## 1. 思想と目的
「量感（Volume）」という目に見えない圧力を、可視化された「線」として実感できるデザインを実装する。
未来のしかるべき日に、今の自分が支えられている（あるいは圧力を受けている）ことを、直感的にユーザーに伝えることを目的とする。

## 2. 基本動作 (Interaction)

| アクション | 挙動 | 目的 |
| :--- | :--- | :--- |
| **シングルクリック (Click)** | **Why?の可視化**<br>詳細モーダルは開かず、カレンダー上で視覚的なフィードバックのみを行う。 | どの日付・どの案件が「今日のこの量感（色）」を作っているのかを瞬時に理解させる。 |
| **ダブルクリック (Double Click)** | **Detailsの表示**<br>関連プロジェクト一覧（モーダル）を表示する（既存動作）。 | 詳細を確認し、具体的なタスク操作を行いたい場合に用いる。 |

## 3. シングルクリック時の視覚効果 (Visual Effects)

クリックされたセル（起点）と、その色の原因となっているアイテム（終点）を視覚的に結びつける。

### A. 関連アイテムの発光 (Flash Items)
*   **対象**: その日付の「量感スコア」に加算されている全てのアイテム（納期チップ）。
*   **効果**: カレンダー上に表示されている該当アイテムのコンポーネントに対し、約2秒間の「強調エフェクト（Glowing / Pulse / Highlight）」を与える。
*   **目的**: ユーザーの視線を、クリックした場所から「原因（納期）」へと誘導する。

### B. 圧力の可視化ライン (Pressure Lines)
*   **描画**: 
    *   **Start**: クリックされた「セルの中心」
    *   **End**: 該当する「アイテムチップの中心」
    *   これらを結ぶ滑らかな曲線（ベジェ曲線）をオーバーレイ描画する。
*   **デザイン**:
    *   光のようなライン（半透明の黄色～オレンジ、Amber系）。
    *   Source → Target へ流れるようなアニメーション描画が望ましい。
*   **持続時間**: 約2秒間表示し、その後フェードアウトする。
*   **画面外の扱い**: 
    *   当面は「画面内に表示されているアイテム」のみを対象として線を引く。
    *   スクロールが必要な位置にあるアイテムへの線は描画しない（または画面端でフェードアウト）。

## 4. 実装メモ
*   `QuantityCalendar` コンポーネント内に、クリック時のステート（`active pressure source`）を持たせる。
*   SVGレイヤーをカレンダーの前面（`z-index`高）に配置し、座標計算を行って線を描画する。
*   各 `CalendarItemChip` に `ref` または `id` を持たせ、座標を取得できるようにする。
