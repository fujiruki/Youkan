# 判断詳細ビュー_ワイヤー固定_1枚.md
最終更新: 2026-01-12 (JST)

本ドキュメントは、**判断詳細ビューのUIを1枚で固定**するための最終ワイヤーである。
実装時は、この構成から逸脱してはならない。

---

## 画面目的

- 判断に集中する
- 現実（納期・条件）を確認できる
- しかし圧を与えない
- Yes / Hold / No を自然に選べる

---

## ワイヤー（1枚固定）

```
┌──────────────────────────────────┐
│ 判断詳細                          │
├──────────────────────────────────┤
│                                  │
│ 【判断対象】                     │
│  ◯◯建具｜製作開始を判断する      │
│                                  │
│──────────────────────────────────│
│                                  │
│ 【判断材料】                     │
│  取付日：10/30                   │
│  想定製作日数：5日               │
│  材料リードタイム：3日           │
│  バッファ：2日                   │
│                                  │
│  （参考）                        │
│  RDD_start：10/23                │
│                                  │
│──────────────────────────────────│
│                                  │
│ 【判断】                         │
│                                  │
│  [   Yes   ]   [  Hold  ]   [ No ]│
│                                  │
│──────────────────────────────────│
│                                  │
│  ← Todayに戻る（未判断）          │
│                                  │
└──────────────────────────────────┘
```

---

## レイアウトルール（厳守）

### 1. 情報階層
1. 判断対象（最上段）
2. 判断材料（中段）
3. 判断ボタン（下段）

逆転は禁止。

---

### 2. 色・強調

- 赤色：使用禁止
- 警告アイコン：使用禁止
- 残日数表現：禁止
- 「締切」「急げ」等の文言：禁止

すべて通常テキスト。

---

### 3. ボタン仕様

- Yes / Hold / No は横並び
- サイズは同一
- 強調（デフォルト選択）なし

意味：
- Yes：引き受ける
- Hold：今は決めない
- No：今回は引き受けない

---

### 4. 操作ルール

- ボタン押下＝即判断確定
- 確認ダイアログ：なし
- 戻る操作＝未判断のまま

---

## 禁止事項（思想テスト）

- 判断前に納期が見える導線
- 判断材料を折りたたむUI
- 「おすすめ」「AI提案」表示
- 判断を急かすアニメーション

1つでもあれば思想違反。

---

## 実装AIへの最終指示

> このワイヤーは装飾案ではない。  
> 判断体験そのものの固定定義である。  
>  
> レイアウト・情報順・選択肢は  
> 理由なく変更してはならない。
