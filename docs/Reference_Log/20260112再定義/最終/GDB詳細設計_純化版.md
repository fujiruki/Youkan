# GDB詳細設計（純化版・最終）
最終更新: 2026-01-12 (JST)

本ドキュメントは、従来の「GDB詳細設計_再定義.md」に代替する正式版である。
GDB（Global Decision Board）の役割を判断に完全特化させる。

## 1. 定義
GDBとは「未来に向けた判断を、今日という時間軸から切り離して終わらせる場所」である。

## 2. GDBに存在してよいもの
- Inbox（未判断）
- 判断が必要なDecision
- 連絡待ち（Waiting）
- 確定した未来（判断不要な事実）

## 3. GDBに存在してはいけないもの
- 今日やる（Ready）
- 実行中・進捗・ログ
- タスク・チェックボックス

## 4. 操作
- 判断する（Yes / No / 保留）
- 保留理由を残す
