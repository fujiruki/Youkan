# イベント駆動設計（Decision確定 → 副作用一覧） v1.0
最終更新: 2026-01-12 (JST)

本ドキュメントは、本システムにおける**イベント駆動設計（Event-Driven Design）**を定義する。
すべての副作用は「Decisionが確定した」という**事実イベント**から発生し、
UIや人の操作に依存しない。

---

## 0. 基本方針（重要）

- 副作用は **直接呼ばない**
- すべて **イベント購読**によって発生する
- UIはイベントを「起こす」だけで、「何が起きるか」を知らない

---

## 1. 中核イベント定義

### Event: DecisionResolved
**発火条件**
- `tbl_Decision.Resolved = false → true`

**ペイロード**
```json
{
  "decisionId": 123,
  "type": "start | material | order | estimate | exception",
  "projectId": 45,
  "relatedId": 67,
  "resolvedAt": "2026-01-12T09:30:00+09:00"
}
```

---

## 2. Decisionタイプ別 副作用一覧

### 2.1 start（制作開始Decision）

**購読サービスと副作用**

1. ProjectService
   - `Project.Status = execution`

2. ProductionService
   - ExecutionBlock を `E_mfg` 日分生成
   - ExecutionContext を Active 化

3. DecisionService
   - start Decision を Resolved に確定
   - 未解決 Decision（estimate/material/order）を残存

4. ExceptionService（条件付き）
   - 見積未完の場合：
     - Exception Decision を生成
     - `ExceptionStart = true`

---

### 2.2 material（材料判断Decision）

1. MaterialService
   - ProductionMaterial.DecisionDone = true

2. RDDService
   - 材料発注RDDを再算出

---

### 2.3 order（材料発注Decision）

1. OrderService
   - Ordered = true
   - OrderedAt = now()

2. RiskService（任意）
   - リードタイム超過リスク記録（警告表示は禁止）

---

### 2.4 estimate（見積Decision）

1. EstimateService
   - Estimate.Completed = true

2. RDDService
   - 全RDD再算出

---

### 2.5 exception（例外Decision）

1. AuditService
   - 例外理由・日時を監査ログへ記録

2. UIService
   - GDB/Today に「例外」バッジ付与（強調禁止）

---

## 3. 実行イベント

### Event: ExecutionBlockCompleted

**発火条件**
- ExecutionBlock.Status = done

**副作用**
- 次の pending Block を待機状態に
- 全Block完了時：
  - Project.Status = done
  - ExecutionContext.Active = false

---

## 4. Life関連イベント

### Event: LifeCompleted / LifeSkipped

- 副作用なし
- Execution停止/再開判定は **定期評価ロジック**で行う
- Lifeイベントは Decision/Event に昇格しない

---

## 5. 禁止パターン（設計違反）

- UIから複数サービスを直接呼ぶ
- Decision確定と同時に副作用を直列実行する
- 例外開始をログ無しで通す
- Eventを保存せず副作用だけ起こす

---

## 6. 実装メモ（AI向け）

- EventはDBトランザクション後に発火
- 冪等性を必ず担保（decisionId + type）
- 失敗時は再実行可能にする（At-least-once）
