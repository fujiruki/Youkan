# History（事実ログ）を「自信」に変える設計.md
最終更新: 2026-01-12

本ドキュメントは、JBWOSにおける History（事実ログ）を
**反省・管理・監視のための履歴**ではなく、
**意思決定の質を可視化し、ユーザーの自信を育てる装置**
として設計するための最終仕様を定義する。

---

## 1. 前提思想（最重要）

- Historyは「評価」しない
- 良し悪しを判定しない
- 行動を矯正しない

Historyの目的はただ一つ。

> **「自分は、ちゃんと考えて選んできた」という感覚を残すこと**

---

## 2. Historyに記録されるもの／されないもの

### 記録される
- 判断（Yes / Hold / No）
- waiting_external からの再判断
- 断った判断
- Executionの開始・終了
- Lifeの実行（軽いログ）

### 記録されない
- やらなかったLife
- 迷って閉じた判断
- 未入力の納期

---

## 3. History画面の基本構造

```
┌ History ──────────────────────────┐
│                                   │
│ 2026/01/12                        │
│  ・◯◯建具｜製作開始（Yes）        │
│                                   │
│ 2026/01/10                        │
│  ・△△案件｜今回は見送る（No）    │
│                                   │
│ 2026/01/07                        │
│  ・□□案件｜取付日未確定→待つ     │
│                                   │
└───────────────────────────────────┘
```

- 時系列のみ
- 数値集計なし
- KPIなし

---

## 4. 表現ルール（超重要）

### 文言ルール
- 「失敗」「遅延」「未達」などは禁止
- 使う言葉は「選んだ」「決めた」「待った」

### 色・装飾
- モノクロ基調
- 強調・警告色なし
- NoもYesも同じトーン

---

## 5. 自信を生む3つの仕掛け

### ① Noが並ぶことを肯定する
- Noは赤くしない
- 「今回は見送る」という自然文言

### ② 待つ判断を残す
- waiting_external を履歴として残す
- 「待った」という行為が可視化される

### ③ やり切ったExecutionの終了点
- 完了時のみ1行で残す
- 工数・時間は出さない

---

## 6. 見返し方の設計（UX）

### 想定される使い方
- 忙しかった月を振り返る
- 「なんで疲れてるのか」を確認
- 断った数を見て安心する

### やらせない使い方
- 成果比較
- 他人との比較
- 反省会

---

## 7. 思想テスト（History編）

- Historyを見て責められた気持ちにならないか
- Noが誇りとして残っているか
- 待つ行為が無駄に見えないか

1つでもNGなら設計違反。

---

## 8. 固定宣言

> Historyは、  
> 過去を裁く場所ではない。  
>  
> **自分が選んできた道を、静かに確認する場所である。**
