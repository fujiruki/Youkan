# Life × Execution 同日混在時の表示優先度設計.md
最終更新: 2026-01-12

本ドキュメントは、Life（人生側の行動）と
Execution（仕事の実行ブロック）が**同じ日に存在する場合**の
Today画面における表示優先度・配置・思想を最終固定する。

---

## 1. 前提思想（最重要）

- Lifeは「管理対象」ではない
- Executionは「今日進める責任ある作業」
- 生活が仕事を侵食してもいけないし、
  仕事が人生を塗りつぶしてもいけない

Todayは「やることリスト」ではなく  
**その日の現実を壊さずに見渡す画面**である。

---

## 2. 結論（表示優先度）

### 優先度の順序
1. Execution（Commit済み・継続中）
2. Life（任意・単発）
3. その他（メモ・参照情報）

※ 優先度＝**上に表示される順**  
※ 重要度や価値の上下ではない

---

## 3. Today画面の基本構造（固定）

```
┌ Today ───────────────────────────┐
│                                  │
│ 【Execution】                    │
│  ────────────────                │
│  ■ ◯◯建具 製作（2日目）          │
│                                  │
│ 【Life（任意）】                 │
│  ────────────────                │
│  ・ゴミ出し                      │
│  ・銀行に行く                    │
│                                  │
└─────────────────────────────────┘
```

---

## 4. Execution表示ルール

- 常に最上段
- 件数が多くても折りたたまない
- 継続中の場合は「◯日目」と表示
- Lifeより視覚的に強い

理由：
Executionは「今日を拘束している事実」だから。

---

## 5. Life表示ルール

- Executionの下に独立セクション
- 「任意」と明記
- 0件でも表示してよい
- 未実行でも失敗扱いしない

Lifeは、
「今日、できたらいいこと」の集積。

---

## 6. 混在時の思想テスト

以下が成立しているかを常に確認する。

- ExecutionがLifeに埋もれていないか
- Lifeが義務のように見えていないか
- Lifeをやらなかった日が失敗に見えないか

1つでもNGなら設計違反。

---

## 7. 例：現実的な1日の見え方

### ケースA：忙しい日
- Execution：2件（製作・材料加工）
- Life：0件

→ TodayはExecutionのみ。  
何も欠けていない。

### ケースB：余白のある日
- Execution：1件
- Life：3件

→ Lifeが下に並ぶ。  
「今日はこれもできるかも」と思える。

---

## 8. 固定宣言

> Todayとは、  
> 今日やることを詰め込む場所ではない。  
>  
> **今日という一日を壊さずに置く場所である。**
