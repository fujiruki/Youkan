# 03. ターゲットユーザーとコンテキスト (User Profile & Context)

## 1. ターゲットユーザー：晴樹 (Haruki)
- **属性**: 1〜5人規模の建具店経営者・職人。
- **役割**: 経営、見積もり、現場採寸、製作、取付、経理を一人称（または少人数）でこなす。
- **性格**: 
    - 責任感が強い。「ちゃんとやりたい」というプロ意識が高い。
    - ゆえに、管理できない状態（カオス）に対して強いストレスと罪悪感を感じる。
    - 「完璧な計画」を作ろうとして、情報収集過多で動けなくなる傾向がある。

---

## 2. 建具屋という仕事の特殊性 (The Uncertainty)
なぜ既存のToDoアプリやガントチャートが晴樹氏を救えないのか。それは建具業特有の「不確実性」にある。

### A. 依存の連鎖
建具は建築の最終工程に近い。ゆえに、前工程（大工、左官、塗装）の遅れや変更のしわ寄せを全て受ける。
- 「枠がまだ出来ていないので採寸できない」
- 「床のレベルが合わないので寸法が変わる」
- 「施主の意向で材質が変わった」
→ **事前に引いたガントチャートは、翌日にはゴミになる。** リスケジュールのコストだけで一日が終わる。

### B. 断片的な情報の集積
一つの建具を作るのに、数回の「判断」が必要になる。
1. 概要見積もり
2. 現場採寸（1回目：枠なし）
3. 現場採寸（2回目：枠あり・確定）
4. 金物選定（施主確認待ち）
5. ガラス種類変更
→ 情報が五月雨式に入ってくるため、**「一つのタスク」として管理できない。**

### C. 物理的移動と割り込み
現場への移動、電話、急な来客。
- PCの前に座ってじっくり計画を立てる時間は取れない。
- **「起動即判断」**できるスピード感がなければ、ツールは使われなくなる。

---

## 3. なぜJBWOSが必要なのか

### 既存ツールの失敗・敗北パターン
1.  **Googleカレンダー**: 「未定」の予定を登録しづらい。「とりあえず」で入れた予定が延期されるたびにドラッグ＆ドロップするのが苦痛になり、放置される。
2.  **紙の手帳**: 書くのは早いが、リマインドがない。過去のページに書いた重要なメモ（変更指示）を見落とし、製作ミス（致命傷）に繋がる恐怖がある。
3.  **高機能ToDoアプリ**: 入力項目（期限、優先度、タグ）が多すぎて、登録自体が仕事になる。「Inbox」が数100件になり、見るだけで吐き気がする「開かずのアプリ」となる。

### JBWOSの勝ち筋
晴樹氏が必要としているのは、高機能な管理ツールではない。
**「今、自分の頭の中にあるモヤモヤ（未確定事項）をすべて安心して預けられ、必要な時にだけ『これ決めて』と提示してくれる秘書」**である。

- 状況が変わっても、タグ（Waiting理由）を変えるだけでいい。
- 予定が狂っても、カレンダーの「ぼやけた色」がズレるだけで、警告は鳴らない。
- 日付を決めなくても、「Inbox」に入れておけば絶対にロストしないという安心感。

JBWOSは、**「管理を諦めた人」を救うのではなく、「管理しようとして溺れた人」を救うための浮き輪**である。
