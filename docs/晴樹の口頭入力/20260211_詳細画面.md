# 詳細画面

## 詳細画面のカレンダー
- [ ] 詳細画面におけるカレンダーの行の高さは50px程度でOKです。
- [ ] アイテムは表示せず、背景色による量感が表示されてほしい。「じゃあこのあたりが空いているからここにマイ期日を指定しよう」という判断を下すことができるように。
    この量感は本アイテムだけではなく、量感カレンダーで表示するものと同じ、フィルタなどの適用も同じとしてください。量感カレンダーと同じ物背景色がここに表示されてほしい。それによって、時間の埋まり方を見ながら、今開いているタスクをいつ実施するかを考えやすくするため。だからその画面にはそのアイテムの量感だけでなく、現在のそのユーザーやフィルターの条件に応じて、関連する全アイテムの量感が表示できる必要がある。
    - [ ] データの全件参照
        詳細画面を開いている間も、バックグラウンドでのフィルタ条件（自分のみ、会社全体など）を維持したまま、全アイテムの計算結果（QuantityEngine のメトリクス）をカレンダーへ渡します。これにより、メイン画面と同じ「混み具合」が詳細画面でも確認可能になります。
    - [ ] 「量感のみ（アイテム非表示）」モードの導入
        ユーザー様の「アイテムは表示せず、背景色のみ」というご要望に応え、カレンダーに layoutMode="volume_only"（仮）を追加します。
        これにより、アイテムのチップ（青や緑の四角）は表示されず、背景のグラデーション（黄色やオレンジ、赤）のみが見えるようになります。
    - [ ] 行の高さとサイズ感
        行の高さを約50pxに調整し、PC画面でもパッと見て空き状況がわかるようにします。アイテムを表示しない分、セルの視認性が高まります。
    - [ ] コンテキストの同期
        メイン画面で「誰のタスクを見ているか」「どの会社のプロジェクトを見ているか」というフィルター条件を自動的に詳細画面のカレンダーにも引き継ぎます。

        アイテムの完全非表示: 「選択中のアイテム」の期限（納期）や My期限も、チップとしては表示させない（背景色のみにする）。セルの枠線によって、「期限」と「目安期間」を強調表現する。
        
        特定日の詳細確認: 日付をクリックすると、その日に影響しているアイテムのリストはヒント表示してほしい。小さくて良い。

- [ ] 詳細画面全体の高さは表示領域高さいっぱいから、上下に15px控えた高さにしてほしい。カレンダーをできるだけ週数表示するため。




# 詳細画面にてプロジェクト化
詳細画面にはプロジェクト化する機能がありますよね。プロジェクト化するとサブタスクを追加できる。しかしこの機能が機能していない。原因を調査し、修正して。

# 詳細画面のカレンダーを量感カレンダーに
詳細画面の納期やマイ期限を入力するカレンダーを、量感カレンダーにしたい。
・前後の月との連結も再現してほしい。そのためにカレンダー部分は上下スクロールでシームレスにめくれるようになる。当然最初開いた瞬間は設定されている納期のある月、納期が未設定なら現在の月が最初は表示される。
・量感カレンダーと同じように量感が表示されている。これによって、開いてる、色が薄い日をマイ期限として設定しやすい。開いていないから受注できないということも確認しやすい。
・マイ期限から目安日数分前までの表示もさりげなくほしい。


- [ ] プロジェクト化してサブタスクを作ったとする、Inbox内のサブタスクのアイテムカードには「納期」が親タスクと同じ日として表示されている。しかし、サブタスクの詳細画面を開くと納期が未設定になっている。理想は、この詳細画面を開くと納期がその親タスクの日に設定されているのが理想。サブタスクは作られた時点で、その納期は親タスクの納期が設定されてほしい。



## アイテムの詳細画面について
- [ ] 納期をクリックしてからカレンダークリック、マイ期限をクリックしてからカレンダークリックでそれぞれを指定したい.


- [ ] 詳細画面の左上の会社・プロジェクトの選択を変更して、詳細画面を閉じたら、その変更が保存され、ダッシュボード上のアイテムのカードにも反映されてほしい。
