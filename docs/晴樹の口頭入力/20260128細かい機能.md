# プロジェクト一覧ページについて
・プロジェクト一覧ページにもURLを付与してほしい。PersonalとCompanyのそれぞれの一覧ページに。
・プロジェクト一覧ページには、ダッシュボードやパノラマモードで作ったプロジェクト（親タスク）も同じプロジェクトだ。ここに表示されるべき。もしかして処理として別れてる？
・プロジェクト一覧において、「基本（Personal)のところで新規プロジェクトを押したら、そのときは普通にプロジェクト名・カラーラベルだけのダイアログがいい。現状は「顧客名」・「目標粗利」の入力が求められるが、個人プロジェクト（例えば家族旅行）などには必要ない。
・プロジェクト作成における「顧客名・現場名」「目標粗利」などは、製造業プロジェクトの影響によって現れるべきだろうか？それとも製造業以外でもまず必要な概念だろうか？
・プロジェクト一覧において、「基本」タブでは、自分の生活や趣味のためのプロジェクト（結婚式・宴会・旅行など）が現れる。自分のタスクの中で「どの会社にも属していない、プロジェクト化されている（もしくは子タスクが存在する）タスク（プロジェクト）」が「基本」タブには表示されるべき。
・プロジェクト一覧において、「仕事」タブでは、会社ごとにセクションが別れてそれぞれの会社のプロジェクトが表示される。自分が担当とサれていないプロジェクトも、すべて表示される。
・プロジェクト一覧からプロジェクトを開くと、サブタスクが表示できる。
・プロジェクト一覧の仕事タブでは、会社が製造業プラグインをONにしている会社のセクションでの表示は製造業プラグインの影響を受ける。どんな影響を受けるだろうか？
・会社のプロジェクトの担当に割り当てられているプロジェクトがある場合は、そのプロジェクトが自分のダッシュボードのタスク（プロジェクト）の１つとしても表示される。
・会社のプロジェクトの中のサブタスクが自分に割り当てられている場合は、そのサブタスクも自分のタスクとしてダッシュボードに表示されている。
・プロジェクトのタスクの担当者を自分がやります。と担当者に自分を割り当てることができる。
・自分がプロジェクトの担当者かオーナーの場合は、他社員にタスクを割り当てることができる。

これによって、自分の担当しているプロジェクトやタスクはダッシュボードにも表示される。プロジェクト一覧からプロジェクトを開けば、他のプロジェクトや他社員のタスクなども確認できる。

# 詳細画面のカレンダーを量感カレンダーに
詳細画面の納期やマイ期限を入力するカレンダーを、量感カレンダーにしたい。
・前後の月との連結も再現してほしい。そのためにカレンダー部分は上下スクロールでシームレスにめくれるようになる。当然最初開いた瞬間は設定されている納期のある月、納期が未設定なら現在の月が最初は表示される。
・量感カレンダーと同じように量感が表示されている。これによって、開いてる、色が薄い日をマイ期限として設定しやすい。開いていないから受注できないということも確認しやすい。
・マイ期限から目安日数分前までの表示もさりげなくほしい。
