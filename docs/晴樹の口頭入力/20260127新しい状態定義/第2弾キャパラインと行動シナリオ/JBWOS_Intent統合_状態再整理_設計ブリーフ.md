# JBWOS 設計更新ブリーフ（Intent統合・状態再整理版）

## このファイルの目的
本ドキュメントは、**JBWOS（Judgment-Based Work Operating System）** の設計議論において、
直近で確定・整理された以下の内容を、**開発AIがそのまま設計・実装判断に使える形**でまとめたものである。

- Intent（急ぎではないが重要なこと）の正式な位置づけ
- 状態（Inbox / Focus / Pending / Waiting / Done）の再定義
- Focusの役割再整理（「今日やる」vs「今日触る」問題）
- 判断の重さと作業の重さの分離
- 期限（納期・マイ期限）と判断タイミングの関係整理
- UI（詳細画面・ボタン・表示）の再設計方針
- なぜそうなったかの**経緯と理由**

---

## 1. Intent を導入・独立させる理由（思想）

### 背景
ユーザー（藤田晴樹）は、以下の特性・理想を持つ。

- 忙しくなると「人生にとって重要だが急ぎではないこと」が消える
- しかし、それらをやらない状態は精神的ノイズになる
- 緊急タスクと同列に並ぶと、Intentは必ず負ける
- コヴィー博士の「第2領域」を守れる仕組みが必要

### 結論
**Intentは「状態」ではなく「意味レイヤー（価値軸）」として扱う。**

- Inbox / Focus / Pending / Waiting は「実務を回すための状態軸」
- Intent は「人生を前に進めるための意味軸」

👉 両者は交差してよいが、混ぜてはいけない。

---

## 2. Intent の正式定義（確定）

### 定義
Intentとは：
> 「急ぎではないが、人生・方向性・長期価値にとって重要なこと」

### Intentの性質
- 納期がない or 自分で決める
- 完了しなくても罪はない
- 存在しているだけで価値がある
- 忘れないことが最大の目的

### 実装方針
Intentは**Statusではなくフラグ（属性）**として実装する。

```ts
Task {
  id
  title
  status: Inbox | Focus | Pending | Waiting | Done
  isIntent: boolean
  intentNote?: string
}
```

---

## 3. 状態（Status）の最終定義

### 状態一覧と役割

| 状態 | 役割 | 意味 |
|---|---|---|
| Inbox | 判断待ち | 今は考えられないものを放り込む |
| Focus | 今日触る | 今日「意識と時間を向ける」もの |
| Pending | 今はやらない | 意図的に後回しにしている |
| Waiting | 自分の手を離れている | 他者・材料・承認待ち |
| Done | 完了 | ただの出口。振り返り義務なし |

---

## 4. 判断の重さ と 作業の重さ の分離

| 軸 | 内容 |
|---|---|
| 判断の重さ | Inbox → Focus / Pending |
| 作業の重さ | 目安日数 |
| 時間配置 | 量感カレンダー |

---

## 5. 期限の整理

- 納期：現実の締切
- マイ期限：自分の判断期限
- 目安日数：作業量

---

## 6. UI再設計の要点

- 「今日やる」→「Focusに入れる」
- 「保留」→「Pendingに送る」
- Intentトグル追加
