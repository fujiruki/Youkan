# Today 画面定義書（聖域の定義）

## 1. 概要 (Overview)
Today画面（以下「Today」）は、JBWOSにおける**「今日一日の実行環境（Execution Environment）」**です。
ここは「タスクを整理する場所」ではなく、**「決まったことを淡々と、迷いなく実行する場所」**です。

### 存在意義: "Sanctuary"（聖域）
なぜ独立しているのか？
*   **外乱の遮断**: InboxやGDB（全体盤）には、未処理の案件や未来の不安（納期など）が渦巻いています。作業中にそれらが視界に入ると、集中力が削がれます。だから**「今日やること」だけを持ち込んだ個室（聖域）**が必要なのです。
*   **モードの分離**:
    *   **GDB**: 「司令官モード」で判断を下す場所。
    *   **Today**: 「兵士モード」で実行に徹する場所。
    *   この2つを混ぜると、作業中に「これ本当に今日やっていいのかな？」という迷いが生じます。

---

## 2. 画面構成要素 (Elements)

ソースコード (`TodayScreen.tsx`) および設計思想に基づく現在の構成要素です。

### ① Header（時間感覚）
*   **Time Progress Bar**:
    *   「1日（24時間）のうち、今何％が終わったか」を可視化。
    *   残タスク数ではなく、「残時間」を意識させることで、自然なペース配分を促します。
*   **「明日の計画」ボタン**:
    *   今日の作業を終えた後、スムーズに未来（Future Board）へ移行するための出口。

### ② Zone 1: Focus（集中領域）
**「今日、責任を持って進めるべきもの」**（最大2件）
*   ここには、**「実行中 (Active)」** のタスクが1つだけカードとして表示されます。
*   **Active Card**:
    *   **Action**: `中断 (Pause)` / `再開 (Resume)` / `完了 (Done)` のみ。
    *   「開始」ボタンはありません。表示されている時点で実行中です。
    *   **Edit**: タイトルをクリックして即座に編集可能（思考のアップデート）。
*   **Candidates (候補)**:
    *   まだ着手していないが、今日やる約束をしたもの。ここから「これからやる」を選んでActiveにします。

### ③ Zone 2: Light（軽作業キュー）
**「これからの予定」**
*   集中力を要しない、あるいは順番待ちのタスクリスト。
*   ここにあるタスクを「Focus」に引き上げる（Prioritize）ことも可能です。

### ④ Brain Dump（放り込み）
**「ノイズの排出」**
*   作業中に「あ、洗剤買わなきゃ」などの雑念が浮かんだら、ここに入力してEnter。
*   即座にInbox（GDB）へ飛び、Today画面からは消えます。
*   **目的**: 作業の手を止めずに、脳内のメモリを解放するため。

### ③ Zone 3: Life（生活）
**「ついで (On-the-way)」**
*   画面最下部に位置するチェックリスト（掃除、洗濯など）。
*   仕事の合間に「ついで」にやることで、生活が整っていく感覚を与えます。
*   **哲学**: 仕事だけして生活が崩壊するのはJBWOSの敗北です。

## 3. 独立した経緯 (History)

### Phase 1: Dashboard Integrated (初期)
最初はダッシュボードの左側に「Inbox」、右側に「Today」が並んでいました。
*   **問題**: 作業中にふと左を見ると、未読のInboxが溜まっているのが見えてしまい、「あっちもやらなきゃ」と気が散る（Pressure）現象が発生。

### Phase 2: Separate Screen (分離)
「作業中はInboxを隠すべきだ」という結論に至り、画面遷移型の独立スクリーンとなりました。
*   **効果**: 「今日やること」以外見えないため、没入感（Immersion）が向上。

### Phase 3: The Sanctuary (聖域化)
単なるリスト表示から、「実行カード」形式へ進化。
*   **進化点**: タイマー機能（Pause/Resume）の実装。これにより「単なるリスト」から「コックピット」へと役割が変化しました。
*   **現在**: "Immersive Mode" として、グローバルヘッダーすら隠し、時計とタスクだけに向き合う場所として完成しました。

---

## 4. ルール (Rules)

1.  **NO LOOK**: Todayにいる間は、GDB（全体盤）やInboxを見てはいけない。
2.  **NO JUDGE**: Today画面で「やっぱり明日でいいか…」という引き伸ばし判断をしてはいけない（それはGDBに戻ってやるべき）。
3.  **ONE WAY**: 基本的にGDB(夜/朝) → Today(昼) という一方通行の流れ。逆流（TodayからGDBへ戻す）は「敗北（Uncommit）」として扱う。
