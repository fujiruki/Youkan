# 詳細画面

## 詳細画面にてプロジェクト化
詳細画面にはプロジェクト化する機能がありますよね。プロジェクト化するとサブタスクを追加できる。しかしこの機能が機能していない。原因を調査し、修正して。

## 詳細画面のカレンダー
- [ ] 詳細画面におけるカレンダーの行の高さは50px程度でOKです。
- [ ] アイテムは表示せず、背景色による量感が表示されてほしい。「じゃあこのあたりが空いているからここにマイ期日を指定しよう」という判断を下すことができるように。
    この量感は本アイテムだけではなく、量感カレンダーで表示するものと同じ、フィルタなどの適用も同じとしてください。量感カレンダーと同じ物背景色がここに表示されてほしい。それによって、時間の埋まり方を見ながら、今開いているタスクをいつ実施するかを考えやすくするため。だからその画面にはそのアイテムの量感だけでなく、現在のそのユーザーやフィルターの条件に応じて、関連する全アイテムの量感が表示できる必要がある。
    - [ ] データの全件参照
        詳細画面を開いている間も、バックグラウンドでのフィルタ条件（自分のみ、会社全体など）を維持したまま、全アイテムの計算結果（QuantityEngine のメトリクス）をカレンダーへ渡します。これにより、メイン画面と同じ「混み具合」が詳細画面でも確認可能になります。
    - [ ] 「量感のみ（アイテム非表示）」モードの導入
        ユーザー様の「アイテムは表示せず、背景色のみ」というご要望に応え、カレンダーに layoutMode="volume_only"（仮）を追加します。
        これにより、アイテムのチップ（青や緑の四角）は表示されず、背景のグラデーション（黄色やオレンジ、赤）のみが見えるようになります。
    - [ ] 行の高さとサイズ感
        行の高さを約50pxに調整し、PC画面でもパッと見て空き状況がわかるようにします。アイテムを表示しない分、セルの視認性が高まります。
    - [ ] コンテキストの同期
        メイン画面で「誰のタスクを見ているか」「どの会社のプロジェクトを見ているか」というフィルター条件を自動的に詳細画面のカレンダーにも引き継ぎます。

        アイテムの完全非表示: 「選択中のアイテム」の期限（納期）や My期限も、チップとしては表示させない（背景色のみにする）。セルの枠線によって、「期限」と「目安期間」を強調表現する。
        
        特定日の詳細確認: 日付をクリックすると、その日に影響しているアイテムのリストはヒント表示してほしい。小さくて良い。

- [ ] 詳細画面全体の高さは表示領域高さいっぱいから、上下に15px控えた高さにしてほしい。カレンダーをできるだけ週数表示するため。

- [ ] 詳細画面のカレンダーの上部にも、このカレンダー用のフィルター選択ボタンがほしい。「全て・個人・会社」ですよね。そして、そのモーダルで開いたアイテムが個人所属の場合は個人、会社所属の場合は会社がデフォルトで選ばれていてほしい。

## サブタスク部分
    サブタスクには、入力欄があり、その下にはサブタスクが一覧される。
    サブタスクの入力欄に入力して＋ボタン、もしくはEnterを押すと、サブタスクが追加され、この下の一覧にも表示される。
    サブタスクには、もちろん所属会社が継承され、親アイテムは現在開いているプロジェクトアイテムが設定される。
    サブタスク一覧のアイテムをクリックすると、そのアイテムの詳細画面が開かれる。

# 詳細画面のカレンダーを量感カレンダーに
詳細画面の納期やマイ期限を入力するカレンダーを、量感カレンダーにしたい。
・前後の月との連結も再現してほしい。そのためにカレンダー部分は上下スクロールでシームレスにめくれるようになる。当然最初開いた瞬間は設定されている納期のある月、納期が未設定なら現在の月が最初は表示される。
・量感カレンダーと同じように量感が表示されている。これによって、開いてる、色が薄い日をマイ期限として設定しやすい。開いていないから受注できないということも確認しやすい。
- [ ] 現在開いているアイテムの目安期間も、マイ期限と同じ色の枠線で囲まれてほしい。
- [ ] 詳細画面の量感カレンダーでは、指示線を表示する機能はOFFにしてほしい。
## 目安期間の対象表示
- [ ] 詳細画面の量感カレンダーの下、メモの上に、「目安期間の対象表示」がほしい。この表示は小さいフォントサイズでよい。
- [ ] 目安期間の対象表示は、目安期間に何日の何時間が含まれているかを表示する機能。
    例）個人ユーザーが詳細画面で開いたタスクが次の場合、
        タスク名　タスクAをする
        所属会社　Z社
        マイ期限　2026/02/20
        目安時間　130分
        個人ユーザーがZ社に設定している時間配分
            2/17　90分
            2/18　30分
            2/19　00分
            2/20　60分
        この場合、目安期間の対象表示は
            「目安期間の対象：　2/17(40分), 2/18(30分), 2/20(60分)」
            説明：　2/20の60分、2/18の30分はその日に設定されている時間を使い切っている（100%)。2/17の40分は、40分　だけを消費し、残り「設定値90分-40分＝50分」が残っている。
        この場合、カレンダーには2/17,2/18,2/20の3日間が目安期間として青枠で囲まれている。 


## カレンダーが延長するように読み込まれても勝手にスクロールしない。
詳細画面で、
・「納期」「マイ期限」欄をクリックした直後、
・そのあとカレンダーをクリックしてその日付を設定した直後
このときに、カレンダーの端っこまで勝手にスクロールしてしまうバグがある。
本来は、どの操作をしても勝手にスクロールしないでほしい。表示されている領域は動かないのが理想。
読み込み済みのカレンダーの端っこに行ってカレンダーが延長された瞬間でも、その瞬間の表示されているカレンダーの領域が勝手にスクロールせず、現在の表示を維持してほしい。勝手にスクロールされるのは使いづらいから。


- [ ] プロジェクト化してサブタスクを作ったとする、Inbox内のサブタスクのアイテムカードには「納期」が親タスクと同じ日として表示されている。しかし、サブタスクの詳細画面を開くと納期が未設定になっている。理想は、この詳細画面を開くと納期がその親タスクの日に設定されているのが理想。サブタスクは作られた時点で、その納期は親タスクの納期が設定されてほしい。

- [ ] 詳細画面のカレンダーも、量感カレンダーと同じように上下にスクロールすると続きを読み込んでシームレスに表示し続けてほしい。
    最初に開いた段階で、当月と前後2ヶ月分は表示されるようにする。
    下方向は表示領域を超えるところまで読み込んでおく。
    それより下、それより上にスクロールバーで移動しようとしたとき、続きを読み込みながら無限にスクロールできるようにして。

- [ ] 量感カレンダーを開いた時、画面の中央に今日の月がくるようにしてほしい。

## 納期とマイ期限の設定方法
- [ ] 納期をクリックしてからカレンダークリック、マイ期限をクリックしてからカレンダークリックでそれぞれを指定したい.
- [ ] 納期・マイ期限をクリックして、次にカレンダーをクリックしたら設定できるという状態のときは、設定しようとしている方を光らせて設定中であることをわかりやすくしてほしい。

- [ ] 開いているアイテムの納期は赤色の枠、マイ期限は青色の枠で表示されている。目安日数が設定されている場合は、マイ期限の青色枠は目安期間すべてが青色枠で表示される。
- [ ] 個人アカウントのユーザー設定によって、表示しているアカウントが所属している会社（もしくは個人）の稼働時間が全日0に設定されている、または設定が不完全な場合に目安期間（青枠）が表示されず、ユーザーが「壊れている」と感じる問題を解消するため、UI上で警告を表示するようにします。

## 納期とマイ期限の前後関係警告
- [ ] 納期よりマイ期限が遅い日に設定されている時、マイ期限のフォント色を赤色にしてほしい。警告したい。


## 左上の所属会社と所属プロジェクト
- [ ] 詳細画面の左上の会社・プロジェクトの選択を変更して、詳細画面を閉じたら、その変更が保存され、ダッシュボード上のアイテムのカードにも反映されてほしい。

- [ ] 詳細画面の左上の、現在開いているアイテムが所属する会社名とプロジェクト名を表示する。プロジェクトは他のアイテムと同様、会社に所属（もしくは個人に所属）するものなので、プロジェクト名を変更しようとする時のプロジェクトリストは、その左の会社名に何が指定されているかで変わる。左の会社名が指定されていなければ、個人のプロジェクトリストが表示される。

## 日セルのクリック
- [ ] 詳細設定画面のカレンダーで、日セルの日付文字をクリックするとその日に関連するアイテムリストが表示されてほしい。しかし、セルの他の部分は、納期・マイ期日設定のためにクリックできるようにしてほしい。なぜなら、納期設定のために日セルをクリックしたときにそのリストが表示されると邪魔だから。

    このタスクの場合、目安時間が２４時間となっているので、だいたい３日だと思います。だとすると、マイ期限を1/23日に設定した場合、青枠が表示されるべきは1/21,1/22,1/23の３マスだと思われます。
    このようにしてほしい。
    つまりは、もし納期とマイ期日をこのように設定した場合、このアイテムが量感（マスの背景色）に影響を与える日がどれか、がわかる表示になるのです。

    JBWOS（特に量感カレンダー）の核心は、休日などの「非稼働日」を考慮したバックワード・アロケーション（遡り配置）です。カレンダーのマス背景色（量感）が濃くなっている「実稼働日」と、青枠で示される「目安期間」が完全に一致するようにしてください。

## 詳細画面のプロジェクト化とサブタスク
- [ ] 詳細画面の右下には、現在開いているアイテムがプロジェクト化されている場合は、そのプロジェクトのサブタスク一覧を表示してほしい。
- [ ] プロジェクト内にサブアイテムが作成された時、
    ・会社：親タスクと同じ
    ・プロジェクト：親タスク自身
    ・マイ期日：空欄
    ・納期：親タスクの納期と同じ日にち
    がデフォルトで設定される。
    これは、
        ・詳細画面の右下のサブタスクの欄からアイテムを作成した場合
        ・全体一覧２のプロジェクトの右の＋ボタンから作成した場合
        ・全体一覧２をProjectFocused状態で開いて入力欄からアイテムを作成した場合
    どの場合でも共通の動作である。


- [ ] 納期とマイ期限が左右に並ぶようにしてください。それによって高さ方向の領域を節約します。
- [ ] また、詳細画面下部はフッターのようにして、その上の中段の領域に重ならないようにして。
- [ ] 中段の中はスクロールができるようにして。

- [ ] 詳細画面中段には左右２列ありますが、スクロールは左右別々で動きます。また、左列のカレンダーも、これまた独立でスクロールで動きます。
