# プロジェクトについて
ダッシュボード上でタスクを「プロジェクト化」をしたら、そのタスクはプロジェクトです。
タスクとプロジェクトは同じテーブル上にあり、タスクの状態の１つであると言えます。そのテーブルに「これはプロジェクトです」という列があるんでしたっけ？それとも、子タスクが存在するなら、それはプロジェクトなんでしたっけ？
タスクは、親タスクを持つことができます。タスクの「親タスクID」列に記録することで、親タスクを記録します。親タスクを持っているタスクは子タスクと呼びます。親タスクを持たないタスクも存在できます。タスクに「親タスクのIDがはAAAです」という記録があれば、IDがAAAであるタスクはプロジェクトと言えます。
タスクは、「会社のID」「担当者ID」を持つことができます。タスクの会社IDが未設定ならば、個人的なタスクだということ。会社IDがあれば、その会社に関わるタスクだということ。
ダッシュボード上からプロジェクト化したタスクは、「基本（Personal）」のプロジェクトとします。つまり会社IDは未設定。プロジェクト一覧の「基本（Personal）」タブに表示されます。    

逆にプロジェクト一覧にて「基本（Personal）」タブでプロジェクトを作成すると、タスクの会社IDは未設定。しかしそれもタスクの「プロジェクトかどうかを示すフラグ」がONなだけで、タスクと同様にダッシュボードにも表示される。

プロジェクト一覧にて「会社（Company）」タブでプロジェクトを作成するとのタスクの会社IDは作成時に選んだ会社のIDが設定されている。しかしそれもタスクの「プロジェクトかどうかを示すフラグ」がONなだけで、タスクと同様にダッシュボードにも表示される。
