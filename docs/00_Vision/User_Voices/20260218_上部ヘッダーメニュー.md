# 完了非表示　チェックマーク
チェックマークがONなら、完了したタスク・プロジェクトを表示しない。OFFなら表示する。

# フィルタ（全て・個人・会社）
    全て：全てのタスク・プロジェクトを表示
    個人：個人が所属するタスク・プロジェクトを表示
    会社：会社が所属するタスク・プロジェクトを表示

全体構造
ヘッダーは、大きく分けて 3つの階層 で構成されています（画面最上部から順に）。

1段目：グローバル・ステータスバー (Global Bar)
システム全体の状況把握と、大枠のナビゲーションを担います。

メニュー引き出し（ドロワー）: 左端のハンバーガーアイコン。履歴、設定、マニュアル、顧客リストなどへの詳細なナビゲーション。
Health Check（ヘルスチェック）: 通信状態やシステムの状態をリアルタイムで表示。
Project Focused ラベル: 特定の案件にフォーカスしている際、「どの案件を見ているか」を表示。解除ボタンも併設。
Time Progress Bar（時間のプログレス）: 1日の24時間を可視化した目盛り。現在時刻が1日のどこに位置するかを直感的に示します。
Motivator Whisper（金言/名言）: ユーザーを鼓舞するメッセージをランダム（または状況に応じて）表示。
Reality Load（トータル負荷情報）: 「今日の持ち時間（キャパシティ）に対して、どれだけタスクが入っているか」を分単位とプログレスバーで表示。
アカウント・テナント表示: 現在のログインユーザー、または切り替え後の会社名（テナント名）を表示。
2段目：ナビゲーション・コントロール (Navigation Bar)
主要な機能間の移動と、各画面特有の表示切り替えを担います。

ダッシュボード・セクション:
登録と集中: インボックスと実行中のタスクに特化。
全体一覧: ボード形式。
全体一覧2: ニュースペーパー（グリッド）形式。
プロジェクト・セクション:
個人 / 会社: 案件のスコープ切り替え。
グリッド / リスト: 表示形式の即時切り替え。
カレンダー・セクション:（今回特に強化した部分）
グリッド: 月間カレンダー形式。
タイムライン: 横軸の時間軸。
ガント: 案件や工程管理。
新プロジェクト作成ボタン: どの画面からでも新しい案件を立ち上げられるアクションボタン。
3段目：統合コンテキストバー (ViewContextBar)
今回カレンダービューにも統合された、最も重要な動的フィルター層です。

コンテキスト・フィルタースイッチ:
自分 / 全て: 個人のタスクを中心に表示。
会社: 所属する全ての組織のタスクを俯瞰。
各テナントボタン: 自分が参加している組織（A社、B社など）の名前がボタンとして自動生成され、クリックすることで組織ごとのカレンダーやリストに即座に切り替わります。
立場ラベル (Perspective Label):
現在どのような「立場」でデータを見ているかをテキストで補足します。
例：「（社名）マネージャーとして」「自分の時間管理」など。
これにより、膨大なタスクの中で「今、何に責任を持って画面を見ているか」を迷わなくなります。


# プロジェクトフォーカスと状態遷移マトリクス
「プロジェクトフォーカスはベースコンテキストに対する『絞り込み（狭める）』である」
本来は「現在の表示範囲（ベース）の中で、さらに特定の案件に集中する（AND論理）」という階層構造が正しい姿です。「フォーカスがフィルタを上書きする（OR論理）」ではありません。


1. 解決したい課題
プロジェクトフォーカス時に、ベースフィルタ（個人/会社）の設定によってサブタスクが意図せず消失する問題。
「会社案件の中の自分のタスクだけ見たい」というニーズと、「自分の担当案件なら中身を全把握したい」というニーズの両立。
2. 提案する論理モデル：条件付き連鎖（Conditional Chain）
以下のロジックで判定を行います：

A. フォーカス対象（親）自体の表示
コンテキストフィルタの設定に関わらず、常に表示（構造の器として維持）。
B. 子要素（タスク・サブプロジェクト）の表示判定
「親」がフィルタを通過する場合（例：個人フィルタ中に個人案件をフォーカス）
⇒ その中の子は、属性に関わらず「全て表示」。
理由：自分の担当領域の中身は、属性の揺れに関わらず全て把握すべきであるため。
「親」がフィルタを通過しない場合（例：個人フィルタ中に会社案件をフォーカス）
⇒ その中の子のうち、「現在のフィルタに合致するもの」のみ表示。
理由：会社案件全体の把握ではなく、その中に混じっている「自分の分」だけを抽出したいため。
3. 具体的な例
フィルタ（背景）	フォーカス対象	対象の属性	子の表示結果
会社	プロジェクトA	会社	全表示（サブプロジェクト、全担当者のタスク）
個人	プロジェクトA	会社	フィルタ一致のみ（自分の担当タスクだけ抽出）
個人	プロジェクトB	個人	全表示（自分の領域なので詳細を把握）
4. サブプロジェクトへの適用
サブプロジェクトも上記「子要素」と同じルールで判定されます。
また、サブプロジェクト自体が「親」として機能する場合（さらに深い階層）、そのサブプロジェクト自体のフィルタ合致状況に基づいて、さらにその下の孫要素の表示条件が決定されます。
