# QuantityEngine
結論から言えば、ViewModel（第1層）に統一し、QuantityEngine からフィルタロジックを除去するのが最も合理的だと考えます。理由は3つです。

理由1：責務の分離
QuantityEngine の本来の責務は「与えられたアイテムリストに対して日別の負荷を計算する」ことです。何を表示するか（フィルタ） と どう計算するか（エンジン） は別の関心事で、QuantityEngine がフィルタを持つべきではありません。
