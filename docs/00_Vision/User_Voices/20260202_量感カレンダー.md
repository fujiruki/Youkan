# 量感カレンダーについて
- [ ] 量感カレンダーでは、各日付のユーザーがどのくらいの時間を使いたいか、というのをMAX値として、その日のタスクの合計時間がそのMAX値にどれだけ近いか、超えているかを色の濃ゆさで表現する。例えば、MAX値が10時間で、タスクの合計時間が5時間なら薄い色、10時間なら濃い色、15時間ならさらに濃い色、というようにする。
- [ ] そのmaxというのは、プライベート２時間＋仕事（複数の会社を合計して）８時間　などというように別々に設定できる。そして、その合計値がその日のmax値となる。ただし、プライベートの予定を入れる時はプライベートの予定とそのmax時間を元にした量感カレンダーが表示されたら良いなと思う。逆に仕事に絞った量感カレンダーでは、プライベートの時間は考慮しせず、仕事の日毎のmax時間、仕事のタスクの目安期間による。
- [ ] ユーザーは設定画面で、会社ごとに毎週の定休日とかを登録できる。それを反映して、定休日の日は背景色を濃い灰色にする。
# 量感カレンダー内のアイテムとセル・それのクリック
- [ ] 量感カレンダーに表示されるアイテムは、アイテム名＋プロジェクト名の先頭４文字。この１行のアイテムカードが量感カレンダーの中、納期の場所に表示されている。左クリックでそのアイテムの詳細画面が開く。
- [ ] 日のマスを左シングルクリックすると、その日の背景色に影響を与えているタスクすべてのカード中心に向けて、その日の中央から曲線がのびて指示する。曲線が伸びるアニメーションは0.5秒のみ。
- [ ] 日のマスを左ダブルクリックすると、その日の背景色に影響を与えているタスクの一覧が表示される。
- [ ] カレンダー上のアイテムカードをクリックすると、そのアイテムが影響与えているマスが光る。カレンダー上を１回クリックするとその表示状態がリセットされる。
- [ ] 量感カレンダーのレイアウトや背景色をそのまま詳細画面の納期選択・マイ期限選択・目安期間表示で使いたい。そのためにも、再利用しやすい構造にしてほしい。
- [ ] カレンダーに表示されるアイテムも表示されるかどうかはダッシュボードと同じような判断基準で行います。一番上のヘッダーのフィルターもう反映されるし担当者となっているアイテム・個人の自分のアイテムなどです。


- [ ] アイテムが作成されるとき、個人のアイテムなら作成者、会社に属するアイテムなら未割り当てで割り当てられる。
- [ ] プロジェクトに所属するアイテムとして作成される場合は、そのプロジェクトの担当者が割り当てられる。
- [ ] アイテムがプロジェクトに移動されたときは、移動先プロジェクトが属する会社に元の担当者のメールアドレスがあれば、担当者はそのまま。移動先プロジェクトが属する会社にもとの担当者のメールアドレスがなければ、その移動先プロジェクトの担当者がその担当者になる。
