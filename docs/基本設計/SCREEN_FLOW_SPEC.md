# 画面遷移・情報構成仕様書 (Screen Flow & Architecture Spec)

## 1. 概要 (Overview)
本アプリケーションは、単なる「建具エディタ」ではなく、案件ごとの建具管理・見積もり・作図を一元管理する「建具工事統合管理システム」として機能する。
そのために、画面構成を以下の**3階層モデル**に再構築する。

**階層構造:**
1.  **案件管理 (Projects Dashboard)**
2.  **建具表 (Joinery Schedule)**
3.  **建具編集 (Door Editor)**

---

## 2. 画面詳細仕様 (Screen Details)

### 2.1 案件管理 (Projects Dashboard)
*   **役割**: アプリケーションのトップ画面。「案件（現場）」単位でのデータ管理を行う。
*   **主な機能**:
    *   案件の新規作成・削除・名称変更。
    *   最近編集した案件のクイックアクセス。
*   **UIイメージ**: カード型グリッドレイアウト。

### 2.2 建具表 (Joinery Schedule)
*   **役割**: **本システムのメイン画面**。特定案件に含まれるすべての建具を一覧管理する。
*   **主な機能**:
    *   **一覧表示 (DataGrid)**: 符号（SD-1等）、サイズ、数量、配置などを表形式で表示。
    *   **インライン編集**: 基本的な情報（金物、材料、数量など）は、画面遷移せずにこの表上で直接編集可能とする。
    *   **全体見積もり**: 案件全体の合計金額（概算）を常時表示。
    *   **一括操作**: 建具の複製（Copy）、削除、DXF一括エクスポート。
*   **データ項目（列）**:
    *   No. / 符号 / 名称
    *   W / H / D (サイズ)
    *   数量
    *   主要材料 (Material Shortcut)
    *   主要金物 (Hardware Shortcut)
    *   単価 / 合計金額
    *   Status (編集中 / 完了 等)

### 2.3 建具編集 (Door Editor)
*   **役割**: 建具1枚の詳細な形状設計、プロパティ設定、作図を行う「作業場」。
*   **主な機能**:
    *   **パラメータ設計**: 寸法、部材設定（既存機能）。
    *   **プロパティ詳細入力**: 金物（型番）、詳細な材種設定。
    *   **単体見積もり**: 建具1枚あたりの詳細な原価計算と内訳表示（Pro機能）。
*   **UI構成**:
    *   左: パラメータ設定
    *   中央: プレビュー (Canvas)
    *   右: 詳細プロパティ & アクション（保存、DXFDL）

---

## 3. データ構造への影響 (Data Structure)

この3階層モデルを実現するため、データベース構造を `1対多 (Project : Door)` に変更する。

### Projects Table (案件)
*   `id`: unique ID
*   `name`: 案件名 (例: 「A邸新築工事」)
*   `client`: 顧客名
*   `createdAt`, `updatedAt`

### Doors Table (建具)
*   `id`: unique ID
*   `projectId`: **Projects.id への外部キー**
*   `tag`: 符号 (例: "SD-1")
*   `name`: 建具名称 (例: "木製框戸")
*   `dimensions`: 寸法データ (JSON)
*   `count`: 数量
*   `specs`: 仕様詳細 (材料、金物情報を含むJSON)
*   `createdAt`, `updatedAt`

---

## 4. 見積もり連携フロー (Estimation Flow)

1.  **Editor**: 寸法と設定値に基づき、リアルタイムで「単価」を計算・保存。
2.  **Schedule**: 各建具の「単価 × 数量」を計算し、案件全体の「総額」を集計・表示。

4.  **見積もり連携**: 計算ロジックを実装し、EditorとSchedule双方に表示。
