# 意思決定ログ：SVPによる環境トラブルの自動解決

## 背景
2026年2月23日の開発セッションにおいて、開発サーバーの起動およびブラウザ検証が以下の理由で失敗する事態が発生した。

1. **ポート占有 (5173)**: 前回のセッション等のゾンビプロセスがポートを掴んだままになっていた。
2. **接続タイムアウト**: 不完全なサーバー状態により、ブラウザエージェントが正常に応答を受け取れなかった。

## 解決策
ユーザーより提案のあった **`/verify-start` (SVP v4.0 / `./svp.ps1`)** ワークフローを実行した。

## 結果
- **自動復旧**: スクリプトが既存の `php.exe` および `node.exe` プロセスを特定・停止し、ポートを正常に解放した。
- **環境の安定**: 再起動後、Backend (8000) および Frontend (5173) の両方で正常な疎通（Health Check OK）が確認された。

## 今後の方針
- **標準プロトコル化**: 今後、Windows環境での起動失敗や接続エラーが発生した場合は、速やかに SVP を実行することを「標準的な復旧儀式」としてドキュメント化し、他のAIエージェントにも周知する。
- **開発ルールの更新**: `AI_DEVELOPMENT_RULES.md` および `AI_HANDOVER_CONTEXT.md` に本ツールを明記し、自律的な解決を促す。
