# GDB & Plugin Integration UI/UX Discussion

## 1. GDB Column Naming (Wording Refinement)

### Current Issues
- **「今日の約束にするか」**: 長すぎる、説明的すぎるため、一覧のヘッダーとして視認性が悪い。
- **「備え（ぼやけ）」**: "ぼやけ" (Blurry) という言葉がシステム用語として直感的でなく、違和感がある。
- **「Intent（やれたらいい）」**: 英語と日本語が混ざっており、重要度が伝わりにくい。

### Prospal (提案: 初心者に優しい「説明的」な名称)
ユーザーフィードバック「シンプルにしなくて良い、説明になっていてほしい」に基づき、状態やアクションを言語化した名称にします。

| Bucket | Current (現状) | **Proposed (推奨案)** | 意図 |
| :--- | :--- | :--- | :--- |
| **Active** | 今日の約束にするか | **【今日やるか決める (Inbox)】** | 具体的アクションを明示。初心者に「ここで何をするか」を伝える。 |
| **Preparation** | 備え（ぼやけ） | **【準備・出番待ち (Standby)】** | "ぼやけ"をやめ、状態（準備中、出番待ち）を説明する言葉に変更。 |
| **Intent** | Intent（やれたらいい） | **【いつかやれたら (Someday)】** | 英語のIntentよりも、日常的な「いつか」という言葉でニュアンスを伝える。 |

## 2. Plugin Project Integration (建具プロジェクト連携)

### Requirement (要件)
- プロジェクト作成時に「建具プロジェクト」を選択できるようにする。
- 作成された建具プロジェクトは、タスク管理側(JBWOS)だけでなく、建具プラグイン側(CAD/見積)のプロジェクトリストにも表示されるようにする。

### Proposed UI/UX Flow

#### 1. プロジェクト作成 (`ProjectCreationDialog`)
- **UI変更**: 「プロジェクトの種類」プルダウンを動的に拡張。
    - デフォルト: `一般プロジェクト`
    - プラグイン連携: `建具プロジェクト` (プラグインから自動登録)
- **挙動**:
    - `建具プロジェクト` を選択すると、建具製作に必要な**初期タスクセット（テンプレート）**が自動展開される。
        - 例: `[ ] 現場調査`, `[ ] 図面作成`, `[ ] 見積提出`, `[ ] 製作`, `[ ] 納品`
    - アイコン表示: フォルダアイコンの代わりに、建具アイコン（🚪）や特定の色で区別する。

#### 2. データ連携 (Data Linkage)
- Itemデータ構造に以下を保持:
    - `isProject: true`
    - `projectCategory: 'tategu-project'` (またはプラグインID)
- これにより、プロジェクトの種類を識別する。

#### 3. 建具プラグイン側の表示 (Integration)
- **現状**: 建具プラグインは独自のファイル/データを見ている（現状は未実装または別管理）。
- **改善案**:
    - 建具プラグインの「案件選択画面」で、JBWOSのアイテムDBをクエリする。
    - `SELECT * FROM items WHERE isProject = true AND projectCategory = 'tategu-project'`
    - これにより、**「タスク管理で作った案件」＝「CADで作図する案件」** が完全に一致する。
    - プロジェクトをクリックすると、その案件用の図面フォルダや見積データをロードする。

### Screen Transition (画面遷移)
1.  **GlobalBoard (JBWOS)**: 「+プロジェクト」→「建具プロジェクト」作成。タスクとして管理開始。
2.  **Navigation**: メニューから「建具設計 (Plugin)」へ移動。
3.  **Tategu Plugin View**: プロジェクト一覧に、先ほど作った「〇〇様邸 建具工事」が表示されている。
4.  **Action**: プロジェクトを選択 → 建具表エディタ(Grid)が開き、その案件の建具データの編集を開始できる。
