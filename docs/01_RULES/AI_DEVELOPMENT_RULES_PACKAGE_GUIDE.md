# AI_DEVELOPMENT_RULES_PACKAGE_GUIDE.md
― 自律的AI開発ルール v4.2 パッケージ概要・使い方 ―

## 1. このパッケージは何か

このパッケージは、
**AIが「考え・決め・作り・戻り・改善する」ことを前提にした  
自律的システム開発のためのルールセット**である。

目的は以下の3点に集約される。

- 人間の判断コストを極限まで下げる
- AIが迷わず前進できるようにする
- 大規模・長期プロジェクトでも破綻しない構造を作る

単なるREADMEではなく、  
**AIに最初に読ませる「開発憲法＋運用マニュアル一式」**である。

---

## 2. パッケージに含まれるファイル一覧

本パッケージには、以下の md ファイルが含まれる。

```
AI_DEVELOPMENT_CONSTITUTION_v4_2_FULL.md
AI_DEVELOPMENT_PLAYBOOK_v4_2_FULL.md
AI_DEVELOPMENT_OPS_v4_2_FULL_WITH_SVP.md
AI_MEETING_WORKFLOW_v4_2_FULL.md
Smart_Verification.md
AI_DEVELOPMENT_RULES_PACKAGE_GUIDE.md（本ファイル）
```

---

## 3. 各ファイルの役割

### 3.1 AI_DEVELOPMENT_CONSTITUTION
**最上位思想・判断基準（憲法）**

- 何を最優先するか
- どこで止まるか／止まらないか
- 日本語原則
- 多言語対応の扱い
- 人間介入の位置づけ

AIが判断に迷ったら、必ずここへ戻る。

---

### 3.2 AI_DEVELOPMENT_PLAYBOOK
**ウォーターフォール対応・開発手順書**

- 企画 → 要件定義 → 設計 → 実装 → テスト → 改善
- 各フェーズで
  - 何を決めるか
  - 何を決めてはいけないか
- フェーズ自覚・宣言ルール
- テンプレ使用前提

AIが「今どこにいるか」を見失わないための地図。

---

### 3.3 AI_DEVELOPMENT_OPS
**現場運用ルール（Windows / Git / 実務）**

- Windows 環境前提
- npm.cmd / && 禁止
- 相対パス原則
- 自動コミット、およびコミット後の出力を読んでコミット成功を確認する前提
- md生成事故の防止ルール

実装・作業中の「事故防止マニュアル」。

---

### 3.4 AI_MEETING_WORKFLOW
**AI専門家会議の正式手順**

- 「〜について会議して」で自動起動
- 専門家ペルソナ固定
- Divergence / Debate / Convergence
- 結論必須・md保存必須

「考えた気になるだけ」を防ぐ装置。

---

### 3.5 Smart_Verification.md
**検証・動作確認の補助プロトコル**

- ローカル開発サーバーの再利用
- 無駄な再起動の回避
- 手動検証の高速化

※ 判断権限は持たない。効率化専用。

---

## 4. AIに読ませる順番（重要）

新しいプロジェクトを始める際、  
AIには必ず以下の順で読ませる。

1. AI_DEVELOPMENT_CONSTITUTION
2. AI_DEVELOPMENT_PLAYBOOK
3. AI_DEVELOPMENT_OPS
4. AI_MEETING_WORKFLOW
5. Smart_Verification.md

この順序を崩すと、
- 判断基準がブレる
- フェーズ無視が起きる
- 勝手な最適化が始まる

---

## 5. 基本的な使い方（人間向け）

### 5.1 プロジェクト開始時
- このパッケージを同一フォルダに配置
- AIに「このフォルダ内の md をすべて読んで従ってください」と指示
- 企画・構想定義フェーズから開始

### 5.2 開発中
- 人間は「方向・確認・質問」だけを行う
- 実装や細かい判断は AI に任せる
- 不安があれば「憲法に照らしてどう？」と聞く

### 5.3 改善・方向転換時
- AI専門家会議を使う
- 結論を md に残す
- 必要なら v4.3 として改定する

---

## 6. このパッケージの思想的特徴

- AIを「道具」ではなく「作業主体」として扱う
- 人間は管理者ではなく「最終責任者」
- 失敗を恐れず、戻れる構造を重視
- 冗長さは悪ではなく、安全装置

---

## 7. 想定される利用シーン

- 個人開発
- AIペアプログラミング
- 長期・大規模システム開発
- 仕様が流動的なプロジェクト
- AI主導で回したい開発全般

---

## 8. 改定・配布について

- 本パッケージは自由に改定可能
- 改定時はバージョン番号を更新することを推奨
- チームで使う場合は、最初に憲法の合意を取ること
