# プロジェクト管理機能設計

## 概要

JBWOSにおけるプロジェクト管理は、**「今日やるかどうかの判断」**という核心哲学を維持しながら、個人タスクと会社プロジェクトを自然に融合させる。

---

## コンセプト

```
┌─────────────────────────────────────────────────────────────┐
│                     ダッシュボード                            │
│  「今日やるかどうか」を判断する場所                           │
│                                                             │
│  ・自分に割り当てられたタスク/プロジェクト（個人・会社問わず）  │
│  ・自分の判断を待っているもの                                │
└─────────────────────────────────────────────────────────────┘
                              ↕
┌─────────────────────────────────────────────────────────────┐
│                  プロジェクト一覧                             │
│  「全体像を把握」する場所                                     │
│                                                             │
│  ・基本(Personal): 旅行・結婚式・趣味など                     │
│  ・仕事(Company): 会社ごとにセクション分け                    │
│    └ 自分担当でないプロジェクトも含め全表示                   │
└─────────────────────────────────────────────────────────────┘
```

---

## データフロー原則

| 条件 | ダッシュボードに表示？ | プロジェクト一覧に表示？ |
|------|----------------------|------------------------|
| 自分が作った個人プロジェクト | ✅ | ✅ (基本タブ) |
| 自分が担当の会社プロジェクト | ✅ | ✅ (仕事タブ) |
| 自分に割り当てられたサブタスク | ✅ | プロジェクト内で表示 |
| 他社員が担当の会社プロジェクト | ❌ | ✅ (仕事タブ) |

---

## 機能要件

### 1. サブタスク期限継承
- サブタスク作成時、親プロジェクトの期限を自動継承
- 明示的に期限設定された場合はそれを優先

### 2. プロジェクト一覧URL
- `/projects/personal` - 個人プロジェクト
- `/projects/company` - 会社プロジェクト

### 3. プロジェクト作成ダイアログのコンテキスト適応

| 作成場所 | ダイアログ内容 |
|----------|---------------|
| 基本(Personal)タブ | シンプル（名前 + カラー + 期限） |
| 仕事タブ（製造業プラグインOFF） | 基本 + 顧客名（任意） |
| 仕事タブ（製造業プラグインON） | 基本 + 顧客名 + 現場名 + 目標粗利 |

### 4. プロジェクト一覧のタブ構成

#### 基本(Personal)タブ
- 会社に属していないプロジェクト
- `tenantId === null` かつ `isProject === true`

#### 仕事(Company)タブ
- 会社ごとにセクション分け
- 自分が担当でないプロジェクトも全表示
- 製造業プラグインONの会社は追加情報表示

### 5. 担当者割り当て
- 「手伝う」ボタンで自分を担当に追加可能
- オーナー/担当者は他社員にタスク割り当て可能

### 6. 量感チップ（将来機能）
- プロジェクト一覧で各プロジェクトの負荷を可視化
- 量感カレンダーと連動

## データモデル拡張

```typescript
interface Item {
  id: string;
  title: string;
  parentId: string | null;     // 親プロジェクトID
  ownerId: string;             // 作成者
  tenantId: string | null;     // 所属会社（nullなら個人）
  assigneeId: string | null;   // 担当者
  isProject: boolean;          // 子タスクがあればtrue
  due_date: number | null;
  status: ItemStatus;
  
  // 製造業プラグイン拡張（条件付き）
  customerId?: string;
  siteName?: string;
  targetMargin?: number;
  processStatus?: string;
}
```

## 製造業プラグインの影響

仕事タブで製造業プラグインONの会社セクションに表示される追加情報:

| 追加表示 | 説明 |
|----------|------|
| 進捗率バー | 工程ベースの進捗 |
| 目標粗利 vs 実績 | コスト管理 |
| 納品日カウントダウン | 残り日数強調表示 |
| 工程ステータス | 「図面作成中」「製造中」「納品待ち」など |
