## 1. domain フィールドの要否検討 (Phase 23)

### 現状
- `Item` 型には `tenantId` と `domain` (`'business' | 'private'`) が存在する。
- フィルタリングロジックでは「`tenantId` が空、かつ `domain` が `business` 以外」を個人タスクとしている。

### 議論
- **意見A**: `tenantId` が無いアイテムに関しては、システム的に「個人の管理下」にある。`domain` フィールドでさらに分類する必要があるか？
- **意見B**: 「会社に所属しているが、非公開（プライベート）にしたいタスク」という概念が必要な場合、`tenantId` があっても `domain === 'private'` という組み合わせがあり得る。
- **結論**: ユーザーの「`tenantId` があるかないかだけで十分ではないか」という意図を尊重しつつ、DBの互換性を考慮して `domain` を廃止するか、あるいは内部的なフラグとして簡略化する方向で進める。まずは `tenantId` の有無を主権とするロジックへ移行し、`domain` への依存を排除する。

## 2. 詳細画面 UI の高度化 (Phase 23)

### 2.1 選択中の「発光」エフェクト
- 納期入力またはマイ期限入力をフォーカスした際、`SideCalendarPanel`（カレンダー側）と対応する入力フィールドを `animate-pulse` や `shadow-glow` 等の CSS で強調する。

### 2.2 青枠（目安期間）の表示
- マイ期限（単日）だけでなく、`estimatedMinutes / 480` (work_days) で計算される「遡り期間」すべてを青い枠線で囲む。
- `RyokanCalendar` 内部の `CalendarCell` において、`isPrepPeriod` 等のフラグを計算して描画する。

### 2.3 警告表示ロジック
- **納期逆転警告**: `dueDate < prepDate` の場合、マイ期限のテキストを赤色 (`text-red-500`) にする。
- **キャパシティ不足警告**: 計算された期間（青枠）が表示されない（キャパシティが 0 の日に当たっている）場合、詳細画面に「稼働設定が未完了です」という警告を表示する。

## 3. ガントチャートヘッダーの再設計 (Phase 24)

### 構成案
- **位置**: 上部ヘッダー（2段目：プロジェクト/フィルタ検索）のすぐ下。
- **機能**:
    - **[ < ] [年月] [ > ]**: ボタン押下で1ヶ月分スクロール移動。
    - **[今月を表示]**: 当日を中央に配置。
    - **[日次設定]**: `DailyCapacityEditor` モーダルを開く。
- **実装上の課題**: 
    - `RyokanCalendar` (または `RyokanGridView`) のスクロール位置を外部から制御し、かつスクロール停止時に中央にある日付から年月を特定してヘッダーに通知する仕組みが必要。
    - `onScroll` イベントから Observer 的に現在の表示範囲を特定し、State に同期する。
