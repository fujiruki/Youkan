# システム企画・構想定義書: Judgment OS (Global Decision Board)

**フェーズ**: 3.1 企画・構想定義
**作成日**: 2026-01-11
**準拠**: AI_DEVELOPMENT_CONSTITUTION v4.2 / JBWOS思想

## 1. システム概要
**「時間管理」から「判断管理」へ。**
本システムは、不確実性の高い現場仕事に従事する建具店経営者が、複数のプロジェクトを横断して「今日やるべき判断・作業」を迷わず決定し、精神的な負荷なく一日を終えるための「判断OS (Judgment OS)」である。
従来のプロジェクト管理ツールではなく、ユーザーの**「判断リソース」を守るための防御壁**として機能する。

核心機能は **Global Decision Board**。
全プロジェクトのタスクを一箇所に集約し、今日やるべきこと（Ready）を最大2件に絞り込むことで、強制的に「判断の完了」を作り出す。

## 2. 解決したい課題（Pain Points）
*   **判断の無限ループ**: 複数案件が同時進行する中で、「次に何をすべきか」を毎回ゼロから比較検討し、脳のリソースが枯渇する（判断疲労）。
*   **「未完了」の重圧**: リストが終わらないことへの慢性的な罪悪感があり、それが原因でツールを開くこと自体を避けるようになる（回避行動）。
*   **時間軸の破綻**: 天候、他業者の進捗、急な呼び出しなどにより、事前に引いたスケジュール線表が即座に無意味化し、リスケジュール作業自体が負担になる。
*   **コンテキストスイッチのコスト**: プロジェクトAからプロジェクトBへ頭を切り替えるたびに発生する「再ロード」の疲労。

## 3. 想定ユーザー（Target Persona）
*   **属性**: 1〜5人規模の建具店経営者・職人。
*   **環境**: 現場作業、製作、見積もり、請求、顧客対応をすべて一人称で行う。
*   **性格**: 責任感が強く、「ちゃんとやりたい」と願っているがゆえに、理想と現実のギャップに苦しんでいる。
*   **課題**: 「管理」能力が低いのではなく、「変動」と「判断量」がキャパシティを超えている。

## 4. 提供価値（Value Proposition）
*   **「今日はこれで十分」の定義**: 物理的な完了ではなく、精神的な「停止線」をシステムが提示し、罪悪感なく休めるようにする。
*   **比較からの解放**: InboxのフィルタリングとReadyの制限により、「選ぶ」という行為自体を最小化する。
*   **外向きと内向きの分離**: 「顧客への説明（誠意ある適当さ）」と「自分の進捗（事実）」を分け、それぞれの精神的コストを下げる。
*   **プロジェクト横断の安心感**: 「あっちの案件、大丈夫だったっけ？」という不安を、Global View（全情報の一元化）によって解消する。

## 5. 非目的（やらないこと / Non-Goals）
*   **× 正確な予実管理**: いつ終わるかを分単位・時間単位で予測・記録しない。
*   **× AIによる自動最適化**: 「これを先にやるべき」とAIが勝手に順序を決めない。自分で決める感覚（自己効力感）を奪わない。
*   **× 生産性の最大化**: 詰め込めるだけ詰め込む効率化を目指さない。「壊れずに長く続くこと（持続可能性）」を最優先する。
*   **× 期限による脅迫**: 赤字での警告、カウントダウンなどの「焦らせる」UIを排除する。
