# 06. 変更履歴

## 凡例

- **[新規]**: 新しい仕様の追加
- **[変更]**: 既存仕様の変更
- **[修正]**: バグ修正に伴う仕様の明確化
- **[統合]**: 既存ドキュメントからの統合

---

## 2026-03-24: SdDD形式への仕様書統合

### 実施内容

SdDD（仕様書駆動開発）の構造を導入。既存の散在したドキュメント群からYoukanの最新仕様を抽出し、以下のファイルに統合した。

| ファイル | 統合元 |
|:--|:--|
| `SPEC.md` | 新規作成（目次・概要） |
| `spec/01_概要.md` | 00_Vision/User_Voices/20260202_Youkanの概要と用語.md, 00_Vision/System_Grand_Design.md, 00_Vision/SYSTEM_PHILOSOPHY_AND_VISION.md |
| `spec/02_機能仕様.md` | spec/00_MASTER_SPEC.md, spec/03_MORNING_PLANNING.md, 00_Vision/Capacity_Management_Spec.md, 00_Vision/User_Voices/* |
| `spec/03_画面設計.md` | spec/02_VIEW_DEFINITIONS.md, spec/00_MASTER_SPEC.md |
| `spec/04_データ設計.md` | spec/01_STATE_MATRIX.md, backend/db.php, backend/ItemController.php, backend/TodayController.php |
| `spec/05_技術設計.md` | package.json, バックエンドコード構成, 00_Vision/System_Grand_Design.md |

### 注意事項

- 旧ファイル（spec/00_MASTER_SPEC.md, 01_STATE_MATRIX.md, 02_VIEW_DEFINITIONS.md, 03_MORNING_PLANNING.md）は参照用として残置
- `docs/SPEC/`（大文字）は`docs/spec/`（小文字）と同内容。SdDDでは`docs/spec/`を正とする
- 旧名称JBWOS時代のドキュメントが`docs/`配下に多数存在するが、新spec/*.mdに統合済みの内容が正
- 既存の`docs/requests.md`の内容は`request_log.md`にも転記した
