## 2026-07-06: 量感カレンダー分母ルールの確定

### 決定事項
| 種別 | 内容 | 備考 |
|:--|:--|:--|
| [変更] | 個人アカウントの「全て」は本人の業務キャパシティ合計を分母にする | 所属会社ごとのキャパシティは足し上げない |
| [変更] | 個人/会社/個別会社フィルタごとに分母を切り替える | 個人=総量-会社配分、会社=会社配分合計、個別会社=該当会社配分 |
| [追加] | 会社/チームスコープの分母を主力メンバーのキャパシティ合計と定義 | 現行実装は `memberships.is_core=1` の `daily_capacity_minutes` 合計 |
| [明確化] | 会社配分未設定の会社は 0 分として扱う | 総キャパシティへフォールバックしない |
| [追加] | メンバー別 `capacity_profile` を `memberships` に保存 | 会社/チーム分母に曜日別・日別例外・会社別配分を反映 |

### 影響範囲
- F-06 量感カレンダーのヒートマップ背景色と CapacityBar
- 詳細画面の量感カレンダーは同じ `QuantityEngine` を使うため同じ分母ルールに従う
- `/members` と `/tenant/members` は `capacityProfile` を返す。`/tenant/members/{id}` は `capacity_profile` を保存する
- 会社/チーム分母は、core member の `capacityProfile` を優先し、未設定時は `dailyCapacityMinutes` を使う

## 2026-05-05: 階層ラッパー型分離・virtual-header- 撤廃（R-030）

### 決定事項
| 種別 | 内容 | 備考 |
|:--|:--|:--|
| [変更] | HierarchicalWrapper を discriminated union 化 | item/header の型分離 |
| [削除] | virtual-header- プレフィックス生成（hierarchy.ts 2箇所） | 偽装ID廃止 |
| [削除] | .replace('virtual-header-', '') 散在コード（12箇所） | 不要化 |
| [削除] | OverviewBoard.tsx のローカル stripId 関数 | 不要化 |
| [修正] | OverviewItem の onNavigateToFlow バグが自動解消 | 型分離の副次効果 |
| [追加] | hierarchy.types.test.ts | 新型のTDDテスト |

### 影響範囲
- マイグレーション不要
- 既存ユーザー操作: 改善のみ（フロー画面のコア機能回復）
- 過去 URL ブックマーク（virtual-header-付き）は今後生成されないため、既存のものは破損リスクなし

### 設計判断
- TDD・MVVM・リアクティブ・メンテナンス性を最優先
- TypeScript の型システムで「ヘッダーから item.id を取り出す」誤りをコンパイル時に検出
- 関連する別R番号タスクで検討した「共通ヘルパー idUtils.ts」「多層防御」は型分離により不要となり全廃

### メンテナンス則
- 階層ラッパー型は discriminated union（type プロパティで分岐）
- ヘッダー型は item: Item を持たない
- React key には wrapper.id を使う
- item.id 加工パターン禁止
