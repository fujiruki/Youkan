# 03. 画面設計

## 0. アプリ全体

- **ブラウザタブタイトル**: `Youkan`（旧称JBWOSからの変更）

---

## 1. View Mode一覧

`DashboardScreen`は`viewMode`により以下の4つのモードを切り替える。
単一のデータソースに対し、目的に応じた異なるレンズ（View）を提供する。

| モード名 | URL | 役割 | 英語名 |
|:--|:--|:--|:--|
| **登録と集中** | `/stream` | コックピット。実行(Today)と判断(Inbox)に特化 | Stream |
| **状況把握** | `/panorama` | 俯瞰。プロジェクト構造と時系列(Gantt)を確認 | Panorama |
| **全体一覧** | `/overview` | カタログ。新聞記事風にアイテムの塊を俯瞰 | Overview |
| **カレンダー** | `/calendar` | 資源管理。量感カレンダーでキャパシティ確認 | Calendar |

## 2. Stream View（登録と集中）

システムのデフォルトビュー。「今日という一日を、無理なく過ごす場所」。

### 2.1 レイアウト: 厳格な3ゾーン構成

#### Zone 1: Commit（今日の判断）
- **最大2件**のキュー
- `focus`ステータスのアイテムが表示される

#### Zone 2: Execution（今日の実行）
- **1つだけ**表示。選ばせない
- UIはシンプルなカード（例:「○○建具 | 製作中」）
- 操作: 中断(Pause)、完了(Done)のみ
- 「開始」ボタンなし（表示 = 実行中）

#### Zone 3: Life（生活）
- 画面最下部に常駐
- シンプルなチェックリスト（掃除、洗濯など）
- Executionの条件でも制約でもない。「今日の現実」を肯定的に記録するために存在

### 2.2 フィルタリング（Strict Filtering）

Inboxとfocusの完全分離:
- **Inbox List（左）**: `status: inbox`のみ。`focus`は**絶対に除外**
- **Focus Area（中央）**: `status: focus`のみ

#### プロジェクトアイテムの非表示（R-029）

- `is_project=1` のアイテムは Stream モード（Inbox / Pending / Waiting の各リスト）に**表示しない**
- プロジェクトは「見出し・構造」であり、Stream での個別作業対象とならない
- プロジェクト構造の確認・操作はパノラマ（Panorama View）・全体一覧（Overview View）で行う

### 2.3 Morning Planning（ドラッグ&ドロップ並べ替え）

- Next Strategyセクションの各`SmartItemRow`をドラッグ&ドロップで並べ替え可能
- FocusCardは常に先頭（index: 0）に固定表示
- 並べ替え完了時に自動保存
- `@dnd-kit/core` + `@dnd-kit/sortable`使用

### 2.4 主要コンポーネント

| コンポーネント | 役割 |
|:--|:--|
| `SmartItemRow` | リストアイテム表示 |
| `FocusCard` | 中央の大きなカード。DnD対応 |
| `QuickInputWidget` | **新規登録欄**。Stream/Newspaper 両画面で共通使用 |
| `SortableFocusQueue` | focusタスクのドラッグ&ドロップ並べ替えUI |
| `ViewContextBar` | フィルタ/モード表示バー |

### 2.5 新規登録欄（QuickInputWidget）のショートカット

| ショートカット | 動作 |
|:--|:--|
| `Enter` | inbox ステータスで登録 |
| `Shift+Enter` | **今日やる**（focus ステータス）で登録 |
| `Alt+Enter`（テキストあり） | inbox で登録し、すぐに詳細モーダルを開く |
| `Alt+Enter`（空欄） | 直前に登録したアイテムの詳細モーダルを開く |
| `Alt+D` | 直前に登録したアイテムの詳細モーダルを開く（後方互換） |

## 3. Panorama View（状況把握）

「いつ、どのプロジェクトが動いているか」を可視化する。

### 3.1 レイアウト: 2ペイン構成
- **左**: プロジェクト・タスクツリー
- **右**: タイムライン（Gantt）

### 3.2 実装
- コンポーネント: `PanoramaBoard`
- 大量のタスクを描画するため仮想スクロール使用

### 3.2.1 バケット構成（R-028）

パノラマの状況把握ボードは6バケット構成:

| バケット | 対応ステータス | 備考 |
|:--|:--|:--|
| Inbox | `inbox` | 未判断の投入直後 |
| Active | `focus` | 今日やる・集中対象 |
| Waiting | `waiting` | 外的要因待ち |
| Pending | `pending` | 保留（外的要因待ち）⏸ amber系 |
| 💭 Someday | `someday` | いつかやる（自発的延期） purple系 |
| Log | `done` | 完了・ログ |

- **配置順**: Inbox / Active / Waiting / Pending / 💭 Someday / Log
- Someday バケットは Pending と Log の間に配置

### 3.3 「今月を表示」ボタンの挙動
- **横方向**: タイムライン（Gantt）を水平スクロールし、今月が表示領域の中央に来るようにする
- **縦方向**: 今月が目安期間（due_date〜prep_dateの範囲）に含まれるアイテム群が表示領域の中央に来るように垂直スクロールする
- 両方向のスクロールを同時に実行し、今月に関連するアイテムが画面中央に集まるようにする

## 4. Overview View（全体一覧）

新聞記事のようにプロジェクトごとのタスクの「塊」を並べて全体像を把握する。
Ganttほど厳密ではなく、Streamほど近視眼的ではない中間ビュー。

### 4.0 初期フィルタ
- 全体一覧（Newspaper View）を開いたとき、フィルタは常に**「全て」（統合モード）**で表示する
- 他のViewからの切替時もフィルタ状態を引き継がず、「全て」にリセットする

### 4.0.1 Someday フィルタースイッチ（R-028）
- フィルターに「**Somedayを含む**」スイッチを追加
- **デフォルト: Someday を除外**（通常の作業表示に雑音を入れない）
- スイッチ ON → someday 状態のアイテムも一覧に表示される
- 目的: 「夢リスト」を明示的に見たいときだけ呼び出せる

### 4.1 レイアウト: Masonry
- CSS `column-count`によるマルチカラムレイアウト
- `break-inside: avoid`でカラムまたぎの分割を防止

### 4.2 目安時間の表示・インライン編集

- 各アイテムの納期（日付表示）の左側に**目安時間**を表示
- 表示形式: `30m`, `1.5h`, `3h` など簡潔な形式
- `estimated_minutes`が0または未設定のアイテムは非表示（スペース節約）
- **インライン編集**: 目安時間表示部分をクリック → インプットフィールドに切替
  - 簡易入力対応: `1h`, `30m`, `1.5h`, `90`（分として解釈）
  - Enter確定 / Escape キャンセル / フォーカスアウトで確定
  - API経由で即保存（`PATCH /items/{id}` の `estimated_minutes`）

### 4.3 プロジェクトヘッダーからフローへの導線

- プロジェクトヘッダーにフローチャートアイコン（GitBranch等）を表示
- クリック → そのプロジェクトのフローチャート画面に遷移
- 導線: `currentView='flows'` + `activeProjectId=プロジェクトID`

### 4.4 クリックハンドリング
- Masonryレイアウトでの`onClick`不安定問題への対策
- アイテムへのクリックは`onMouseUp`でハンドリング
- `user-select: none`を併用

## 5. Calendar View（カレンダー）

「いつが忙しいか」「どこに空きがあるか」を直感的に把握する。

### 5.0 Someday アイテムの非表示（R-028）
- `someday` 状態のアイテムはガントビュー・カレンダービュー**すべてで表示しない**
- 理由: 期限がないものは時間軸（カレンダー・ガント）に置けない
- 実装: フィルターアウト（表示オプションなし）

### 5.1 旅館カレンダー（Ryokan Style）
- 通常の時間割ではなく、旅館の予約表のように「リソース」と「日付」のマトリクスで表現
- 日付セルの背景色でタスク密度を表現（ヒートマップ）

### 5.2 サブビュー
- グリッドビュー（`RyokanGridView`）
- タイムラインビュー（`RyokanTimelineView`）
- ガントビュー（`RyokanGanttView`）

### 5.3 ガントビューの表示モード

ガントビューは「プロジェクト別」と「一覧」の2モードを切替可能。

### 5.4 ガントビューのインタラクション

#### 目安時間インライン編集
- 左のタイトル列の右端に目安時間バッジ（`30m`, `1h` 形式）を表示
- クリックで入力欄に切替。`1h`, `30m`, `1.5h`, `90`（分）などの形式を受付
- Enter/blur で確定、Escape でキャンセル

#### マイ期限（prep_date）のクリック設定
- マイ期限が未設定のアイテムの日付セルをクリックすると、クリックした日をマイ期限として設定
- 設定後、青い割当バーが表示される

#### 右クリックメニュー
- タイトル列・青い割当バーを右クリックすると DashboardScreen と同じコンテキストメニューを表示
- メニュー表示中に `Del` キーでアイテムを完全削除

#### プロジェクト別（showGroups=true）
- プロジェクトをヘッダー行として表示し、配下のタスクをインデント表示
- プロジェクト自体が見出しの役割を持つ

#### 一覧（showGroups=false）
- **プロジェクト化されたアイテム（is_project=true）は非表示**。リーフタスクのみ表示
- 全アイテムがフラットリスト（インデントなし）
- 各アイテムのタイトル末尾に所属プロジェクト名を `[プロジェクト名]` で控えめに表示（`item.projectTitle` を使用）
- プロジェクト未所属のアイテムにはプロジェクト名を表示しない
- **ソート順**: 納期/マイ期限いずれも未設定のアイテムが先頭 → 納期・マイ期限のうち設定されている方（両方あれば早い方）の昇順

#### 完了の表示切替トグル（R-036）
- ガントビューヘッダー（`CalendarHeader`）に「完了を表示」スイッチを配置
- on: 完了済（`status=done`）アイテムを表示
- off: 完了済アイテムを非表示（フィルターアウト）
- デフォルト: on
- 状態スコープ: セッション単位（ブラウザを閉じたらデフォルトに戻る。`FilterContext` の `filterMode` と同じ思想）
- バックエンドAPI への影響なし、フロント側のフィルタとして実装

#### Google カレンダー予定の表示（R-039 Phase 3 UX）
- ガントビューのヘッダー（各日付列）に Google 予定の `📅` チップを最大 2 件まで表示
- 既存タスクバー（青い割当バー）と干渉しないよう、ヘッダー領域内（日付番号の下、CapacityBar の下）にスタックする
- 1 日 3 件以上の予定がある場合は末尾に「他 X 件」リンクを表示し、タップで `MobileBottomSheet` の全件展開シートを開く
- 予定チップタップで `EventDetailModal` を開く（既存 Phase 2 と同じコンポーネント）
- 設定画面の「表示するビュー」で `gantt` が OFF の場合は描画しない（取得自体を行わない）

## 5.6 量感セルの進捗棒グラフ（R-034 Phase 1, R-035）

各日付セルの下端に進捗バーを描画する。

| 項目 | 値 |
|:--|:--|
| 高さ | 4px |
| 位置 | セル下端（`absolute bottom-0`） |
| 未完了部分の色 | `emerald-500`（落ち着いた緑系） |
| 完了部分の色 | `emerald-200`（同系統の淡色） |
| 100% 超 | `red-500` で全体切替 |
| 数値表示 | なし |
| ホバーツールチップ | なし |
| 対象 | グリッドビュー（必須）、ガントビュー一覧表示（任意） |

完了アイテム所要時間も量感の分子に含めて「進捗」を表現。既存背景色グラデーション（旅館カレンダーのヒートマップ）は維持し、棒グラフは併存する。

実装:
- 新規コンポーネント `CapacityBar`（`src/features/core/youkan/components/Calendar/CapacityBar.tsx`）
- Props: `{ totalMinutes, completedMinutes, capacityMinutes }`
- React.memo で再レンダリング最小化

### 5.6.5 グリッドビューの Google カレンダー予定表示（R-034 Phase 2）

グリッドビュー（カレンダー View の月表示）の各日付セル内に、Google カレンダーのイベントをテキストで表示する。Phase 2 で導入する新機能であり、進捗棒グラフ（§5.6）と併存する。

**表示仕様:**

| 項目 | 値 |
|:--|:--|
| 通常イベント | `📅 HH:MM タイトル`（例: `📅 10:00 会議`） |
| 終日イベント | `📅 タイトル`（時刻なし） |
| 1 日のイベント数上限 | 先頭 3 件（推奨。設定で 1〜5 件に変更可） |
| 超過時 | 末尾に「他 X 件」リンクを表示 |
| フォントサイズ | `text-xs`（タスクカードと同等以下） |
| 色 | `text-slate-700`（タスクと区別しつつ可読性確保） |
| 配置 | セル内、タスクカード一覧の下（または上、要 UI 実機検証） |

**操作:**

| アクション | 挙動 |
|:--|:--|
| イベント行タップ | イベント詳細モーダルを開く |
| 「他 X 件」タップ | その日の全イベント一覧（ボトムシート or ポップオーバー、UX は実装時に決定） |

**イベント詳細モーダルの内容:**
- タイトル（DB に保存された `title`）
- 開始〜終了時刻（終日の場合は「終日」表示）
- 場所（`location` があれば）
- 「Google カレンダーで開く」リンク（イベントの HTML URL に遷移）
- 編集 / 削除は不可（Youkan 側からは読み取り専用）

**完了アイテム表示統一（§5.7 / R-035）の対象外:**
Google カレンダーのイベントは Youkan のタスクではないため、`status=done` スタイル（`slate-400` + 取り消し線）は適用しない。常に通常表示。

**量感計算との関係:**
- 表示と並行して、イベントの所要時間（`end - start`、終日は 240 分）を CapacityBar の分子に加算する
- イベント名表示と量感加算はセットで動作する（片方だけ ON/OFF 不可）

**実装メモ:**
- 新規コンポーネント `GridCellExternalEvents`（`src/features/core/youkan/components/Calendar/GridCellExternalEvents.tsx` 想定）
- Props: `{ events: ExternalEvent[]; maxVisible?: number; onEventTap: (e: ExternalEvent) => void; onMoreTap: () => void }`
- イベント詳細モーダル: `ExternalEventDetailModal`（既存 `DecisionDetailModal` と同階層）
- データ取得: `GET /api/calendar/grid` の `external_events` フィールドから受け取る

### 5.6.6 ガント／タイムラインビューの Google カレンダー予定表示（R-039 Phase 3 UX）

Phase 2 でグリッドビューに導入した Google 予定表示を、ガント／タイムラインビューにも展開する。

**表示対象ビューの設定:**

| 項目 | 値 |
|:--|:--|
| 設定 UI | 設定画面 §12.1「表示するビュー」セクション |
| 種類 | グリッド／ガント／タイムラインの 3 チェックボックス |
| デフォルト | 全 ON（`['grid','gantt','timeline']`） |
| 保存先 | `localStorage` キー `ykn_external_events_views`（JSON 配列） |
| 取得制御 | 設定で OFF のビューでは `useExternalEvents` 内部で fetch をスキップし空 Map を返す |

**ガントビューでの表示位置:**
- 日付ヘッダー（`RyokanGanttView` の `Date Headers`）内、`CapacityBar` の下に `📅` チップを縦積み配置
- ヘッダー列の幅（24px）に合わせ、フォントサイズ `text-[8px]`、最大 2 件まで表示、それ以上は「他 X 件」リンク
- タスクバー領域には影響しない（描画は日付ヘッダー領域のみ）
- `externalEventsByDate` の取得 key 形式は `YYYY-MM-DD`（`useExternalEvents` の出力形式と一致。ガント内のローカル `toYmdKey` で生成し、`data-gantt-date` 属性などで使われる `normalizeDateKey`（`toDateString` 形式）とは別 key とする）（R-040）

**タイムラインビューでの表示位置:**
- 既存の `CalendarCell` を流用するため、グリッドビューと同じ「セル下部に最大 3 件」表示
- 設定が ON のときは `RyokanCalendar` から `externalEventsByDate` を伝搬

**操作:**

| アクション | 挙動 |
|:--|:--|
| チップタップ | `EventDetailModal`（既存 Phase 2 と同一）を開く |
| 「他 X 件」タップ | `MobileBottomSheet` の全件展開シートを開く（PC/スマホ共通） |

**実装メモ:**
- `useExternalEvents(from, to, viewMode)` のシグネチャに `viewMode` を追加し、設定リストに含まれないビューでは即時空 Map を返す
- `VolumeCalendarScreen` 側のグリッド限定ガードは撤廃し、hook 内部で判定する
- 既存 `ExternalEventChip` / `EventDetailModal` を流用（新規コンポーネントは作らない）

#### R-042-Y1 ロード戦略（月単位キャッシュ＋ ±6 ヶ月初期取得）

- 初期取得範囲を `currentDate ± 6 ヶ月`（計 13 ヶ月）に拡大し、開いた時点で前後半年分の予定がすべてのカレンダーサブビューに即出る
- `useExternalEvents` のキャッシュ key を月単位 `YYYY-MM` に変更し、`loadedRange` / `loadMore(direction, months)` / `isLoadingMore` を新たに公開
- `loadMore` は未取得月のみを連続グループ化して 1 リクエストで fetch するため、過剰な API 呼び出しを防ぐ
- R-042-Y2 で IntersectionObserver の sentinel をグリッド／ガント／タイムラインに配置し、ロード境界 200px 手前で `loadMore` を発火する

#### R-042-Y2 IntersectionObserver sentinel 配置（lazy load 発火）

`useExternalEvents.loadMore(direction, months)` をスクロール端で自動発火するため、各カレンダービューに sentinel 要素を仕込む。共通フック `useLazyLoadSentinel` を新設し、`rootMargin` デフォルト `'200px'`（議事録 2026-06-04 §4 採用案）で IntersectionObserver を構築する。

| ビュー | スクロール軸 | sentinel 配置 | 発火内容 |
|:--|:--|:--|:--|
| グリッド | 縦 | スクロールコンテナ先頭・末尾に高さ 1px の sentinel | 先頭交差 → `loadMore('before', 3)` / 末尾交差 → `loadMore('after', 3)` |
| ガント | 横 | body の左端・右端に幅 1px の sentinel（`absolute` 配置で日付列をまたぐ） | 左端交差 → `loadMore('before', 3)` / 右端交差 → `loadMore('after', 3)` |
| タイムライン | 縦／横 | scrollRef 内部の先頭・末尾。`isMini`（縦並び）で高さ 1px、通常（横並び）で幅 1px | 同上 |

二重発火防止:

- `isLoadingMore=true` の間は `useLazyLoadSentinel` の `enabled=false` を渡し、observer 自体を作らない（既存の observer は disconnect される）
- 既キャッシュ範囲に対する `loadMore` 呼び出しは `useExternalEvents` 内部で skip 済（Y1 で対応）

経路:

- `VolumeCalendarScreen` が `useExternalEvents` から `loadMore` / `loadedRange` / `isLoadingMore` を取り出して `RyokanCalendar` に渡し、さらに各サブビューへ伝搬する。
- 各 View の props 型に `onLoadMore?: (dir: 'before' | 'after', months: number) => void` / `isLoadingMore?: boolean` を追加（オプショナル）。

注: ロード中のスケルトン UI は R-042-Y3 で別実装する。Y2 では視覚表現は触らない。

### 5.6.8 詳細モーダルのタイトル編集欄常時表示（R-037）

`DecisionDetailModal` のタイトル編集欄は、タイトルが空文字・空白文字のみ・通常文字列のいずれであっても、常に編集可能な `input` 要素として表示する。

背景:
- 旧実装は「タイトル表示用 `h2` ↔ 編集用 `input`」を `isEditingTitle` フラグで切り替えていた
- タイトルを 0 文字保存すると `h2` 表示部分自体が消え（中身が空のため空白）、再度クリックする領域も失われて編集不能になるバグが発生した

仕様:

| 項目 | 値 |
|:--|:--|
| 描画要素 | 常に `input[type="text"]` |
| プレースホルダ | `タイトル未入力`（空文字・空白のみ時の視覚ガイド） |
| 空文字保存 | 許容（`editedTitle !== item.title` であれば `onUpdate` を必ず呼ぶ） |
| 確定操作 | `onBlur` または `Enter` キー（Enter は blur をトリガー） |
| `isEditingTitle` フラグ | 廃止（モード切替自体を撤廃） |

目的: 0 文字保存後でも編集欄が消えず、ユーザーが常にタイトルを書き直せる状態を保つ。判断疲れを増やすモード切替を撤廃する。

### 5.6.7 グリッドビュー ヘッダー追従＆「今月を表示」ボタン対応（R-038）

グリッドビュー（月マス表示）でもガントビューと同様に、`CalendarHeader` の年月表示と「今月を表示」ボタンが機能する。`CalendarHeader` 自体は §5.4 の「完了の表示切替トグル」と同じく `variant: 'gantt' | 'grid'` で機能差を切り替えるが、年月表示の追従と「今月を表示」ボタンは両 variant 共通の振る舞いとして提供する。

- **ヘッダー表示**: `VolumeCalendarScreen` は `viewMode === 'gantt' || viewMode === 'grid'` のときに `CalendarHeader` を表示する（gantt: 全機能 / grid: 月切替＋今月＋日次設定のみ）。
- **縦スクロール時の追従**: `RyokanCalendar.detectVisibleMonth` がコンテナ縦中央に最も近い `data-date` 付きセルを特定し、その日付から月を算出して `onVisibleMonthChange(Date)` を発火する。`data-date` 属性は `normalizeDateKey()`（=`Date.toDateString()`）形式なので、月キーは文字列の前方一致ではなく `new Date()` でパースして取り出す（Invalid Date 防止）。
- **「今月を表示」ボタン**: `scrollToCurrentMonthCenter` のグリッド/タイムライン分岐は、今月 1 日の `data-date` を持つセルを `scrollIntoView({behavior:'smooth', block:'center'})` で中央配置する。1 日が DOM 上に存在しなければ今日のセルへフォールバックし、いずれも無ければ従来の `scrollToDateElement(now)` で代替する。`scrollIntoView` 直後に `detectVisibleMonth` を 1 度走らせ、ヘッダーの年月表示を即時同期する。
- **ガントビューの挙動は変更しない**（既存の横スクロール中心配置とアイテム群中央配置をそのまま維持）。

## 7. Flow View（フロー）

タスク間の依存関係（順序）をフローチャート風に可視化・編集する5つ目のView。
`@xyflow/react` を使用。

### 7.1 コア概念

- アイテム同士に**順序（依存関係）**を持たせる。「AのあとB」
- `parent_id`（階層）、`project_id`（所属）とは**完全に別概念**
- 依存関係は分岐・合流が可能（A→B, A→Cや、B→D, C→D）
- 循環参照（A→B→A）は禁止

### 7.2 レイアウト

- **時間軸**: 上から下へ流れる
- **ノード**: アイテムタイトルが表示されたカード
- **エッジ**: ノード間のリンク線（依存関係）。ドラッグ時に表示
- **プロジェクトグループ**: 同じプロジェクトに属するノードは薄い背景色・大枠で囲む
- **グループ間順序**: グループ同士も順序付け可能
- **独立ノード**: 順序未設定のアイテムも独立して表示可能

### 7.3 未配置アイテムリスト

- 画面右上に最前面固定表示
- フローキャンバスに未配置のアイテム一覧
- ここからドラッグ&ドロップでキャンバスに配置可能

### 7.4 操作

**マウス操作:**
- アイテムAの上にBを重ねる → 「AのあとB」のリンク構造作成（位置は移動しない）
- ノードのドラッグで位置調整
- エッジのドラッグで接続先変更
- ＋ボタンで後続ノード追加

**キーボードショートカット:**

| ショートカット | 動作 |
|:--|:--|
| `Enter` | 選択ノードの下に新規ノード追加（テキスト入力即開始） |
| `Tab` | 選択ノードから分岐して新規ノード追加（右に枝分かれ） |
| `Delete` / `Backspace` | 右クリックメニュー表示中→そのアイテムを完全削除。未表示時→選択ノード/エッジを削除 |
| `F2` | 選択ノードのタイトル編集 |
| `Escape` | 選択解除 / 編集キャンセル |
| `Ctrl+L` | 選択した2ノード間にリンク作成 |
| `Ctrl+Shift+L` | 選択ノードのリンクを解除 |
| `↑↓←→` | ノード間をフォーカス移動（依存関係に沿って） |
| `Ctrl+A` | 全ノード選択 |
| `Space` | 選択ノードの詳細モーダルを開く |
| `Home` | フロー全体がビューポートに収まるようにズームフィット |
| `Ctrl++` / `Ctrl+-` | ズームイン/アウト |
| `Ctrl+0` | ズームリセット（100%） |
| `Ctrl+I` | 未配置リストのQuick Inputにフォーカス |
| `Alt+D` | 直前に操作したノードの詳細を開く |

### 7.5 ガントチャート連動

- ガントチャートの日程移動が依存関係を飛び越えられない制約
- 前提タスクの完了予定日より前に後続タスクを配置できない

### 7.6 プロジェクト単位表示

Flow Viewはプロジェクト単位で利用することを主とする。

#### 7.6.1 プロジェクト選択画面（初期画面）

Flow Viewを開くとまずプロジェクト選択画面が表示される。

- **カード一覧**: プロジェクトごとにカードを表示
  - 表示項目: プロジェクト名 / タスク数 / 完了率 / 直近期限

## 12. 設定画面

ユーザー個別の設定を集約する画面。ヘッダー右側の歯車アイコン（Settings）から開く。Phase 2 時点では Google カレンダー連携セクションを中心に置く（他項目は既存のまま）。

### 12.1 Google カレンダー連携（R-034 Phase 2）

Google カレンダーの OAuth 認証起点。設定画面内の独立セクションとして配置する。

**未連携時の表示:**

```
┌──────────────────────────────────────────┐
│ Google カレンダー連携                     │
├──────────────────────────────────────────┤
│ Google アカウントの primary カレンダーを │

│ 予定が表示され、量感計算に反映されます。 │
│                                          │
│ [ G Google でログイン ]                  │
│                                          │
│ ※ unverified app の警告が出ますが、       │
│   「詳細」→「安全でないページに移動」    │

└──────────────────────────────────────────┘
```

「Google でログイン」ボタン押下で `POST /api/google/oauth/start` を叩き、返ってきた OAuth URL に遷移する（同一タブでも別タブでも可、UX は実装時に決定）。

**連携済み時の表示:**

```
┌──────────────────────────────────────────┐
│ Google カレンダー連携                     │
├──────────────────────────────────────────┤
│ 連携アカウント: door.fujita@gmail.com    │
│ 最終同期: 2026-06-02 14:23               │
│                                          │
│ [ 🔄 今すぐ更新 ]  [ 🔓 連携解除 ]       │
│                                          │
│ 共有カレンダー（Phase 3 で振り分け）:    │
│ ┌────────────────────────────────────┐  │
│ │ 家族カレンダー    [個][会][無視]  │  │
│ │ 仕事用カレンダー  [個][会][無視]  │  │
│ └────────────────────────────────────┘  │
│ ※ Phase 2 では UI のみ。ラジオの保存は    │
│   Phase 3 で実装予定。                    │
└──────────────────────────────────────────┘
```

**ボタン挙動:**

| ボタン | 挙動 |
|:--|:--|
| 今すぐ更新 | `POST /api/google/calendar/refresh` を呼ぶ。60 秒クールダウン中は disabled（残り秒数を表示） |
| 連携解除 | 確認ダイアログ → `DELETE /api/google/oauth`。トークン＋キャッシュをサーバ側で削除 |
| 個 / 会 / 無視（ラジオ） | Phase 2 では UI のみ（disabled 表示でも可）。Phase 3 で `user_calendar_assignment` テーブルに保存予定 |

**OAuth コールバック:**
- `GET /api/google/oauth/callback?code=...` をリダイレクト URI に設定
- 成功時は設定画面（連携済み表示）に戻す
- 失敗時はエラーメッセージをトースト表示し、未連携表示に戻る

**エラー表示:**
- リフレッシュトークン失効: 「Google 側で連携が解除されました。再連携してください」と表示し、未連携表示に切り替え
- API レート超過: トースト「Google API のレート制限です。しばらく待ってください」

**実装メモ:**
- 統合先: 既存 `PersonalSettingsScreen.tsx`（`src/features/core/youkan/screens/PersonalSettingsScreen.tsx`）の中に独立セクションとして追加。新規 `SettingsPage` は作らない（個人設定画面が既に存在するため）
- 新規セクション `GoogleCalendarSection`（`src/features/core/youkan/components/Settings/GoogleCalendarSection.tsx`）
- API ラッパー: 新規 `src/api/googleCalendar.ts`（`GoogleCalendarApi.startOAuth / getStatus / refresh / revoke`）
- 既存 `hooks/useGoogleCalendar.ts`（Phase 1 で実装された「Google カレンダーに予定追加 URL を開く」ユーティリティ）は別用途のためそのまま残す。Phase 2 の連携状態は `GoogleCalendarSection` 内部で `useState` 管理（共有需要が出てきたら後でフック切り出し可）
- OAuth コールバック: バックエンドは設定画面に `?google_oauth=success` / `?google_oauth=error` 付きでリダイレクト。フロント側で検知してトースト表示後、`history.replaceState` で URL クエリを除去

**「表示するビュー」セクション（R-039 Phase 3 UX）:**

Google 予定をどのカレンダーサブビューに表示するかを切り替える。連携済み時のみ表示する。

| 項目 | 値 |
|:--|:--|
| UI | グリッド／ガント／タイムラインの 3 チェックボックス |
| デフォルト | 全 ON（`['grid','gantt','timeline']`） |
| 保存先 | `localStorage` キー `ykn_external_events_views`（JSON 配列） |
| 変更タイミング | チェックボックス操作の即時保存 |
| 効果 | OFF にしたビューでは `useExternalEvents` が fetch せず空 Map を返す。量感計算にも反映されない |

```

│ 表示するビュー                           │
├──────────────────────────────────────────┤
│ [✓] グリッド   [✓] ガント   [✓] タイムライン │
│ ※ チェックを外したビューでは Google 予定は │
│   表示されず、量感計算にも反映されません。 │
└──────────────────────────────────────────┘
```
