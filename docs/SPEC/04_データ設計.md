## 4. アイテム状態定義（State Matrix）

### 4.1 基本ステータス（JudgmentStatus）

「進捗」ではなく**「判断（今考えるか・否か）」**で分類する。

| ステータス | 日本語 | 意味 | 表示場所 |
|:--|:--|:--|:--|
| `inbox` | 受信箱 | 未判断。判断待ち | Inbox（左カラム） |
| `focus` | 今日やる | 「今日意識と時間を向ける」と判断した状態 | Stream（中央カラム） |
| `waiting` | 待機中 | 他者待ち、または開始日待ち | Waiting（左カラム） |
| `pending` | 保留 | 「今はやらない」と意図的に判断した状態 | Pending（左カラム） |
| `done` | 完了 | 作業が終了した状態 | Log / Archive |

### 4.2 フラグとの組み合わせ

#### Inbox フェーズ

| ステータス | 組み合わせ | 意味 |
|:--|:--|:--|
| `inbox` | 標準 | 純粋な未処理アイテム |
| `inbox` | `interrupt: true` | 割り込み/緊急。Inbox最上部に強調表示 |

#### Focus フェーズ

| ステータス | 組み合わせ | 意味 |
|:--|:--|:--|
| `focus` | 標準 | 今日のタスクキュー |
| `focus` | `isEngaged: true` | 現在実行中（Active Focus）。FocusCardとして中央で最大化 |
| `focus` | `is_executing: true` | 実行中フラグ（Backend永続化）。isEngagedと同義 |
| `focus` | `isProject: true` | プロジェクト全体のFocus |

#### Waiting フェーズ

| ステータス | 組み合わせ | 意味 |
|:--|:--|:--|
| `waiting` | 標準 | 他者待ち（Delegated） |
| `waiting` | `prep_date`が未来 | 開始日待ち（Scheduled）。カレンダーに表示 |
| `waiting` | `prep_date`が過去/今日 | 開始可能（Ready）。Inbox/Focusへ移動候補 |

#### Pending フェーズ

| ステータス | 組み合わせ | 意味 |
|:--|:--|:--|
| `pending` | 標準 | いつかやる（Someday） |
| `pending` | `due_date`が未来 | 期限付き保留 |

### 4.3 特殊フラグ

| フラグ | 値 | 意味 |
|:--|:--|:--|
| `isProject` | true | プロジェクト化。子タスクを持てる |
| `projectId` | ID | 特定プロジェクトの子タスク |
| `tenantId` | ID | 特定の会社に所属 |
| `domain` | private/business | 公私区分 |

### 4.4 ライフサイクル（状態遷移）

```
inbox → focus      （今日やる）
inbox → waiting    （人に頼む / 開始日待ち）
inbox → pending    （今はやらない）
inbox → done       （もう終わった）

focus → done       （完了）
focus → inbox      （やっぱり保留解除）
focus → waiting    （待ちに変更）
focus → pending    （やっぱり保留）

waiting → inbox    （待ち解除）
waiting → focus    （待ち解除して即実行）
waiting → done     （待っている間に解決）

pending → inbox    （再検討）
```

## 5. API設計

### 5.1 エンドポイント一覧

バックエンドのルーティングは`router.php`経由で`index.php`に委譲。
APIプレフィックス: `/api` (Viteプロキシ) → 内部的には`/`にストリップ。

| メソッド | パス | コントローラ | 概要 |
|:--|:--|:--|:--|
| GET | `/items` | ItemController | アイテム一覧取得 |
| GET | `/items/:id` | ItemController | アイテム詳細取得 |
| POST | `/items` | ItemController | アイテム作成 |
| POST | `/items?action=reorder_focus` | ItemController | focusの並べ替え |
| PUT/PATCH | `/items/:id` | ItemController | アイテム更新 |
| DELETE | `/items/:id` | ItemController | アイテム削除 |
| GET | `/today` | TodayController | 今日のView用データ取得 |
| GET/POST | `/calendar` | CalendarController | カレンダーデータ |
| GET/POST | `/projects` | ProjectController | プロジェクト操作 |
| GET/POST | `/side-memos` | SideMemoController | サイドメモ操作 |
| POST | `/auth/login` | AuthController | ログイン |
| POST | `/auth/register` | AuthController | ユーザー登録 |
| GET | `/assignees` | AssigneeController | 担当者一覧 |
| GET/POST | `/tenants` | TenantController | テナント操作 |
| GET | `/users` | UserController | ユーザー操作 |

### 5.2 認証

- JWT Bearer Token認証
- リクエストヘッダー: `Authorization: Bearer <token>`
- クエリパラメータ: `?token=<token>`（フォールバック）

### 5.3 アクセス制御

アイテムの可視性ルール:
1. **個人タスク**（tenant_id IS NULL）: created_by = 自分 OR assigned_to = 自分
2. **会社タスク**: 所属テナントのアイテム AND（created_by = 自分 OR assigned_to = 自分）
3. **プロジェクトフォーカスモード**: 上記に加え、focusしたプロジェクトとその子孫のアイテムも表示
