## 2. 各機能の詳細

### F-01: アイテム管理

アイテムはタスクとプロジェクトの総称。すべてitemsテーブルに統一保存される。

**アイテムが持つフィールド:**
- タイトル、メモ、ステータス
- 納期（due_date）、マイ期限（prep_date）
- 目安時間（estimated_minutes）、目安日数（work_days）
- 所属会社（tenant_id）、担当者（assigned_to）
- 所属プロジェクト（project_id）
- ドメイン区分（private / business）
- アーカイブフラグ（is_archived）、削除日時（deleted_at）

**プロジェクト化:**
- `is_project = 1`でプロジェクト化
- 子タスクは`project_id`で親プロジェクトに紐付く
- プラグインの作用でフィールドを増やす仕組みあり

### F-02: ステータス管理（拡張: R-028）

詳細は [04_データ設計.md](04_データ設計.md) の状態定義を参照。

アイテムは6つの状態を持つ:

| 状態 | 説明 | キャパシティ計算 |
|---|---|---|
| `inbox` | 思いつき投入直後（未判断） | 含む |
| `focus` | 今日やる／集中対象 | 含む |
| `waiting` | 外的要因待ち（情報・決裁・入金待ち等） | 含む（待ち時間として） |
| `pending` | 保留（判断不能で先送り） | 含む |
| `someday` | いつかやる（自分の意志で寝かせる、期限なし、義務感なし） | **除外**（やる気を圧迫しない夢リスト） |
| `done` | 完了 | 集計のみ（実績） |

**基本フロー:**
1. **発生**: `inbox`（思いつきをInboxに投入）
2. **判断**: inbox → focus / waiting / pending / someday / done
3. **実行**: focus → isEngaged: true（実行中）→ done（完了）

#### 状態遷移ルール
- `inbox → someday`（判断結果として「寝かせる」）
- `someday → focus`（やる決断、希望日入力可）
- `someday → done`（やらないと決めた）
- `someday → inbox`（再判断したい）
- `pending → someday`（外的要因待ちを解除して自発的延期に変換）

#### pending と someday の差別化（必須要件）
- `pending` = **外的要因待ち**（自分は動けない）：⏸ アイコン、amber 系
- `someday` = **自発的延期**（自分が動かないと決めた）：💭 アイコン、purple 系
- ラベル文言: 「保留（外的要因待ち）」「いつかやる（自分で寝かせる）」

#### キャパシティ計算からの除外（Youkan独自の差別化軸）
`someday` 状態のアイテムは QuantityEngine の集計対象外。これにより「やる気を圧迫しない夢リスト」として静かに存在する。

**完了日の記録:**
- ステータスが `done` に変更された瞬間に `completed_at` を記録する
- `done` から他のステータスに戻した場合は `completed_at` を NULL にリセットする
- `completed_at` は量感カレンダーで「この日に何を完了したか」を振り返るために使用する

### F-03: Quick Input

Stream View上部に配置される高速入力欄。
- 思いついた瞬間にタイトルだけでInboxに放り込める
- `Ctrl+I`でフォーカス
- 「考えながら入力しない」ための0秒投入口

### F-04: 判断詳細（Decision Detail Modal）

すべてのViewから呼び出し可能な「判断のための個室」。

**表示内容（順序厳守）:**
1. **判断対象**: アイテム名と判断内容
2. **判断材料**: 納期・マイ期限（ここで初めて表示される）、想定日数等
3. **選択**: Yes（今日やる）/ Hold（今は決めない）/ No（見送る）

**ルール:**
- 納期・残日数はこのモーダル内でしか見えない
- 赤色・警告アイコン禁止。「あと○日」表現禁止
- 確認ダイアログなし（ボタン一発確定）
- 戻る = 未判断

### F-05: Morning Planning（朝の段取り）

focusステータスのタスクの実行順序をドラッグ&ドロップで並べ替える機能。

**思想:**
- システムは「この順番でやるべき」とは言わない
- 「今日focusに入っているタスクはこれだけです」と事実を提示する
- ユーザーが自分でドラッグして順番を決める

**実装:**
- Stream ViewのNext Strategyセクションが対象
- `@dnd-kit/core` + `@dnd-kit/sortable`使用
- 並べ替え完了時に`POST /items?action=reorder_focus`で`sort_order`を自動保存
- ソート: `sort_order ASC` → `updated_at DESC`（フォールバック）

### F-06: 量感カレンダー

日別の忙しさをヒートマップ（色の濃淡）で可視化するカレンダー。

**表示ロジック:**
- 各日付のキャパシティ（MAX値）に対するタスク合計時間の割合で色を決定
- 薄い色 = 余裕あり、濃い色 = 満杯、さらに濃い色 = 超過

**目安期間の算出（Backwards Allocation）:**
1. 基準日（納期 > マイ期限の優先順位で決定）から過去へ遡る
2. 稼働日のみ工数を消化（休日・キャパ0日はスキップ）
3. 消化した日付を「青枠」で描画

**操作:**
- 日セルをシングルクリック → 背景色に影響するタスクへ曲線で指示
- 日セルをダブルクリック → 影響タスクの一覧表示
- アイテムカードをクリック → 影響するセルが光る

**完了タスクの表示（振り返り機能）:**
- カレンダーの各日付セルで、その日に完了したアイテム（`completed_at`がその日のタイムスタンプ範囲内）を確認できる
- 「この日に何をしたか」を振り返る用途

**キャパシティの分母（フィルタ連動）:**

| 場面 | 分子 | 分母 |
|:--|:--|:--|
| 個人×プライベート×全て | 自分の全アイテム | 業務キャパシティ合計 |
| 個人×プライベート×個人 | 個人タスクだけ | 個人キャパシティ |
| 個人×プライベート×会社 | 会社タスクだけ | 会社キャパシティ |
| 個人×会社モード | A社タスク全件 | 主力社員のA社キャパ総和 |
| 会社×社内モード | domain:privateタスク | 社員キャパ集計 |
| 会社×会社モード | domain:businessタスク | 社員キャパ集計 |

### F-07: プロジェクト管理

- Inboxに1行書くだけでプロジェクト化可能
- 後から詳細画面で子タスクを追加（有機的成長）
- プロジェクトフォーカスモード: 特定プロジェクトの全タスクのみ表示

### F-08: マルチテナント

**アカウント種別:**
- **個人アカウント**: 一人の人間。自分の生活（My Tasks）を管理
- **会社アカウント（法人格）**: 会社そのもの。メンバー管理・設定用

**所属関係:**
- **メンバー**: 法人格によって追加された個人アカウント
- **オーナー**: 管理権限を持つ個人アカウント

### F-09: フィルタ/モード切替

**個人アカウントのフィルタ:**

| フィルタ | 表示内容 |
|:--|:--|
| 統合（Integrated） | 個人タスク + 所属全テナントのタスク |
| 個人（Personal） | 個人タスクのみ |
| 会社（Company） | 所属テナントの業務タスクのみ |

**プロジェクトフォーカスモード:**
- ステータスの「Focus」とは別概念
- ダッシュボードの表示スコープを特定プロジェクトに限定

### F-10: Side Memo

- 「忘却の退避所」
- どのViewからでも出し入れ可能
- 思いついたことを即座にメモし、後でInboxに移動

### F-11: キャパシティ管理

- テナント別の週間パターン設定（曜日ごとの稼働時間）
- 日次例外オーバーライド（祝日だが稼働、体調不良で早退等）
- キャパライン: focusキューの累積時間がDaily Capacityを超えた地点に赤線

### F-15: キーボードショートカット

| ショートカット | 動作 |
|:--|:--|
| Ctrl+G | Stream (Inbox) へジャンプ |
| Ctrl+I | Quick Inputにフォーカス |
| Alt+D | 直前に操作したアイテムの詳細を開く |
| Delete | 選択中のアイテムを削除 |

### F-18: 親→子孫カスケードの一般則

Youkanでは `parent_id` / `project_id` で表現される階層において、親アイテムへの操作は配下の子孫すべてに自動伝播する。

#### 対象操作と伝播内容

| 操作 | 子孫への伝播 |
|---|---|
| テナント変更（`tenant_id` 更新） | 子孫の `tenant_id` を移動先に揃える |
| プロジェクト変更（`project_id` 更新） | 子孫の `tenant_id` のみ追従（`project_id` 構造は保持） |
| アーカイブ（`is_archived = 1`） | 子孫も `is_archived = 1` |
| ゴミ箱（`deleted_at = ?`） | 子孫も同様 |
| 復元（`is_archived = 0, deleted_at = NULL`） | 子孫も同様 |
| 物理削除（`DELETE`） | 子孫も物理削除 |

#### 親子関係の判定

`is_project` の値に依存しない。`parent_id` または `project_id` で参照される全ての子孫が対象。
これにより、通常タスク（`is_project=0`）の親をアーカイブした場合も、その子subtaskは連動して非表示になる。

#### 権限チェック

テナント変更時、移動先 `tenant_id` がユーザーの `joinedTenants` に含まれていなければ HTTP 403 を返す。
これにより未所属テナントへの越境転送を防止する。

### F-19: ForAI機能（AI連携エクスポート）

現在のフィルタで表示中のアイテムを AI に渡しやすい markdown 形式に変換してエクスポートする機能。

**概要:**
- 好みの AI チャットにあなたの現在の状況を渡し、タスク相談が可能になる
- フィルタを変えるとモーダル内容もリアクティブに更新される

**ボタン位置:**
- PC: `YoukanHeader` 右側、Settings ボタンの隣
- スマホ: ハンバーガーメニュー（`MenuDrawer`）の Tools セクション内

**モーダル構成（`ForAiModal.tsx`）:**
- 上部: 立場ラベル（例: 「個人」「藤田建具店」「青年部のマネージャー」）
- 中央: テキストエリア（生成済み markdown、コピペ可、見切れ OK）
- 下部: 「クリップボードへコピー」ボタン + 「md ダウンロード」ボタン

**出力対象（現在のフィルタで表示中の全アイテム）:**
- `executionItem`（実行中）
- `todayCommits`（今日やる確定）
- `todayCandidates`（Focus 候補）
- `gdbActive`（Inbox）
- `gdbPreparation`（Ready）
- `gdbIntent`（Pending）
- Someday は含めない（夢リスト哲学、R-028 と整合）
- log（done）も含めない

**出力形式（markdown）:**
```markdown
# Youkan タスク状況 (YYYY-MM-DD)
## 立場: 藤田建具店

## 実行中
- [ ] (2F組子製作) 寸法決め — 納期: 2026-05-10 / 目安: 2h / メモ: ...

## 今日やる（確定）
- [ ] ...

## 今日やる（候補）
- [ ] ...

## Inbox
- [ ] ...

## Ready
- [ ] ...

## Pending
- [ ] ...
```

各アイテムに含める情報: タイトル、所属プロジェクト名、納期、マイ期限、目安時間、目安期間、メモ、担当者、所属会社（テナント）。
プロジェクト構造は階層インデント、最大3段で打ち切り（深い場合は … で省略）。

### F-20: タスク読み上げ機能

スマホで直近タスクを読み上げる機能。Web Speech API（ブラウザ標準、無料）を使用。

**概要:**
- 「今なにをしないといけないんだっけ」というときにタスクを読み上げてくれる秘書
- 運転中・歩きながらの利用を想定（スマホ即アクセス重視）

**ボタン位置:**
- PC: `YoukanHeader` 右側、ForAI ボタンの隣
- スマホ: フローティング（`fixed bottom-20 right-4 z-50`、SideMemoWidget と衝突しない位置）

**読み上げ画面（`SpeechView.tsx`、フルスクリーンオーバーレイ）:**
- 上部固定: 立場ラベル + 閉じるボタン
- 中央: 歌詞型スクロールリスト（現読み上げ中は中央でハイライト、上にスクロール完了済、下に未読）
- 下部固定: ⏮ 10秒戻る / ⏯ 再生・一時停止 / ⏭ 次へ / ✕ 停止

**読み上げ順序:**
1. `executionItem`（実行中）
2. `todayCommits`（今日やる確定）
3. `todayCandidates`（Focus 候補）
4. `gdbActive`（Inbox）
5. `gdbPreparation`（Ready）
6. `gdbIntent`（Pending）

各グループ内は既存 `compareFocusItems`（`logic/sorting.ts`）で `min(目安期間初日, マイ期限, 納期)` の早い順。

**除外:**
- 完了は読まない
- Someday も読まない

**読み上げ文言:**
「{プロジェクト名} の {アイテムタイトル}」基本形。

### F-21: スマホフィルター（R-033）

スマホ画面（<768px）でフィルター操作を提供するUI。

- ヘッダー右に Filter アイコン（漏斗マーク）＋現状フィルタラベル（例: "藤田建具店"）を表示
- タップで MobileBottomSheet を開く
- 選択肢: 全て / 個人 / 会社 / 各テナント（ViewContextBar 相当の一覧）
- 選択で filterMode を更新＋シートを閉じる
- PC では従来の ViewContextBar タブ群を維持（スマホのみ適用）

### F-22: スマホ用その他メニュー（R-033）

DecisionDetailModal の「その他」操作をスマホでボトムシート化する機能。

- 「その他...」ボタン押下時、`useIsMobile()` で分岐
  - スマホ: MobileBottomSheet を展開
  - PC: 従来のドロップダウンを維持
- メニュー項目: プロジェクトに変換 / 完了 / いつかやる / アーカイブ / ゴミ箱
- 既存の項目・動作は変更なし（表示形式のみ変更）

## 3. 外乱（割り込み）の扱い

- 電話や急な依頼は「タスク追加」ではなく**「判断追加」**として扱う
- InboxまたはPanoramaBoard（状況把握ボード）に入れる。いきなりExecution（作業）には入れない
- `interrupt: true`フラグでInbox最上部に強調表示

## 4. デバイスメタファー

| デバイス | メタファー | 用途 |
|:--|:--|:--|
| PC | 「机」 | 思考するための広さを確保。情報俯瞰 |
| スマホ | 「一枚の紙」 | 今やること（Today）に一点集中。横画面禁止 |
