# 01. Youkan 概要

## 1. Youkanとは

**Youkan（羊羹）** は「Judgment-Free Work-life Operating System」。
タスク管理ツールではなく、**判断を正しいタイミングで少量ずつ終わらせるためのOS**。

名前の由来は目玉機能「量感カレンダー（Ryoukan Calendar）」から。
旧称は **JBWOS（Judgment-Based Work Operating System）**。コード内にJBWOSの名称が残っているが、Youkanへの統一を進めている。

---

## 2. 誰のためか

### 主要ターゲット: 晴樹（開発者本人）

- 32歳、既婚、子供あり。日中の仕事時間を有効活用したい
- 建具店経営（打合せ → 現場調査 → 見積 → 受注 → 設計 → 材料発注 → 製造 → 搬入 → 施工 → 請求）
- **苦手なこと**: タスク管理、スケジュール管理、進捗管理、仕事を断ること
- 思い出したタスクを1分後には忘れてしまうことがある
- 製造に5日かかる仕事のスケジュールを組んでいても、途中で他の仕事が入り納期遅れになる

### 拡張ターゲット

- 個人事業主の職人（鉄工所、家具屋、ハンドメイド作家）
- 事務作業より現場や製作が好きだが、事務もやらなければならない人

---

## 3. 解決する課題

### 判断疲れ（Decision Fatigue）
膨大なタスクの中から「次に何をすべきか」を決めるだけで脳が疲弊する。Youkanはこの負荷を極限まで減らす。

### スケジュールの可視化不足
「このあたりは忙しい」「ここは空いている」が見えない。量感カレンダーで色の濃淡として直感的に把握可能にする。

### 仕事の受け過ぎ
空き状況が見えないため「まあできるだろう」と受けてしまい、結局できない。量感カレンダーで受注可否を即答できるようにする。

---

## 4. 核心思想

### 3つの「しない」（Anti-Patterns）

| 原則 | 内容 |
|:--|:--|
| **評価しない** | 進捗率、達成度、ストリークなどでユーザーを格付けしない |
| **急かさない** | 納期、残日数、警告色、未達通知で圧力をかけない |
| **混ぜない** | 「決めること（判断）」と「やること（実行）」を同じ空間に置かない |

### 早期完了のゲーム化（Backwards Allocation）
- 未来の締め切り直前に仕事の塊（オレンジ色）を表示する
- 手前は空白（緑色）に見える
- 「未来のオレンジを消すために、今の緑を使う」というゲームにする
- 「やらなきゃいけない」ではなく「今やっておけば未来が楽になる」というポジティブな動機づけ

### 最終判断基準
> 迷ったら「晴樹が朝の判断を30秒で終えられるか？」を問いかけること。
> これを阻害する機能は、たとえ便利でも実装してはならない。

---

## 5. 製品の存在意義

効率化ツールそのものだが、目指すのは**精神的充足**。

- 「忙しい時期」と「暇な時期」をぼんやり可視化し、「この時期なら創作に没頭できる」という気づきを与える
- やりきれない量のタスクをキャパラインで可視化し、「できない自分」を責めずに「物理的に無理」と納得させる
- 稼ぐべき金額と自由に使える時間のバランスを計算し、**仕事を早く終わらせれば自由だという確信と解放感**を与える

## 6. 用語定義

主要な用語については [00_Vision/JBWOS_TERMINOLOGY_DICTIONARY.md](../00_Vision/JBWOS_TERMINOLOGY_DICTIONARY.md) を参照。
以下は特に重要なもの。

| 用語 | 定義 |
|:--|:--|
| **アイテム** | タスクとプロジェクトの総称。itemsテーブルに保存 |
| **タスク** | プロジェクト化されていないアイテム。サブタスクを持たない |
| **プロジェクト** | サブタスクを持つことができるアイテム |
| **納期** | そのタスクを必ず終えなくてはいけない期日 |
| **マイ期限** | 納期よりも手前で完了しておきたい自分なりの期限 |
| **目安時間** | タスク達成にかかると見込んでいる時間（単位: 分） |
| **目安期間** | 基準日から工数を稼働日ベースで逆算した占有期間 |
| **Engage（実行中）** | 現在のタスクに手を動かしていると宣言した状態 |
